What the flow verifies

Per module M and reader R: (1) lhd compile M.sv --reader R --emit-dir lg:orig --emit-dir verilog:gen; (2) re-read gen/M.v via yosys-verilog (its flatten inlines the cgen_memory_* instances back to a top-level memory); (3) lhd lec --ref lg:orig --impl lg:gen. Because the re-read goes through yosys, every slang row is effectively a cross-reader proof (slang-encoded vs yosys-encoded) — which is what exposed the encoder bugs below (they all cancel in same-reader miters, so the pre-existing tests missed them). lhd check (yosys equiv) is the independent oracle used to tell a real reader/cgen bug from an LEC false-refute.

new Blackbox Sub collapse — missing/empty leaf modules

Commits b819f2b36 + 3dd425a45 (pass/lec encode/query + cgen). The XSCore SRAM macros (array_*_ext, ram_ext, …) have no RTL, and per-module verification ("one module at a time") leaves every child empty. A (* blackbox *) port-only stub makes all three readers keep the instance as a Sub (without the attribute, yosys flatten deletes an undriven-output empty module). In lhd lec a Sub with no resolvable combinational definition is collapsed: each output becomes one fresh symbol shared across the two designs (keyed module#occ:port, reader-invariant), and each input becomes a miter comparison point — truncated to the declared port width, since a wider driver is truncated by the connection (commit 5; this fixed the SRAM-boundary REFUTE class). Sound when both designs carry the instance: identical inputs (checked) ⇒ identical (shared) outputs. Verified: array_0 (wraps the missing array_0_ext) and SRAMTemplate round-trip PROVEN via all 3 readers; a corrupted blackbox input REFUTEs. A cgen bug was fixed en route (a Sub output driving a module port was re-declaring the port).

Remaining failure taxonomy

  • real reader/cgen slang-reader correctness bugs — lhd check (yosys-equiv oracle) also fails, so cgen genuinely differs. Narrow and per-module: AgeDetector, Debug, FIFOReg, wide address concat (HPTW), data forwarding (SQData8Module). The real engineering tail.
  • infra cvc5 checkSat SEGFAULT on large SAT instances (multipliers / FP: ClmulModule, FcsrModule, Conversion_*, CSA*) — the documented abc↔cvc5 CaDiCaL symbol clash (docs/cvc5_bug.md); both vendor CaDiCaL, ≥1 global still collides at the lhd link. Unfixed (deep build issue).
  • memory memory cross-reader (ram_40x47, multi-clock SRAMs) — native vs yosys structure memory ports differently (the M4 layer); some memory rereads fail (cgen_memory include / multi-clock binding).
  • lec 1 unisolated LEC false-refute (FTBEntryGen, wide-address arithmetic; the minimal reproducer PROVEs, so the trigger is structural/downstream).

Full-design status

Flat whole-XSCore LEC remains impractical: the yosys-slang reference path does an unconditional flatten, and on full XSCore that runs >25 min / 21 GB without reaching write_rtlil. Per-module verification (children blackboxed) is the working path and is what this report measures; a hierarchical/congruence LEC composing the per-module proofs is the natural next step.
