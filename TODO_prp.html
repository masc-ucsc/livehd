  • 1p Deferred lambda templates: var-args + call-site specialization — spec'd 2026-06-07 (contract docs/contracts/task_1p_plan.md). A not-fully-typed lambda — untyped non-self input, a (...args) var-arg, or an unbound generic <T> — becomes a template: kept as LNAST, lowered to no LGraph at definition time; the concrete form is produced per call site. This unifies the two existing paths: comb templates inline (the 1i path, extended to bind ...args), pipe/mod/fluid templates specialize into a concrete module per signature and instantiate as a Sub (1u-A). Removes the lambda-needs-types definition-time error (prp2lnast.cpp:4837) so var-args/generics work on a module boundary.

  • Decisions (locked 2026-06-07):
    • Scope. This goal builds the deferred-template + call-site specialization mechanism with var-args as the driving use case (comb inline + pipe/mod/fluid specialize). Generics <T> get a designed-for seam (markers + mangling reservation) but per-T body substitution is a separate follow-up goal.
    • Naming = readable suffix. Specialized modules get a human-readable suffix from the concrete arg types/shape, counter on collision: foo(u8)→foo__u8, foo(u16)→foo__u16, add1(a,b,c:u8)→add1__3xu8, generic T=u22→add6__T_u22, collision→foo__u8_1. Same signature → same name → natural dedup (one shared module, each call its own Sub instance).
    • No in-unit call site = template-only. A pub export or top-level template never specialized in its own file stays a deferred LNAST template and produces no LGraph — no error; it materializes only when an importer/caller specializes it. The old lambda-needs-types error is deleted.
    • Specialize early, keyed on the actual's DECLARED type. pipe/mod/fluid specialization runs at the call site during the runner walk (beside the 1i inliner), keyed on each actual's declared type — no dependence on late width inference, no pipeline re-entry. An untyped actual feeding an untyped/var-args boundary port is a call-site error ("annotate the actual"). comb is unaffected (always inlines). Inferred-width support, if ever wanted, layers on as a deferred late-specialize pass.

  • Declare-folding — bind the initial/reset value ON the declare. A declaration with an initializer currently lowers to declare(var, type, mode) followed by a separate store(var, value) (e.g. mut c:u8 = nil → declare(c, u8, mut) then store(c, nil); confirmed 2026-06-01). For a register/flop this split loses the "this is the value bound at declaration" relationship: a later same-name store is structurally indistinguishable from the initializer, so the power-on / reset value can't be reliably recovered when the declare lowers to Fflop / Latch (and prim_type_register's reset_pin needs exactly that value). Fold the initializer into the optional 4th [value] child of declare — the slot already sketched in the landed declare( var, <type>, const(mode), [value] ) — so the init rides on the declare node and only genuine re-assignments emit a trailing store. Producer: rewrite_decls_to_declare / process_lvalue_for_assign in inou/prp/prp2lnast.cpp (the decl-cluster merge already has both the declare and its value store in hand); consumers process_declare in upass/attributes + upass/bitwidth and the lnastfmt declare-shape validator must accept the 4th child. (Context: the assign-retype guard landed 2026-06-01 already establishes that a type cast may ride only a declare or a wrap/sat write — see inou/prp/tests/errors/assign_retype.prp.)
