Group 1 — foundation

  • 1m Pyrope pub exports + import() — spec'd 2026-06-07, design contract in docs/contracts/task_1m_plan.md (supersedes the deleted root import.md; absorbs ex-2o — the two-phase func_extract / Inline_reason / pending-import-poison model is dead).
  • Design summary:
    • pub legal on file-scope comptime/const values (initializer must fold comptime at elaboration, else error) and on comb/mod/pipe/fluid. pub mut/pub reg = error. Every entity still gets its own tree (foo.prp → unit foo, entity bar → tree foo.bar, the existing func_extract convention); pub only controls membership in the file's pub list.
    • import("file1") = the whole pub tuple (values + lambda tree-url refs), read-only always; no dot selector — pick fields with normal dot access. Direct access via urls: import("ln:file1.bar") (any tree, pub not required) and import("lg:any.graph.name") (any graph; black box at LNAST, Sub-wired by GraphIO at tolg). Statement form kept as sugar: import "file1" as b ≡ const b = import("file1") (string-literal module only — dotted-identifier choice removed).
    • Resolution against explicit inputs only: the invocation's other .prp files + loaded ln:/lg: dirs (manifest pub index; search crosses forests). No filesystem discovery. Missing = error, ambiguous across inputs = error.
    • Pub persistence = synthesized foo.__pub wrapper tree (arbitrary comptime values ride LNAST serialization) + manifest.json pub index per unit.
    • Orchestration: bazel runs one lhd elaborate per file in parallel (ln: deps ride along); a multi-file invocation iterates — any order, whole-file retry when a live import is unresolved (liveness needs constprop: if cond { import }), pub list published atomically at file completion, no-progress round = error. No topological pre-ordering, no pending markers.
  • Implementation phases (detail in the contract): A front-end (pub recording, pub mut/reg errors, import lowering); B export side (wrapper tree + manifest pub index); C import side (resolution at the constprop/runner seam, read-only bind, defer signal, equals = tree-url identity); D kernel iterate loop + registry + diagnostics; E lg: black-box Sub wiring; F tests.
  • Tests to add when landing:
    1. Pub-value import, same invocation: pub comptime foo = 3 → cassert(x.foo == 3); read-only (write to x.foo errors).
    2. Lambda import + call: comb via tuple field and via ln: url (inline splice); mod/pipe → Sub instance; cassert(x.bar equals bar2) tree-url identity.
    3. Two-invocation flow through lhd: elaborate exporter to ln:dir, then importer with ln:dir riding along.
    4. Verilog-derived lg: import (black box; no Verilog-side pub needed — lg: is unrestricted).
    5. Conditional import: dead-branch import of a missing unit is not an error; live one resolves.
    6. Diagnostics: missing unit, ambiguous unit across two inputs, unknown (non-pub) field on the pub tuple, pub mut/pub reg, non-foldable pub value, import cycle (no-progress).

  • Declare-folding — bind the initial/reset value ON the declare. A declaration with an initializer currently lowers to declare(var, type, mode) followed by a separate store(var, value) (e.g. mut c:u8 = nil → declare(c, u8, mut) then store(c, nil); confirmed 2026-06-01). For a register/flop this split loses the "this is the value bound at declaration" relationship: a later same-name store is structurally indistinguishable from the initializer, so the power-on / reset value can't be reliably recovered when the declare lowers to Fflop / Latch (and prim_type_register's reset_pin needs exactly that value). Fold the initializer into the optional 4th [value] child of declare — the slot already sketched in the landed declare( var, <type>, const(mode), [value] ) — so the init rides on the declare node and only genuine re-assignments emit a trailing store. Producer: rewrite_decls_to_declare / process_lvalue_for_assign in inou/prp/prp2lnast.cpp (the decl-cluster merge already has both the declare and its value store in hand); consumers process_declare in upass/attributes + upass/bitwidth and the lnastfmt declare-shape validator must accept the 4th child. (Context: the assign-retype guard landed 2026-06-01 already establishes that a type cast may ride only a declare or a wrap/sat write — see inou/prp/tests/errors/assign_retype.prp.)
