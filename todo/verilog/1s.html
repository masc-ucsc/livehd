1s Direct slang reader v11 foundation

Upgrade the direct --reader slang path (inou/slang) from slang v10.0 to the latest release tag v11.0, preserving the currently passing direct-SV-to-LNAST tests. This task does not replace yosys-verilog or yosys-slang; those remain the production Verilog-to-LGraph readers. 1s is the foundation that makes the direct slang reader buildable, diagnosable, source-located, and safe to grow in 2s.

Depends on

  • 1f (landed 2026-06-10) — supplies the HHDS Source_locator, SourceId attr, and located diagnostic plumbing that inou/slang should use instead of ad-hoc file/offset reporting. Decided 2026-06-10: hard ordering — 1f landed 2026-06-10; 1s. The importer is then born on the final provenance system: subtask D mints SourceIds directly (no interim Lnast::set_loc/set_fname migration), and subtask C populates diag::Span.source_id instead of the pre-1f file/byte-offset fallback fields.

Current state (verified 2026-06-10)

  • Pin. MODULE.bazel pins slang v10.0 as an http_archive built by rules_foreign_cc CMake (packages/slang.BUILD, Ninja, static libsvlang.a) with a 39-line packages/slang.patch that replaces include(external/CMakeLists.txt) with a direct fmt add_subdirectory via FMT_CMAKE_PATH, swaps Boost::headers for an EXT_BUILD_DEPS include hack, and installs expected.hpp. The @fmt archive (11.1.4) exists only for slang.
  • v11.0 build facts. CMake ≥3.20, C++20. fmt minimum is 12.1 (FetchContent of 12.1.0) — the @fmt archive must be bumped. SLANG_USE_BOOST no longer exists: external/CMakeLists.txt does find_package(Boost 1.87 CONFIG) and falls back to a vendored single-header boost_unordered (defines SLANG_BOOST_SINGLE_HEADER). SLANG_USE_MIMALLOC still exists but now defaults ON (keep OFF). New optional FetchContent deps (mimalloc 3.3.2, cpptrace 1.0.4) must stay disabled — the Bazel sandbox has no network, so every FetchContent path has to be bypassed by the patch.
  • Driver. slang_driver.cpp is a near-verbatim copy of upstream's v10 tools/driver/driver.cpp including -E/--macros-only/--ast-json/--time-trace passthroughs LiveHD never uses. inou_slang.cpp feeds only --quiet --ignore-unknown-modules plus -I/-D/-U, and runs one full driver per file on the in-process thread_pool with two FIXME: slang multithread fails comments already in place. Inou_slang's constructor passes "pass.lec" as its pass name — a stale copy-paste to fix in B.
  • Diagnostics. core/diag.hpp already provides livehd::diag::Diagnostic/Span/Sink (JSONL, category taxonomy including unsupported, notes/hints), and lhd checks the sink after every step. Post-1f, spans carry source_id + resolved line/col.
  • Tests. 16 per-case slang_compile-* sh_tests (add, add1, add2, assigns, compare2, consts, issue_047, mt_basic_test, offset_input, satlarge, satsmall, simple_weird2, trivial, trivial_and, trivial_join, trivial_offset) drive lhd compile --reader slang + lhd synth ln: round-trip; slang_compile.sh default mode also covers inou/slang/tests/verilog/*.v (sky130 cases), and slang_test.sh smoke-runs the standalone slang binary with --ast-json. The pass/fail comment block in inou/slang/BUILD is stale — refresh it in F.

Scope

  • Use the upstream slang v11.0 release tag, not top of tree. Port MODULE.bazel, packages/slang.BUILD, and packages/slang.patch to the new release and its API changes (notably the ASTVisitor/VisitFlags breakage called out in the release notes).
  • Keep the direct reader's current behavior envelope: build cleanly, run the existing inou/slang tests, and keep the same currently passing slang_compile-* cases passing. Any behavior expansion belongs in 2s.
  • Replace stdout/stderr FIXME / Pass::info parse failures with structured diagnostics through LiveHD's diagnostic sink. Unsupported constructs should fail cleanly with source spans and hints, not assert, abort, or synthesize placeholder names such as FIXME_op.
  • Mint SourceIds for LNAST nodes from slang source ranges using the HHDS Source_locator. This is an ingress task: the direct reader should attach provenance while building LNAST, then the normal ln: save/load path should preserve it.
  • Add a direct-reader option to suppress generated mod timechecks for slang input. This belongs in 1s because any existing flop/sequential module lowered as mod can fail before 2s improves unit-shape inference.
  • Preserve parallel build-system usage. Multiple independent lhd elaborate ... --reader slang / lhd synth ln:... invocations must be safe to run concurrently under Bazel or other schedulers. Keeping the old in-process thread-pool parsing is optional; if slang v11 makes it brittle, prefer one isolated Driver/Compilation per invocation and let the build system expose parallelism.

Reference points

  • Current LiveHD shape. inou/slang/slang_driver.cpp embeds a local slang driver and slang_tree.cpp directly lowers a small subset of top modules, ports, continuous assigns, narrow always blocks, simple expressions, bit/range selects, and concatenations into Lnast_builder.
  • CIRCT. samples/circt/lib/Conversion/ImportVerilog is the best reference for v11-compatible diagnostics and systematic AST visitor coverage; use it as a pattern for source range conversion and explicit unsupported diagnostics.
  • yosys-slang / OpenROAD / Dyno. These are more useful for 2s. For 1s, only borrow small compatibility and driver ideas when they reduce upgrade risk.
