2s Direct slang reader cleanup and coverage

Refactor inou/slang after the 1s slang v11.0 upgrade so the direct --reader slang path is a maintainable SystemVerilog-to-LNAST importer instead of a narrow slang_tree.cpp switch. The goal is not to replace the yosys-verilog / yosys-slang production readers; it is to make the direct LNAST path cleaner and to pass substantially more Verilog tests when emitting ln: / lnast-dump:.

Depends on

  • 1s — slang v11.0, structured diagnostics, source locations, and current-test preservation.
  • 1a (landed 2026-06-11) — the LNAST→LGraph synthesis closure needed for tests that continue past LNAST into LGraph/codegen equivalence.

Design direction

  • Importer context, not global switch glue. Split the current Slang_tree into a lowering context with explicit module, expression, statement, type, lvalue, and diagnostic helpers. CIRCT's pattern is the model: one Context struct (samples/circt/lib/Conversion/ImportVerilog/ImportVerilogInternals.h) owns convertType / convertStatement / convertRvalueExpression / convertLvalueExpression / convertLocation plus scoped value-symbol tables, an lvalue stack for compound assigns, and a loop stack for break/continue, with per-concern visitor files (Structure.cpp, Expressions.cpp, Statements.cpp, Types.cpp). LiveHD should stay much smaller but keep the same separation of concerns.
  • Backend-shaped lowering. Borrow the OpenROAD/yosys-slang idea of a compact backend interface where expression/procedural code emits through reusable helpers instead of directly knowing every LNAST builder idiom. OpenROAD proves the seam works: its importer is the yosys-slang walker rebuilt against the ~30 virtual emission methods of samples/OpenROAD/src/syn/src/elab/backend_builder.h (Unop/Biop/Mux/Shift/add_dff/add_instance/…) — same walker, different IR. For LiveHD the seam already exists as Lnast_builder; grow it into the complete emission interface so visitor code never open-codes LNAST builder idioms.
  • Structured statements stay structured. All four reference importers target netlists, so they must flatten if/case into mux trees with per-branch variable-state snapshots (yosys-slang src/statements.h SwitchHelper + VariableState). LNAST is a tree: lower slang's structured statements 1:1 to LNAST if/match nodes and let the upass SSA machinery do branch merging. Borrow only the per-variable blocking-vs-nonblocking mixing check (yosys-slang src/procedural.cc ~196-240) as a located diagnostic.
  • One conversion boundary. CIRCT funnels every sign/width/domain change through a single materializeConversion helper (Expressions.cpp): truncate, then sext/zext chosen by source signedness, then domain change. Replace the sext/mask snippets scattered through slang_tree.cpp (the BinaryOp tail and the Conversion case) with one such helper used at every typed boundary.
  • Rvalue/lvalue discipline. CIRCT shares one expression visitor with an isLvalue flag so select/concat code is written once and an assignment converts the LHS first, pushes it on the lvalue stack, then converts the RHS. yosys-slang instead decomposes assignment targets into descriptor variants (src/lvalue.cc LValue::analyze(): variable, concatenation, range select, member access, memory write); Dyno carries dynamic indexing as compact address terms (AddressGenTerm{idx,factor,max}). Replace process_lhs's three-case switch with an lvalue descriptor lowered through assign/set_mask/concat splitting.
  • Two-tier constant evaluation. Try slang's own expr.eval() first and emit a literal; lower to LNAST only when the value is not compile-time (yosys-slang wraps this as EvalContext; Dyno const-folds at build time). For loop bounds, unroll slang-side with an explicit cap and a located unroll-limit diagnostic (yosys-slang UnrollLimitTracking, default 4000) rather than relying on the upass runner to unroll emitted loops; decide per construct in subtask B.
  • Clean unsupported-feature policy. CIRCT's default template visit() fallback emits unsupported <category>: <ast-kind> with the source range plus an attached note, so every AST kind is either handled or diagnosed — nothing falls through silently. yosys-slang adds a named diagnostic catalog (src/diag.h, 50+ codes) and an AST-JSON dump on abort (src/abort_helpers.cc) that is worth copying as a debug aid behind a flag; Dyno's ~32 bare abort() sites are the anti-pattern. LiveHD: unsupported SystemVerilog is a located, clean compile error, never an assert or placeholder LNAST.
  • Coverage grows from tests. Pull examples from the existing inou/yosys/tests cases and from samples/yosys-slang/tests/unit and tests/various. Prefer small focused tests that pin one feature through --reader slang to LNAST before adding broader compile/equiv tests.

Sample-repo reference map

Local clones under samples/. CIRCT is the architecture and diagnostics model, yosys-slang the synthesizable-semantics model, OpenROAD the proof that the walker/backend seam works, Dyno a compactness reference only.

  • CIRCT — samples/circt/lib/Conversion/ImportVerilog/: ImportVerilogInternals.h (the Context struct); Expressions.cpp (shared rvalue/lvalue visitor, assignment via lvalue stack, materializeConversion, system-call dispatch incl. $clog2/$signed); Statements.cpp (if/case/loop lowering with a loop stack); Structure.cpp (two-phase module header/body conversion, instance port matching, unconnected-port handling, generate-block name prefixing, convertProcedureKind, the always @(*)→always_comb option); Types.cpp (full slang type map with an unsupported-type fallback); ImportVerilog.cpp (convertLocation — the model for 1s-D SourceId minting).
  • yosys-slang — samples/yosys-slang/src/: memory.h (InferredMemoryDetector, the 2s-D reference); lvalue.cc (assignment-target decomposition); addressing.cc (dynamic bit/range select lowering: shift-based, with endianness base offset and stride for nested arrays); async_pattern.cc (TimingPatternInterpretor classifies always blocks and extracts clock edge / async-reset patterns — feeds 2s-C unit-shape inference); procedural.cc (blocking/nonblocking rules, unroll limits); diag.h/diag.cc (named diagnostic catalog).
  • OpenROAD — samples/OpenROAD/src/syn/: the importer under third-party/slang-elab/ is yosys-slang retargeted through the virtual interface in src/elab/backend_builder.h; src/syn/test/elab_testcases.h holds ~50 inline SV feature cases checked by SAT equivalence — a good shape for small direct-reader feature tests. Its SynthesisSettings struct (unroll limit, ignore-timing, ignore-initial, blackboxed modules) is a good template for inou.slang options.
  • Dyno — samples/dyno/tools/dyno-sv/include/Frontend.h: a ~2k-line single visitor; useful for its compact value model (RValue/RegLValue/ConcatLValue with inline address-generation terms for nested indexing) and its two-phase module collect-then-process; its bare-abort() error policy is what 2s must avoid.

Testing strategy

The Normal Computing CIRCT simulator writeup is a useful reference for test-suite shape: progress came from combining standard-compliance tests, real-world codebases, formal/equivalence checks, and mutation testing. LiveHD should apply that pattern only to the synthesizable --reader slang target, not to UVM, cocotb, VPI, or full simulator testbench semantics.

  • Feature compliance slice. Build a filtered, synthesizable subset from sv-tests-style cases, samples/yosys-slang/tests/unit + tests/various (~60 focused regressions), and the inline SV cases in samples/OpenROAD/src/syn/test/elab_testcases.h. Track pass/fail/unsupported by feature tag: expressions, selects, procedural, instances, params, arrays, memories, generate, and diagnostics.
  • Existing LiveHD regression ladder. Promote inou/yosys/tests cases into direct-reader expectations in stages: first "elaborates to non-empty LNAST", then "round-trips through ln:", then compile/equiv through the LNAST→LGraph closure (1a, landed 2026-06-11). By the end of 2s, every test source in inou/yosys/tests/ must have an equivalent --reader slang expectation at the strongest available tier; missing coverage is a failing acceptance gate, not backlog.
  • Real synthesizable designs. Add a small curated corpus from OpenTitan/Ibex-style modules and CVDP-style design problems, stripped to DUT sources and synthesizable wrappers. Test compile/LEC, not testbench runtime.
  • Differential oracle. For features both paths support, compare direct --reader slang output against yosys-slang through generated Verilog/LEC or stable structural observables. The direct reader can differ in LNAST shape, but synthesized behavior must match.
  • Mutation smoke. Add small mutations over passing DUTs (operator flips, width/sign perturbations, parameter changes, removed default cases) and require either an expected LEC mismatch or a located diagnostic. This guards against "parses more" hiding incorrect lowering.

Feature targets

  • Module structure. Instances, generated blocks, parameters, simple parameter overrides, port default values, named/ordered port connections, unconnected ports, and explicit blackbox/unknown-module diagnostics.
  • LNAST unit shape. Translate SystemVerilog modules to the most precise Pyrope/LNAST unit form that can be proven safely: emit comb for pure combinational modules, emit pipe for simple pipeline-shaped modules when the importer can prove the contract, and fall back to mod otherwise. The direct-reader mod timecheck suppression option is a prerequisite from 1s; 2s should use it while improving inference so fewer modules need the fallback. Seed the inference from always-block classification: slang's ProceduralBlockKind plus edge-in-sensitivity detection for plain always (Dyno's rule), and yosys-slang's async_pattern.cc TimingPatternInterpretor for clock-edge / async-reset pattern extraction.
  • Types and declarations. Nets vs variables, signedness, packed arrays, unpacked arrays where they map cleanly to LNAST tuples/memory, enum values, packed structs where bitstream lowering is enough, and clear diagnostics for unsupported classes/interfaces/modports until they are intentionally designed. Define an explicit x/z policy at the boundary: CIRCT models two-state vs four-state as a type domain converted in one place; the current reader leaks 0b? literals ad hoc. Decide where unknowns are allowed (constants, default cases) and diagnose the rest.
  • Expressions and lvalues. Full common arithmetic/compare/logic operators, shifts, reductions, ternary expressions, replication, streaming or explicit diagnostic, nested bit/range/member selects, concatenation lvalues, and correct sign/width conversion at every boundary.
  • Procedural code. always_comb, always_ff, initial where synthesizable or diagnostic-only, blocking vs nonblocking assignment handling, if/case lowering, bounded constant for/foreach unrolling, and clean handling of break/continue/return in supported contexts.
  • Memories. Use yosys-slang's memory-candidate analysis (samples/yosys-slang/src/memory.h, InferredMemoryDetector: pass 1 collects static fixed-range unpacked arrays at module scope, pass 2 disqualifies on blocking writes, mixed assignment styles, or lvalue shapes beyond element/range select) as the reference for deciding when arrays become memories versus scalarized LNAST, but emit the LiveHD LNAST memory vocabulary expected by the existing LNAST-to-LGraph path.
