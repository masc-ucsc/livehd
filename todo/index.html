    • 2n Pyrope LSP server — Phases B–E (hover/nav/workspace) remain.
    • 2i-import Import & library-link redesign — a library is a directory (flat short pub names), a module's identity is its forward-canonical structure digest (from 2f-semdiff, not name/path), and name collisions resolve at merge: structurally-identical modules dedup, distinct ones rename-both symmetrically (alu__riscv/alu__gpu) — never silently. The same .prp compiles in the interactive single-file flow (imports → sibling source, importer-dir-relative; fixes the LSP false "unresolved import") and a hermetic parallel-bazel flow (imports → artifacts via lhd scan). Closes porting-log friction A1/A7/2f/6a/6c/C1.
    • 2opt-import Lazy --top resolution — lhd compile --top XXX … ln:… today parses and elaborates every input, then picks XXX with a post-elaboration name filter (filter_top), body-loading ln: forests the top never touches. A cheap resolve pre-pass finds the one input that declares XXX (a pub in a .prp, or an exported unit in an ln: manifest.json — no forest body load) and walks its transitive import closure, opening an input only when reachable. Reuses the landed discover_imports fixpoint (2i-import S1), adds a lexer-only scan_pubs (sibling of lhd scan's scan_imports), and follows ln: edges so only the reachable subset is body-loaded.
    • 2m-multiport_mem Multiport / multi-clock memory codegen: the --reader slang chunked-wensize SRAM writes now elaborate, but tolg/cgen can't emit them — per-port clock binding (driving case XiangShan data_mem_8x4, three distinct port clocks → "no clock to bind" → truncated Verilog), the multiclock cgen_memory wrapper family (only 1rd_1wr exists), and a related single-clock array_ext.v cgen hang.
    • 2w-prp_writer pass.prp_writer codegen completeness: prp→prp re-emission now hard-errors on unimplemented constructs (safety net; --set prp_writer.debug=true overrides) and the pipeline (stage[N]/@[N]), type_spec and tuple_concat round-trip + LEC (new prp-p2p-* family). Remaining: the unimplemented timecheck/for/func_def nodes, ~34 equiv pairs that emit clean-but-not-equivalent Pyrope, and the v→prp derived-clock / multi-clock-memory cases.
    • 2f-semdiff Structural diff/match between two LGraphs — a structural LEC mirroring the lec API (--ref/--impl, two tops). Anchored frontier propagation (forward then backward, meet-in-the-middle) marks corresponding nodes/pins with a new greppable match attribute; the unmatched gaps are the diff. This is the MATCH piece feeding the lec reduction roadmap.
    • lec Relational equivalence engine roadmap — combinational, sequential (inductive flop-cut + BMC reset phases), memory arrays, and combinational Sub flattening have landed on the cvc5-direct backend (reference: pass/lec/lec.md); remaining is the fast-incremental def-diff reduction (M3, on 2f-semdiff), sequential/black-box assume-guarantee + CEGAR (M5), and the persistent certificate cache (M6).
    • 2p-piper pass.piper — the lgraph pipeline-timing pass that verifies @[N] (exact cycle-count type check) and inserts stage[N] flops, decided where the global σ timing lives. A new Ntype_op::Stage cell marks an undecided stage (allowed 0..n flop range; codegen is illegal while one survives); tolg shrinks to emitting Stage cells + attrs, and piper resolves them (min-legal depth, LEC-checked) — consolidating the timing logic that today is smeared across upass.timecheck (kept as best-effort early diagnostics), the inline tolg Time_checker (deleted), and upass.pipe (folded in).
    • 2f-formal Single-graph formal property engine on the lec Encoder — encode-once + collapse, deterministic cone-scaled rlimit budget. A reusable library (is_true/equal/bits_equal/is_constant) for future pass/memory & pass.abc callers, plus an always-on tier proving user assert/assume and built-in obligations — first up Hotmux selector one-hotness (no frontend work) — with the same prove ⇒ remove / disprove-over-free-inputs ⇒ error / else keep-for-runtime cascade, and a silenceable warning whenever something is deferred to runtime.
    • 2d-cex_debug Actionable lec counterexamples for the REFUTE case: report the first diverging internal cut (not the output it propagated to) source-mapped to a file:line (via flop_state_key()'s SourceId path + the cgen sourcemap), the full per-cycle primary-input trace, a machine-readable cex.json, and a one-command lhd lec --replay that re-simulates BOTH netlists and VCD-diffs them (reusing core/vcd_reader). Plus auto flop-correspondence from 2f-semdiff stamps so differently-named pipelines need no hand-built match.txt. Filed from the lec_flow.txt debugging retrospective.
