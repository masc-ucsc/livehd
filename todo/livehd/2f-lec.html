2f-lec LEC remainder — parallel hier driver, formal-block consumption, hints out, correspondence quality

How and why pass/lec works today is the reference doc pass/lec/lec.md. Landed: cvc5-direct combinational miter, inductive flop-cut + BMC-from-reset sequential phases, memory arrays, combinational Sub flattening, the auto ind+bmc fork portfolio with combinational skip, bottom-up hierarchical decomposition with proven-child black-box collapse (lec.hierarchical) + REFUTE-under-collapse flat confirmation, the per-module semdiff structural skip, the parallel input case-split (lec.partitions/lec.split=auto, 2026-07-10), and the witness→lecfail.prp→lhd sim→VCD flow. The hier driver is also now scoped to the picked top pair and its transitive descendants (fixed 2026-07-10 — it previously proved every def the two libraries happened to share). This page tracks what remains, mostly as a consumer of 2f-fcore.

Verdict discipline (unchanged, the page-level contract)

PASS ⇒ definitively equivalent; FAIL ⇒ definitively different; anything else times out to inconclusive. Every reduction below (collapse, cache, split, helpers) must preserve this — abstraction may only cause extra work (flat confirmation), never a wrong verdict.

  • Parallel hier driver on the fcore DAG (consumes F1). Replace the sequential leaves-first loop in lec_hierarchical (lhd/lhd_kernel.cpp) with the taskflow proof-DAG: all ready leaves in flight (4–8 workers), a parent unlocks when its children settle, a proven child becomes an o.collapse black-box exactly as today, and the REFUTE-under-collapse ⇒ flat-confirmation rule is preserved. The semdiff structural skip and the verdict cache become the first rungs of the per-def portfolio ladder. Per-def progress lines stay (agents stream-parse long runs).
  • Consume formal blocks (old 2f-verify V7, lec half). (a) Proven invariants (sidecar or mined, impl-side) asserted in the miter are entailed facts — pure solver pruning, can never change the verdict. (b) User assumes follow the fcore contract: over internal signals they are proof obligations (checked on the impl side before use); over primary inputs only, they are environment constraints by nature (inputs are free — nothing to prove) and restrict the shared inputs ⇒ conditional equivalence. Legitimate and often wanted, but the verdict must disclose it ("PROVEN under 3 input assume(s)") so a constrained pass is never mistaken for unconditional equivalence. (c) assume_nocheck_formal is honored unchecked with the per-encounter warning and a distinct disclosure — "PROVEN under N unchecked assume(s)" — so an unchecked-conditioned pass is never confusable with a proven/input-conditioned one; assume_nocheck_synth is ignored (fcore contract, ruling 2026-07-10).
  • LEC hints output (fcore producer). Persist the winning engine and case-split selector into the --workdir cache as strategy hints — keyed by canonical entity name so they survive design edits, and consulted to order the next run's portfolio (user note 2026-07-10) — and optionally emit sidecar formal-block suggestions (assumes that made this LEC fast) with provenance comments — the verify mining concept, extended to LEC.
  • Auto flop-correspondence from semdiff on REFUTE (absorbed from 2d-cex_debug). Correspondence today is lec.match only (explicit user pairs); renamed-but-corresponding flops become unmatched cut points and one-side flops are invisible to the root scan. On a refuting pair, run semdiff::structural_match(matching_names) and feed the stamped flop pairs into the name_alias map exactly as if lec.match had supplied them — differently-named pipelines then need no hand-built match.txt. Best-effort: retiming deliberately breaks flop correspondence; degrade to I/O-level divergence and report unpaired flops as "unpaired", never drop them silently.
  • Collapse quality — the diverged-use memory. Register and memory collapse is landed (flop-cut, memories correspond by address). Rarely, the same memory cannot be collapsed because its use diverges between the sides; the engine must detect the correspondence/usage mismatch and conservatively keep it uncollapsed (descended) rather than force a pairing. Add a regression that plants a diverged-use memory and asserts the verdict is still correct (no false PROVEN via a bogus collapse, no false REFUTE — flat fallback).
