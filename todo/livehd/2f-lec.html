2f-lec LEC remainder — parallel hier driver, formal-block consumption, hints out, correspondence quality

How and why pass/lec works today is the reference doc pass/lec/lec.md. Landed: cvc5-direct combinational miter, inductive flop-cut + BMC-from-reset sequential phases, memory arrays, combinational Sub flattening, the auto ind+bmc fork portfolio with combinational skip, bottom-up hierarchical decomposition with proven-child black-box collapse (lec.hierarchical) + REFUTE-under-collapse flat confirmation, the per-module semdiff structural skip, the parallel input case-split (lec.partitions/lec.split=auto, 2026-07-10), and the witness→lecfail.prp→lhd sim→VCD flow. The hier driver is also now scoped to the picked top pair and its transitive descendants (fixed 2026-07-10 — it previously proved every def the two libraries happened to share). This page tracks what remains, mostly as a consumer of the shared formal core (proof-DAG runner, portfolio, verdict cache, case-split — landed in pass/lec).

Verdict discipline (unchanged, the page-level contract)

PASS ⇒ definitively equivalent; FAIL ⇒ definitively different; anything else times out to inconclusive. Every reduction below (collapse, cache, split, helpers) must preserve this — abstraction may only cause extra work (flat confirmation), never a wrong verdict.

  • Consume formal blocks (old 2f-verify V7, lec half) — LANDED 2026-07-10. (a) Proven invariants (sidecar or mined, impl-side) asserted in the miter are entailed facts — pure solver pruning, can never change the verdict. (b) User assumes follow the fcore contract: over internal signals they are proof obligations (checked on the impl side before use); over primary inputs only, they are environment constraints by nature (inputs are free — nothing to prove) and restrict the shared inputs ⇒ conditional equivalence. Each helper set fingerprints the verdict-cache key, preventing a constrained proof from aliasing an unconditional or differently-constrained proof.

Landed 2026-07-11 — two-tier state correspondence (tier-2 uncertain pairing)

  • Speculative full-match pairing with drop-and-retry (settled 2026-07-10, landed 2026-07-11; supersedes "auto flop-correspondence from semdiff on REFUTE"). Tier-1 stays name-first (canon_flop_name + explicit lec.match); for renamed state, pairing is the only route to unbounded PROVEN (a renamed-but-reset pair still bounded-Proves via BMC — the regression pins exactly that delta). Tier-2 (uncertain) pairs come from semdiff's full-match signature pass (3f-semdiff, Match_result::state_pairs; explicit lec.match pairs seed the signatures as anchors) whenever unmatched state survives tier-1 — zero tier-2 work on the all-names-match path — and are injected as Lec_options::uncertain_match (validated by lec::validate_uncertain_pairs: existence, collision, init-equality — the collision maps walk the FULL hierarchy (forward_hier), because the miter cuts flops at every descended level and an alias colliding with a descendant's canon name would fuse two same-side flops onto one cut key, a false-PROVEN hazard the adversarial review caught). The discipline lives INSIDE prove_equal (uniform across hier/flat/pass callers): ind-PROVEN accepted and disclosed ("PROVEN with N uncertain tier-2 pair(s) applied — self-certifying"); REFUTED under pairs is never final (drop ALL, re-solve once — pair-free re-refute = real FAIL, else Unknown); bounded bmc-Proven never claimed with pairs applied (suppressed to Unknown, no retry — a pair-free bmc is gated Unknown anyway); timeout/Unknown ⇒ no retry. Unpaired state is reported with the reason (ambiguous / kind-init mismatch / no full match). Memory pairs are exported but filtered (is_mem): mem correspondence is shape+occurrence, never name-aliased. Promotion: a PASS persists the pairs as entity-keyed pair hints (formal_cache.json pair_hints, salt-surviving like strategy hints); replay re-validates at injection and keeps the uncertain discipline. Hint storage is all-or-nothing (a pair with a debug-nid name blocks the whole hint — a PARTIAL hint's replay would suppress the fresh signature pass while leaving the unnamed flops unmatched, wedging a cold PASS into a permanent warm Unknown) and self-healing (a replayed hint whose solve does not end Proven is stale — clear_pair_hint drops it so the next run pairs fresh; a planted crossed hint goes UNKNOWN → cleared → PROVEN on the re-run). Uncertain pairs enter the verdict-cache key as a DISTINCT um=[…] segment (same pair set ⇒ same key, so hint-replay runs hit the stored verdict and the Unknown ledger composes; never aliasing a certain lec.match run, whose bounded-Proven policy differs). Knob: lec.state_pairing (default on). Regression: lhd/tests/lec_state_pairing_test.sh (renamed-pipeline PROVEN + warm hint replay, real-diff FAIL through the pair-free confirm, crossed-hint bounded-PASS suppression + self-heal, init-mismatch refusal, all-names-match zero-work, pairing-off baseline).

Landed 2026-07-12 — diverged-use memory collapse guard

  • Keep a diverged memory uncollapsed instead of forcing a shape×occurrence pairing. Memories collapse to one shared current-state array by SHAPE (size×bits) × RTL occurrence order — the key discovery: a memory's get_hier_name() is a POSITIONAL node-id (n10, n11), not an RTL name, so both sides carry n10 at occurrence-0 regardless of identity and a name check cannot tell a crossed pairing from a correct one; semdiff likewise reports symmetric same-shape mems as (ambiguous). The only trustworthy divergence signal is semdiff's genuine-mismatch reason. So pass/semdiff now exports Match_result::{a,b}_mem_diverged (mem cells unpaired with a kind/init mismatch or no full match — NOT the occurrence-safe ambiguous), the driver routes them plus the confident mem state_pairs into Lec_options::{mem_diverged, mem_match}, and build_shared_mems (query.cpp) leaves a shape bucket UNCOLLAPSED when it holds a diverged memory or when semdiff opines and contradicts the occurrence pairing. Non-regression rule: the guard only tightens when semdiff has a POSITIVE opinion about the bucket — otherwise the pre-guard occurrence collapse is preserved (a self-LEC of a 2-same-shape-memory design still PROVEs). Because the guard only ever SKIPS a collapse, it can remove a false PROVEN but never introduce a wrong verdict. Regression pass/lec/tests/lec_mem_diverge_test.sh (crossed same-shape mems → never false PROVEN across ind/bmc/decompose; self-LEC still PROVEN). Note: a Verilog initial block DECOMPOSES a small array into flops (no $mem) — the collapse path only sees write-only $mem_v2 cells.
