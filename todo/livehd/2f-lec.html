2f-lec LEC remainder — parallel hier driver, formal-block consumption, hints out, correspondence quality

How and why pass/lec works today is the reference doc pass/lec/lec.md. Landed: cvc5-direct combinational miter, inductive flop-cut + BMC-from-reset sequential phases, memory arrays, combinational Sub flattening, the auto ind+bmc fork portfolio with combinational skip, bottom-up hierarchical decomposition with proven-child black-box collapse (lec.hierarchical) + REFUTE-under-collapse flat confirmation, the per-module semdiff structural skip, the parallel input case-split (lec.partitions/lec.split=auto, 2026-07-10), and the witness→lecfail.prp→lhd sim→VCD flow. The hier driver is also now scoped to the picked top pair and its transitive descendants (fixed 2026-07-10 — it previously proved every def the two libraries happened to share). This page tracks what remains, mostly as a consumer of 2f-fcore.

Verdict discipline (unchanged, the page-level contract)

PASS ⇒ definitively equivalent; FAIL ⇒ definitively different; anything else times out to inconclusive. Every reduction below (collapse, cache, split, helpers) must preserve this — abstraction may only cause extra work (flat confirmation), never a wrong verdict.

  • Parallel hier driver on the fcore DAG (F1, LANDED 2026-07-10). lec_hierarchical now uses the Taskflow proof-DAG: all ready leaves in flight (4–8 workers), a parent unlocks when its children settle, a proven child becomes an o.collapse black-box exactly as today, and the REFUTE-under-collapse ⇒ flat-confirmation rule is preserved. The semdiff structural skip and the verdict cache become the first rungs of the per-def portfolio ladder. Per-def progress lines stay (agents stream-parse long runs), and formal.jobs bounds the fork-isolated solver children.
  • Consume formal blocks (old 2f-verify V7, lec half) — LANDED 2026-07-10. (a) Proven invariants (sidecar or mined, impl-side) asserted in the miter are entailed facts — pure solver pruning, can never change the verdict. (b) User assumes follow the fcore contract: over internal signals they are proof obligations (checked on the impl side before use); over primary inputs only, they are environment constraints by nature (inputs are free — nothing to prove) and restrict the shared inputs ⇒ conditional equivalence. Legitimate and often wanted, but the verdict must disclose it ("PROVEN under 3 input assume(s)") so a constrained pass is never mistaken for unconditional equivalence. (c) assume_nocheck_formal is honored unchecked with the per-encounter warning and a distinct disclosure — "PROVEN under N unchecked assume(s)" — so an unchecked-conditioned pass is never confusable with a proven/input-conditioned one; assume_nocheck_synth is ignored (fcore contract, ruling 2026-07-10). Each helper set fingerprints the verdict-cache key, preventing a constrained proof from aliasing an unconditional or differently-constrained proof.
  • LEC strategy hints (fcore producer) — LANDED 2026-07-10. The winning engine and case-split selector persist in the --workdir cache as strategy hints — keyed by canonical entity name so they survive design edits. On a verdict-cache miss, auto first tries the prior winning engine and selector; a stale hint falls through to the complete normal portfolio and can never change the verdict. Sidecar suggestion emission remains coupled to the pending verify mining/hints producer rather than this replay path.

Pending

  • Two-tier state correspondence — speculative full-match pairing with drop-and-retry (settled 2026-07-10; supersedes "auto flop-correspondence from semdiff on REFUTE"). Correspondence today is tier-1 only: preserved names (canon_flop_name) plus explicit lec.match pairs; a renamed flop becomes an unmatched cut point, and incomplete correspondence gates ind — the only unbounded-PROVEN engine — to Unknown. So for renamed state, pairing is not a speedup: it is the only route to unbounded PROVEN (a bounded-Proven PASS needs no flop correspondence — BMC's completeness gate covers only output compare points). Add tier-2 (uncertain) pairs from semdiff's full-match signature pass (3f-semdiff) whenever unmatched state survives tier-1 (near-free on the common all-names-match path): fed into name_alias exactly as if lec.match had supplied them, but tracked as uncertain. Pair precondition: reset/init values must be equal; a both-reset-less pair qualifies (power-on correspondence by assumption — the exact semantics tier-1 name pairing already gives reset-less flops); never pair init with init-less. This closes the pre-existing ind corner — a power-on-only init (no reset pin) never enters the ind formula at all (a reset-pin init does, but a mismatch there surfaces only as a distrusted ind-SAT) — and it is what makes a speculative ind-PROVEN self-certifying: any inductive, output-implying, initially-true relation certifies equivalence, so the pairing being "the intended one" is not load-bearing for soundness. Verdict discipline (mirrors REFUTE-under-collapse flat confirmation): REFUTED under a non-empty uncertain-pair set is never final — drop ALL uncertain pairs and re-solve once; a re-refute (now pair-free) is a real FAIL, else the honest ceiling is Unknown, since dropping any pair already makes the correspondence incomplete (hence drop-all + one retry, no CEGAR loop). Timeout ⇒ no retry: the retry's ceiling is the Unknown the timeout already reports, and the Unknown ledger keeps it skipped. Trust notes: under auto a wrong uncertain pair mostly costs a lost proof, not a false FAIL — ind-SAT is already distrusted (hint only), and BMC pairing is dominantly s0 equality constraints (model-removing: a BMC CEX under extra pairs is usually still a legal diverging trace). Not airtight, though: the alias set also feeds the reset-prologue bank-hold heuristic (query.cpp:1998-2024, 2379-2388) and the mem↔flop-bank bridge uniqueness test (query.cpp:2126-2134), which can change transition semantics — the false-FAIL guard is the drop-and-retry rule itself, not monotonicity. The dangerous direction is PROVEN: bounded-Proven must never be claimed with uncertain pairs applied — s0 over-constraint can mask a real bounded CEX (false PASS); strip uncertains on that fallback path. Unpaired flops are still reported "unpaired" with the reason (no full match / ambiguous bucket / init mismatch), never dropped silently; retiming still deliberately breaks flop correspondence — degrade to I/O-level divergence. Promotion: a PASS obtained with uncertain pairs validates them — persist as entity-keyed pair hints (2f-fcore §3 record kind) so warm runs inject them for free; replayed hints keep the uncertain discipline and are re-validated at injection (both flops must still exist and the init-equality precondition must still hold — entity-keyed hints survive edits that can change either), since they alter the obligation set and are not pure-ordering strategy hints.
  • Memory pairing tier-2 — same-shape permutation retry. Memories already pair by attributes, not names (mem_state_key = size×bits + forward_class occurrence), so attribute matching is landed; the residual gap is N>1 same-shape memories whose RTL order differs between the sides — occurrence pairing mispairs them: the proof is lost (ind-SAT, distrusted), and on the BMC path the mispair crosses the shared initial arrays, so a bounded design whose outputs expose those reads can even produce a trusted false REFUTE today. When an ind-SAT touches memory keys with same-shape ambiguity, retry alternative permutations as uncertain pairs under the same drop-and-retry discipline (N is tiny in practice; past a small bound, keep today's behavior). Key-derivation quirk to fix while here: the ind-side bridge collector groups occurrence by size×bits plus port counts (query.cpp:2868-2870) while every other site groups by size×bits only.
  • Collapse quality — the diverged-use memory. Register and memory collapse is landed (flop-cut, memories correspond by address). Rarely, the same memory cannot be collapsed because its use diverges between the sides; the engine must detect the correspondence/usage mismatch and conservatively keep it uncollapsed (descended) rather than force a pairing. Add a regression that plants a diverged-use memory and asserts the verdict is still correct (no false PROVEN via a bogus collapse, no false REFUTE — flat fallback).

Tests

  • Hier fixture on the parallel driver: identical verdicts to the sequential baseline, lower wall-clock, deterministic across runs (F1 acceptance lives in fcore; the lec-specific assertion is collapse/flat-confirm behavior parity).
  • Formal-block consumption: a miter that times out bare but proves with a sidecar invariant asserted; a user-assume run whose verdict line carries the "under N assume(s)" disclosure; an assume_nocheck_synth in the design changes nothing in the lec verdict or encoding.
  • Scope regression: two whole-design libraries sharing many defs + --ref-top a leaf — exactly that leaf's subtree is proven (no unrelated lec[hier] lines).
  • Differently-named pipeline flops (renamed clone): tier-2 full-match pairs them and the pair reaches PROVEN with no supplied match.txt; the pairs persist as pair hints and a warm re-run skips the signature pass.
  • Planted bogus uncertain pair: the REFUTE-under-uncertain-pairs retry drops it and the run ends Unknown naming the unpaired flops — never FAIL from a speculative pair. A genuinely-different renamed design still ends FAIL: the pair-free confirming re-solve refutes on its own.
  • Bounded-proven guard: a reset-less divergence maskable by a wrong s0 pairing never reports bounded-Proven while uncertain pairs are applied.
  • Init precondition: a tier-2 candidate whose reset values differ is not paired (and the report says why it stayed unpaired).
  • Same-shape memory permutation: two same-shape memories swapped in RTL order prove via the permutation retry; the diverged-use memory regression (above) still degrades soundly.
  • Timeout with uncertain pairs applied: no retry fires, the Unknown ledgers, and the warm re-run skips it (same pair set ⇒ same m=[…] key).
  • All-names-match design: the signature pass never runs (tier-1 complete ⇒ zero tier-2 work on the common path).
  • Retimed clone (correspondence deliberately broken): degrades to I/O-level divergence with the unpaired flops listed; verdict stays sound.
