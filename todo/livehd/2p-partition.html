2p-partition build a new graph_library from the colors

Consume the per-node colors produced by 2c-color and emit a new graph_library in which each nonzero color id is a module: generate one new lgraph per nonzero color id, instantiate it once per colored region, and build a new top that wires all the partition modules together. The generated design, emitted with the same top, must be LEC-equivalent to the original.

This is the PARTITION Pre TODO promoted to a task. It is a single pass on a given --top (flow A): it partitions exactly one level. Multi-level partitioning is just repeated CLI calls — color → partition → color … keep_colored=1 → partition — using the keep_colored option in 2c-color to preserve already-partitioned lower levels.

Depends on

  • 2c-color — provides the per-instance colors (hier_storage, flat fallback) + the active serialized color metadata attr on the top graph that this pass reads.

What a partition is

  • Region = a maximal connected component of same-colored nodes. A color id may appear as several disjoint regions (the 2c-color continuous option forbids that when a consumer needs one region per color).
  • Nonzero color id → one module def; region → one instance. All regions of a color id are generated as a single lgraph and instantiated once per region in the new top — this is exactly the "one lgraph per id, instantiated wherever it appears" contract from 2c-color.
  • Cut edge → a port. An edge whose driver-color ≠ sink-color crosses a partition boundary: it is a driver-pin (output) on the driver's color-module and a sink-pin (input) on the sink's. The original design's own primary inputs/outputs stay ports of the new top.

The canonical interface (approach A) + debug_color

  • Derive each color's interface from one representative region — its crossing edges become the module's ports, named from the crossing wire's pin_name, in a canonical order. Every other region of that color is assumed to match (2c-color's structural-identity invariant guarantees it), and a region whose cut signature differs is a hard error.
  • This mismatch will be the common failure — and the bug is almost always upstream in the color pass, not here. So --set pass.partition.debug_color=1 turns the first-mismatch hard error into a full diagnosis: materialize the candidate interface for every region of each color, diff them, and pinpoint which color, which two regions, and which node / edge / port diverges, so the coloring can be fixed.

Name + source-locator preservation (first-class)

The whole point is a readable, LEC-checkable rebuild, so preservation is a requirement, not a nicety. HHDS has no move-node-across-graphs API, so each color-module is built by recreating nodes/pins/edges in a fresh graph — every recreate must carry the provenance:

  • Reg + wire names. Carry livehd::attrs::pin_name on every recreated pin (pin_name_of / set_pin_name / wire_name, graph/node_util.hpp:145,235,294). A wire promoted to a boundary port keeps its original name; preserve as many internal wire names as possible (reg names especially — they anchor the LEC).
  • Source locators. Carry hhds::attrs::srcid on every recreated node (graph/attrs.hpp:90, graph/node_util.hpp:419) and populate each new graph's Source_locator (file content) so the carried srcids still resolve — the emitted Verilog keeps its sourcemap back to the original RTL.

CLI surface

lhd pass partition --top <mod> --set pass.partition.<param>=v  lg:dir --emit-dir lg:dir2
lhd pass partition --top <mod>                                lg:dir              # no output: stats only
lhd pass partition --top <mod> --set pass.partition.debug_color=1  lg:dir --emit-dir lg:dir2
# no --emit-dir  ⇒ stats are implicit (color/region/port counts, cut sizes); the flag only bumps verbosity
# --config <file> supplies a bulk option set

The lhd pass command surface is part of the color/partition deliverable, not the existing lhd synth pipeline. This task adds pass.partition.* option discovery/validation and graph-library input/output handling for lg:dir → --emit-dir lg:dir2.

Verification — Verilog-level LEC

Emit Verilog from the partitioned lg:dir2 with the same top and prove it equivalent to the original Verilog with lgcheck (yosys sat -prove) — the flattened partitioned netlist must be functionally identical to the flattened original. Ships as a bazel equiv test. (Full standalone LEC tooling is the later LEC Pre TODO, which depends on this task + MATCH; 2p uses the existing lgcheck for its acceptance test.)

Subtasks — pending

Core task complete (see landed). Non-blocking follow-ups:

  • Hierarchical-input partition. The pass partitions one flat level: the top's own colored nodes become per-region modules. A top that itself contains Sub instances (multi-level input) is not handled yet — those would need their target defs copied into the output library. The multi-level flow (color → partition → color keep_colored → partition) is unaffected; only a single call's input being already-hierarchical is deferred.
  • cgen multi-fanout Sub-output decl. When a region's output pin feeds several other regions directly (Sub→Sub), inou/cgen/cgen_verilog.cpp emits the connecting wire's reg declaration twice (once from the producer's out_pins() handle, once from each consumer's inp_edges handle — the same driver-bit/class-index mismatch the Memory branch already works around). It is cosmetic: both decls now carry the correct width (partition stamps bits/sign on the Sub pins) and yosys tolerates the duplicate, so LEC passes.

Design notes

  • Module naming. Use the color metadata's per-color name hint; default to a stable <top>__c<id>. Color 0 (NO_COLOR) is invalid partition input: color must produce complete nonzero coloring, and partition must emit a diagnostic if any partitionable node still has color 0.
  • Multi-fanout cut. A driver feeding several sinks in another color is one output port that fans out at the top, not one port per sink.
  • Hierarchy. Single level per call; the multi-level loop is scripted with 2c-color's keep_colored (color upper level, partition, leaving the already-colored lower levels pruned from the walk).
