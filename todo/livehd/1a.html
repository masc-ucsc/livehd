Subtasks — landed

  • C — 1a-ops: unique if/match → Hotmux across the flow. landed 2026-06-11; LEC pairs hotmux_{unique,match,match_partial} + O2 compile mode, suite green. What shipped:
    • LNAST: new structural unique_if node (same child shape as if; .ln keyword uif — the pre-staged TOKEN_KW(uif) is now wired). Being a node TYPE, it rides every generic tree copy (decl-merge, SSA, runner staging / template clone) with no carry sites. is_if_like() covers if-or-unique-if checks (scope walk, timecheck).
    • prp2lnast: the anonymous unique token on if_expression selects unique_if; match always emits it (a match IS a unique-if chain). when/unless flat ifs and the while-desugar stay plain if.
    • tolg: lower_unique_merge — one shared selector (bit i = cond_i, top bit = OR(conds)==0 for the else/fall-through slot) and ONE Hotmux per merged variable; arms that don't write a variable feed its pre-if value. Gotchas pinned: the none-bit must be EQ-against-0, NOT Not (a 1-bit bool Not carries infinite high bits — safe under And, corrupting under the SHL/Or packing), and selector/SHL widths are signed-convention (bits = magnitude+1) or a multi-fanout declared wire truncates the top one-hot position (single-fanout inlined selectors masked both bugs).
    • cgen: process_hotmux — case over the one-hot constants, default: 'hx models the non-one-hot runtime error; Hotmux joins Mux in the always-declared-dest-var and missing-bits exemption sets.
    • cprop: the scalar_pass guard was op > Mux — Hotmux (38, between Mux 36 and IO 39) was silently skipped; now op > Hotmux. Const one-hot selector folds to the selected arm; zero/multi-hot warns and keeps the cell; all-identical arms collapse.
    • bitwidth: process_hotmux — selector gets the unsigned N-bit envelope (never narrowed on one-hot-ness, a runtime property), output unions the data arms like Mux.
