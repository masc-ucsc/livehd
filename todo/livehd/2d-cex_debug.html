2d-cex_debug Actionable LEC counterexamples — remaining gaps over the landed lecfail.prp + VCD flow

When lhd lec REFUTES it has found a concrete input/state under which the two designs differ — a real bug in exactly one of (a) the impl lowering, (b) the ref lowering, or (c) the LEC encoder itself. Scope is strictly the REFUTE case; UNKNOWN (a cone cvc5 cannot discharge) is a solver-capability concern owned by lec / 2f-formal.

Landed baseline (30/06 + 03/07/2026) — what this page used to spec

The core of the original design is in and automatic on every refute:

  • First-diverging internal cut (the ROOT). On a BMC REFUTE the engine captures every flop's paired per-cycle next-state — both sides, by canonical key, no filter (wit_state, pass/lec/query.cpp:~1850) — and the witness names the EARLIEST diverging STATE cut(s) ahead of the primary output that merely inherits them (query.cpp:~2049-2100; ALL cuts at the earliest cycle, sorted, so a wide-fanin bug is not reported as a hash-order pick). Pure getValue on the already-solved SAT model — no new SMT terms, no UNKNOWN risk. Regression: pass/lec/tests/cex_debug_test.sh (the conditional-reg mis-write class: internal s named as root, inherited out carried).
  • lecfail.prp + single-VCD replay. On Refuted, with lec.witness and a user --workdir, emit_lecfail_witness (lhd/lhd_kernel.cpp:4404) re-emits BOTH sides as Pyrope, writes a self-contained testbench (lecfail.prp: one __lecfail_dut_pair wrapper instantiating ref and impl, driving the uncapped structured Witness_trace input sequence incl. the reset prologue), then runs lhd sim --set sim.vcd=true to dump ONE lecfail.vcd with both designs side-by-side. Knobs lec.prpfail / lec.prpfailrun, default on under --workdir; BMC-only (the inductive single-step CEX may be unreachable, so it carries no trace). Test: pass/lec/tests/lec_witness_prpfail_test.sh.

This supersedes the page's original manual design: the daily "real bug or false refute?" loop (re-simulate the CEX on both sides, watch where they diverge) is now one automatic artifact pair. The cex.json contract, the lhd lec --replay mode, the slop/LNAST replay back-end, and the Verilog-testbench/two-VCD-diff forensic path are therefore dropped. What remains below are the gaps the landed flow does not cover, re-specified as extensions of it.

Remaining gaps

1 — Source-map the first diverging cut to file:line (M; the keystone)

  • Gap: the root clause names the diverging cut by canonical flop key with ref/impl values — but no source location. flop_state_key() (pass/lec/encode.cpp:79) has an attrs::srcid → resolve_span fallback but is dead on every engine path (declared encode.hpp:190, never called); the live keys come from canon_flop_name() (encode.cpp:52) and carry no span, and query.cpp reads no srcid at all.
  • Work: during the existing flop-collection loop build a key → node → attrs::srcid → span side map; render s (cpu.prp:554) in the root clause and stamp the same location as a comment next to the corresponding signal in lecfail.prp. Tolerate the multi-stage \x02p<k> key suffix. For a Verilog-origin side, map back through the cgen ECMA-426 sourcemap (inou/cgen/cgen_verilog.cpp note_src()/write_srcmap()). This is real srcid plumbing, not a polish step.

2 — Machine-readable witness artifact (S/M)

  • Gap: the structured Witness_trace (pass/lec/query.hpp:28-42: reset_cycles, diverge_cycle, diverge_outputs, per-cycle {reset_asserted, inputs{name,value,width}}) is exact and uncapped, but only crosses the worker pipe in a private binary form (serialize_result, query.cpp:138) and is consumed solely by emit_lecfail_witness. Tooling that wants the trace must re-parse the generated lecfail.prp or the capped display witness string. The per-cycle paired flop values (wit_state) are read for the root scan and then discarded — never surfaced at all.
  • Work: serialize, next to lecfail.prp (e.g. <workdir>/lecfail.json, mirroring lhd sim's --result-json), the trace + the root-cut record (key, cycle, ref/impl values, and the gap-1 source span) + optionally the per-cycle wit_state values. Same getValue reads the display path already does; an extension of the lecfail flow, not a new replay contract.

3 — Auto flop-correspondence from 2f-semdiff (M)

  • Gap: correspondence today is lec.match only — explicit user pairs, inline or @FILE (lhd/lhd_kernel.cpp:4856, alias map query.cpp:614-632). Hier-LEC calls semdiff::structural_match only as the module-level "structurally identical ⇒ PROVEN, no solver" reduction (lhd_kernel.cpp:4073-4086), never as a flop-pairing source. So renamed-but-corresponding flops keep separate cut keys: the miter treats them as unmatched cut points, and the wit_state capture pairs by identical canonical key (in.find(k), query.cpp:~1853) so one-side-only flops are silently invisible to the root scan.
  • Work: on a refuting pair, run structural_match(matching_names) and feed the stamped flop pairs into the name_alias map exactly as if lec.match had supplied them — differently-named pipelines then need no hand-built match.txt. Best-effort: retiming deliberately breaks flop correspondence; degrade to I/O-level divergence and report unpaired flops as "unpaired", never drop them silently.

4 — Memory-cut readback (deferred; spike first)

  • Gap: has_mem (query.cpp:1106-1119; same rule on the formal path at :~2800) keeps any memory design on cvc5's lazy BV solver — the witness-safe eager bitblast-internal blaster has no array theory — and the lazy solver's spurious-SAT history can make a witness getValue abort inside CaDiCaL. So memory read values never appear in the witness, and the root scan is flop-only.
  • Work: a gated spike first — establish when getValue on the plain-BV rd_fresh read symbols is safe under the lazy solver (or find an eager-with-arrays configuration); only then extend wit_state / lecfail.prp / the gap-2 artifact with per-cycle mem-read records ({mem, addr, data}). This extends the lecfail/witness flow; it is not a separate artifact.

Tests

  • Extend pass/lec/tests/cex_debug_test.sh: the root clause carries a file:line pointing into the impl .prp (gap 1).
  • Extend lec_witness_prpfail_test.sh: lecfail.json exists, parses, and its input sequence matches the one driven by lecfail.prp (gap 2).
  • A pair with differently-named pipeline flops: REFUTE names the semdiff-paired root cut with no supplied match.txt (gap 3).

Traps (carried forward, still true)

  • The source span is NOT free. flop_state_key is dead code, and even it only spans unnamed flops; budget gap 1 as real srcid work.
  • Never add probe outputs. Synthetic outputs perturb the SAT instance into UNKNOWN (observed with LEC_DBG_NIDS); read internals from the solved model via getValue() only.
  • Localization-only. Reading wit_state is pure; pushing state-cut DISTINCT terms into bad can flip proven-in-window designs to REFUTED — flag-gate any change to bad.
  • Memory rides the getValue-unsafe solver (gap 4); the flop-state machinery is array-free and bulletproof.
  • BMC-only. The inductive witness is a single, possibly unreachable step; it never fills Witness_trace, so every artifact here is BMC-gated (as the lecfail flow already is).
  • Sign-magnitude widths. cgen emits 65/66-bit sign-mag nets; any new value compare (JSON diffs, mem reads) must compare at declared width or it reports phantom diffs on the spare sign bit.
