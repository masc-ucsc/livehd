2d-cex_debug Actionable LEC counterexamples: rich trace, source-mapped first-divergence, and source-level replay

When lhd lec REFUTES it has found a concrete input/state under which the two designs differ — a real bug in exactly one of (a) the impl lowering (prp2lnast/tolg/cgen), (b) the ref lowering (slang or the other design), or (c) the LEC encoder itself. Today the verdict prints only the diverging primary output and its ref/impl values; everything needed to localise the bug — the input trace that drove it, the first internal cut that actually diverged (outputs merely inherit it), the source line behind that cut, and a way to replay the trace on both designs — has to be rebuilt by hand every time. This task makes the counterexample self-sufficient. Scope is strictly the REFUTE case; the UNKNOWN case (a cone cvc5 cannot discharge) is a solver-capability concern owned by lec / 2f-formal, not this page.

Plan revision (22/06/2026 brainstorm)

This page was re-planned after a design review. Three pivots, each verified against the code:

  • The driving case is pyrope ↔ pyrope, not verilog ↔ pyrope. The everyday use is verifying a Pyrope transformation (retiming for frequency, restructuring for readability/optimization). Both sides then share the entire toolchain, so the debug surface must be Pyrope source — variables and lines the user actually wrote — not cgen-generated, sign-magnitude Verilog nets.
  • Replay is via slop over LNAST, not a Verilog testbench. Both --reader prp and --reader slang produce LNAST, so a slop generator (an upcoming task, landing regardless of this one) replays both sides at source granularity, in-process, with full visibility into every intermediate variable. slop bypasses tolg and the cvc5 encoder, so it already catches false-refutes. The Verilog-testbench/VCD path is demoted to an occasional forensic tool (see the blame table). The earlier idea of a bespoke constprop-LNAST replay emulator is dropped: slop is the correct, two-sided realization of it.
  • The model already holds inputs + flops + memories + outputs; it is recorded-and-discarded, not missing. The first deliverable is mostly deleting a filter, not new machinery (see "What the SAT model already holds").

Update (30/06/2026) — lhd sim exists; do a do-first slice before the LEC/SIM verification flow

This page (22/06/2026) assumed the only source-level sim was the upcoming slop generator, so Stage 2's --replay waited on it. That gap is now partly filled: lhd sim (inou.cgen.sim) compiles an LGraph to executable C++ and already supports per-cycle stepping, a process-wide seeded RNG, checkpoint save/load/--restart-at, located asserts, and --result-json. That is enough to realize the replay-and-classify half of this task without waiting for slop. In blame-table terms lhd sim is the iverilog(Verilog) column (LGraph→C++, bypasses only the encoder), so by itself it already separates an encoder false-refute (sim agrees, LEC refutes) from a genuine bug (sim disagrees). The slop/LNAST column (which additionally splits tolg/cgen bugs from the encoder) stays a later refinement.

Why now: the small_todo Backend LEC/SIM verification flow (regenerated Verilog↔Pyrope per module, LEC + SIM table) will produce many LEC refutes/inconclusives, each needing the "real bug or false-refute?" classification this task provides. Concrete case already in hand: mem_pending_sync_init LEC-refutes under cvc5 because the encoder models a type==1 sync read at latency 0 — a sim replay of the CEX on both sides would show them agreeing, immediately flagging an encoder false-refute rather than a generation bug. (The 3 landed init fixes already turned the type==2 / ROM init cases green; the sync-read latency is the residual encoder gap.)

Do-first slice (land before / alongside small_todo):

  • Stage 0 in full — first-diverging internal cut + per-cycle state capture from the already-solved SAT model (pure read-only, ~40–60 lines, no slop, no UNKNOWN risk). Answers "which register/wire diverged first, and when."
  • A minimal cex.json = per-cycle primary-input trace (wit_ins) + rstN/sN labels + the first-divergence record. Defer the full source-map/name_map keystone (cut name + values are enough to start classifying).
  • lhd lec --replay cex.json backed by lhd sim: drive lhd sim on BOTH sides with the cex inputs (seed initial state where present), run ≤bound cycles, compare per-cycle outputs and checkpoint reg/mem state — the same checkpoint-compare the small_todo SIM column needs, so build the two together. Emit the blame-table verdict (sim-agree ⇒ encoder false-refute; sim-disagree ⇒ genuine bug).

Defer: the full source-mapping + name_map (Stage 1 keystone), the slop/LNAST replay (Stage 2 as originally written — now a refinement that adds the tolg/cgen-vs-encoder split), and the Verilog-VCD forensic path. The cex.json contract below is unchanged; the do-first slice fills a subset of it and uses lhd sim as the replay engine instead of slop.

Depends on

The lec engine (the witness already exists in skeleton form) and the upcoming slop generator (LNAST → slop; the cex.json below is co-designed as its seed + input + I/O-oracle format). Reuses, rather than rebuilds: the cgen ECMA-426 sourcemap (inou/cgen/cgen_verilog.cpp note_src()/write_srcmap(), cgen_verilog.hpp:104), the flop correspondence keys (pass/lec/encode.cpp:52 canon_flop_name() → the live \x01n:<hier> keys), the lec.match parser, the new per-cut decompose (query.cpp:822-838, --set lec.decompose), the VCD reader (core/vcd_reader.{hpp,cpp}, for the optional forensic path), and 2f-semdiff's structural_match() (pass/semdiff/semdiff.cpp:299) for auto flop-correspondence between renamed pipelines.

Correction (was wrong in the original draft): the source span is not free. flop_state_key() (encode.cpp:79) is defined but never called by any engine path — the live next-state keys come from canon_flop_name() (\x01n:<name>, no byte range), and even flop_state_key only emits a span for unnamed flops. Source-mapping a cut is real work (key → re-find node → read attrs::srcid → resolve span), and it is the keystone of this task, not a polish step.

The debug loop today (what this replaces)

Per REFUTE the manual loop is: bisect by --set lec.bound to find the breaking cycle; add the env-gated dumps (LEC_DUMP_FLOPS, LEC_DUMP_WIT) to pass/lec/query.cpp and walk backward from the diverging output to the first diverging flop; hand-build a testbench with the solver's exact register init + instruction trace to ground-truth which side is right; and map a canonical cut name (a_exmem_exresult) back to prp2/cpu.prp by eye. Every step exists only as scratch. The three things that would have saved the most time: a rich counterexample, a one-command source-level replay, and source-mapped names.

Sample use cases (the design targets)

Case 1 — pyrope ↔ pyrope transform (the common case). You retime or restructure a module for frequency; LEC refutes. You need to see, in Pyrope, the first cycle/variable where the optimized version diverges from the original — without reading generated Verilog. Note: under retiming, registers are deliberately moved, so flop-by-flop correspondence may not exist; localization must fall back to I/O + combinational cones with full source-level value visibility.

Case 2 — a conditional reg write whose un-covered path is left a don't-care by the lowering instead of holding — the shape hit in simplechisel/build_dualissue_d/prp2/cpu.prp:554 (the ID/EX flush block), which surfaced as an io_dmem_address divergence. Minimal illustration:

  • impl reg s:u8 = 0 ; if inc { s = s + val } ; out = s  — on !inc a mis-lowering leaves s don't-care instead of holding.
  • ref holds s on !inc (the correct semantics).

Today the whole verdict is one inherited-output line:

lec: REFUTED first divergence at checked step 3/6: io_dmem_address(ref=18446744073709551614 impl=9223372036854775805)

— which output, and the two values. Not: which cycle's inputs caused it, which internal flop diverged first, or where in the source.

This task should instead emit (all values come from the SAME SAT model already solved — no re-solve):

lec: REFUTED at checked step 3 (cycle 23)
  first diverging cut:  a_exmem_exresult     ref=-2   impl=9223372036854775805 (don't-care)
      source:  prp2/cpu.prp:554   `if idex_flush { b_idex_rd1 = 0 } else { ... }`
      ref flop:  pipeB_ex_mem.reg_ex_result        (matched via lec.match / semdiff)
      propagates to output io_dmem_address @ step 3
  primary-input trace (cycles 20..23, replayable):
      cyc20  io_imem_instruction=0x00..  io_imem_good=1  io_dmem_good=0 ...
      cyc21  io_imem_instruction=0x13..  ...
      cyc22  io_imem_instruction=0x..    ...   (branch enters EX)
      cyc23  ...
  replay:  lhd lec --replay cex.json     # re-sims BOTH sides with slop, reports the
                                         # first Pyrope variable/cycle that diverges

The win is the root cut + its source line + the replayable input trace, turning a 2-hour manual hunt into reading one verdict.

What the SAT model already holds (verified against query.cpp)

On REFUTE, all four of inputs / registers / memories / outputs are in the solved model; solver.getValue() works on any retained term once produce-models is set (query.cpp:199, when opts.witness). Status:

  • Inputs — present, reportable. wit_ins{cyc,name,term} (query.cpp:701-704) holds the full per-cycle primary-input trace incl. the reset prologue. Already getValue'd (:931); the printed trace just truncates to ≤ first_bad. Gap = serialization.
  • Outputs — present, reportable. wit_outs{cyc,name,rv,iv} (:746-748); this is the entire current witness.
  • Registers (per cycle, both sides) — present, read-only, recorded-and-discarded. The throwaway st<uid> current-state symbols (:655) are useless, but the next-state maps rn/in (:754-764) are keyed by canonical hier name, identically on both sides, and current@cyc == next@cyc-1 by the equality pinning. So getValue(rn[key])/getValue(in[key]) per cycle yields full per-cycle flop state of both designs, paired — with no new SMT terms (so the probe-output→UNKNOWN trap does not apply). Already proven by the LEC_DUMP_WIT/dbg_ns path (:779-801, :845-876), which is gated behind an env var and a hardcoded keep() filter (:780-784). Deleting that filter is most of the first deliverable.
  • Memories — present but on the getValue-unsafe path. Arrays are threaded per cycle and never queried. The trap: has_mem (query.cpp:185-195) disables the witness-safe eager bitblast-internal blaster and falls back to cvc5's lazy BV solver, whose own comment warns it can return spurious SAT and make a subsequent getValue abort inside CaDiCaL. Read values are cheaper than expected (read douts rd_fresh[k] are plain BV symbols tied by equality to SELECT(arr,addr), so getValue(rd_fresh[k]) is a plain BV read, not an array query) — but they still ride the lazy solver. Treat memory readback as a gated spike, not a casual extension.

Blame decomposition (why slop, and when the Verilog path still helps)

cvc5 compares LGraphs (post-tolg, via the encoder). slop replays from LNAST (bypassing tolg + encoder); an independent Verilog simulator replays cgen(LGraph) (bypassing only the encoder). Given a REFUTE, the two independent sims triangulate the (a)/(b)/(c) blame:

slop(LNAST)   iverilog(Verilog)   ⇒ blame
 disagree      disagree            genuine bug — the transform changed behavior  (the EXPECTED case)
 agree         agree               encoder bug — false refute
 agree         disagree            tolg/cgen bug exposed by the transform
 disagree      agree               rare/contradictory — investigate

Key point: slop by itself already separates "encoder/tolg bug" from "genuine/frontend bug" and catches false-refutes (it never touches the encoder). The Verilog path only adds value in the slop-agree rows (separating tolg/cgen from encoder) — a rare forensic question, not the daily loop. Hence slop is the primary back-end and the Verilog-testbench/VCD path is optional.

Design decisions (locked)

  • REFUTE only. A timed-out cone (UNKNOWN) is a capability gap, not a counterexample; do not conflate. UNKNOWN localisation is already served by --set lec.decompose's "N/M cuts PROVEN, K unresolved {…}" line.
  • Root, not leaf. Report the EARLIEST diverging cut (an internal flop/wire/output), since downstream outputs only inherit it. Extend the first_bad scan (query.cpp:897) over internal cuts, not just wit_outs primary outputs.
  • Flop correspondence is best-effort. For code-restructuring it holds (names/boundaries correspond → first-diverging-flop works). For retiming it is deliberately broken (no corresponding flop); degrade gracefully to I/O + combinational-cone localization, leaning on slop's source-level visibility. Pair renamed-but-corresponding flops via semdiff / lec.match.
  • Extract from the existing model, never add probe outputs. Pull internal cut values with solver.getValue() on the SAT assignment. Adding synthetic outputs perturbs the instance enough to flip cvc5 to UNKNOWN (observed with LEC_DBG_NIDS). Reading rn/in for localization is pure; pushing state-cut DISTINCT terms into bad (making a masked internal divergence refutable) can flip verdicts, so flag-gate it.
  • Replay at the source level via slop, not via Verilog. The default --replay back-end is slop (two-sided, in-process, Pyrope-granular). The Verilog-testbench/VCD path is optional, for tolg-vs-encoder forensics only.
  • Machine-readable + human. Emit a cex.json (step, root cut, source span, name-map, correspondence, full per-cycle trace) alongside the human verdict; it is the shared contract consumed by --replay, the slop generator, scripts, and (optionally) the Verilog path — without re-solving.

The cex.json contract

One artifact, co-designed with the slop generator: it is simultaneously the seed (initial state + per-cycle inputs), the oracle (the per-cycle flop/output values cvc5 solved, to confirm a replay reproduces the divergence), and the localization (first cut + source). Sketch:

{
  "engine": "bmc", "bound": 6, "reset_hold": 20,
  "first_divergence": {
    "kind": "flop" | "output", "cut": "<canonical key>",
    "cycle": 23, "checked_step": 3, "ref": "-2", "impl": "9223372036854775805",
    "src": { "file": "prp2/cpu.prp", "line": 554, "snippet": "if idex_flush { ... }" }
  },
  "name_map":      [ { "key": "n:...", "lnast": "...", "src": "file:line" } ],
  "correspondence":[ { "ref": "...", "impl": "...", "via": "semdiff|lec.match|name" } ],
  "io_oracle": false,   // true under retiming: validate replay at I/O only (no flop corr.)
  "cycles": [
    { "cycle": 0, "label": "rst0",
      "inputs":     { "io_imem_instruction": "...", ... },
      "state_ref":  { "<key>": "..." }, "state_impl": { "<key>": "..." },
      "mem_reads":  [ { "mem": "<key>", "addr": "...", "data": "..." } ],
      "outputs":    { "io_dmem_address": { "ref": "...", "impl": "..." } } },
    ...
  ]
}

The name_map (LGraph flop key ↔ LNAST/source name) is the contract between this task and the slop generator; it is exactly the Stage-1 source-mapping work. io_oracle + correspondence let the replay run when no flop correspondence exists (retiming).

Traps

  • Source span is NOT free. flop_state_key (encode.cpp:79) is dead on the live path; the live keys (canon_flop_name → \x01n:<name>) carry no byte range, and even flop_state_key only spans unnamed flops. Budget Stage 1 as real srcid work.
  • Memory rides the getValue-unsafe solver. has_mem disables the eager blaster (query.cpp:185-195); the lazy fallback can make getValue abort. The flop-state win is array-free and bulletproof; memory is a gated spike.
  • Don't add outputs to probe internals. LEC_DBG_NIDS showed synthetic outputs perturb the SAT instance into UNKNOWN; read internals from the existing model via getValue().
  • Adding state-cuts to the refutation condition changes verdicts. Reading wit_state for localization is pure and safe; pushing state-cut DISTINCT into bad can flip "proven-in-window" designs to REFUTED. Localization-only by default; flag-gate any change to bad.
  • Retiming breaks flop correspondence. "First diverging flop" can be meaningless when registers are moved; fall back to I/O + cones and slop source visibility. Don't silently drop unpaired flops — report "unpaired."
  • BMC-only. Every trace/replay concept assumes the multi-cycle BMC engine (now default). The inductive witness is a single, possibly-unreachable step.
  • Sign-magnitude widths. cgen emits 65/66-bit sign-mag nets; every diff (state, output, replay, VCD) must compare at the real (declared) width via fit_to, or it reports phantom diffs on the spare sign bit.
  • lhd tool cat hides inline/const sink edges — a 2-arm mux with a constant arm prints as 1-arm and reads as "dropped arm" (cost a long false lead). Any netlist dump used for root-cause must show inline constants.
  • Reset prologue cost is in the SMT re-encoding, not the sim. A 1-cycle repro still re-pays the ~20-cycle reset/flush prologue. slop replay sidesteps this for the sim; a --set lec.from-state=@file would let the miter skip the prologue too. (Note: slop checkpoint cannot help here — the cost is the SMT encoding, which checkpoint does not touch.)
