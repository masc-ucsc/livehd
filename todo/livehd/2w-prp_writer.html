2w-prp_writer prp_writer codegen completeness

pass.prp_writer (--emit-dir pyrope:DIR/) re-emits LNAST as Pyrope. It was built for the v→prp direction (slang reads a .v, the writer emits Pyrope, lhd lec re-checks — the prp-v2prp-* family). The prp→prp direction (re-emit a Pyrope source through the upass) is still maturing: this task finishes it.

Landed

  • Safety net (default-on). The writer no longer silently emits a /* TODO */ stub and lets the compile pass. An unimplemented construct is recorded and pass.prp_writer raises a fatal unsupported diagnostic, so lhd compile … --emit-dir pyrope: exits non-zero. Pass --set prp_writer.debug=true to keep the partial (TODO-laden) output and let the compile pass — for inspecting what is missing. prp_writer is registered in the --set registry (kSetPasses) with a debug label.
  • Pipeline. stage[N] x = v (a reg declare carrying a trailing stages node) re-emits as stage[N] x = v instead of the broken reg x = N; the output-port cycle annotation @[N] / @[A..=B] is recovered from the io-store stages node (the slang stages(nil,nil) still opts out with @[]). Driving case mod_call_pipe now round-trips and LEC-passes.
  • type_spec. A bare type assertion is folded into the variable's first declaration (mut x:T = v).
  • tuple_concat. dst = (...a, ...b) spread concatenation.
  • Regression tests. The prp-p2p-* family (inou/prp/tests/p2p_test.py) round-trips a Pyrope source through the writer and LECs it against the golden .v; the pipeline cases are wired in (mod_call_pipe, pipe1_pass, pipe2_two_out, pipe3_mul, pipe_range, pipe_state_acc, mod_delay3, reg_out_feedforward). Add a name to that allow-list once it round-trips.

Landed — 2026-06-18 (subtasks A + C1)

  • A — unimplemented statement nodes (all emit cleanly now; safety net no longer fires for any equiv case).
    • timecheck (the inert x@[N] assertion) is dropped (the @[] opt-out): the timing it carries is already re-emitted by the writer as the stage[N] declaration and the out:T@[N] interface annotation, so the statement is redundant; and its SSA-renamed ref frequently names a stage var not yet assigned at that point (tmp@[3] precedes stage[3] tmp = …), which would forward-reference an undeclared name on re-parse. (A literal statement @[N] is left for the future LGraph timecheck.)
    • for re-emits as for v[, idx[, key]] in [ref ]iter { body } (write_for); it survives to the writer only inside a generic / un-monomorphized template lambda (every concrete instantiation is unrolled by the runner).
    • func_def now emits the nested lambda (kind name<generics>(in:T,…) -> (out:T,…) { body }, via the extracted emit_port_group) instead of a stub. No equiv LEC case exercises it; covered by the writer unit test.
    • Writer bug: the pre-declare scan wrongly collected named call arguments (mul(a=in1, b=in2), wrap(v=…, type=…)) as statement stores, hoisting bogus mut a/b/v/type = 0 lines. Fixed by requiring a store's parent to be a stmts node to be a hoist candidate.
  • C1 — v→prp derived clock (mclk_derived) LEC-passes. A clock_pin/reset_pin bound to a net now emits clock_pin=ref <net> (a bare name resolved to the net's declare-time value 0 → the names clock_pin '0' tolg error), and the net's combinational driver (mut gclk = clk_b & gate) is emitted before the reg declares that bind it (the declare-first hoist otherwise put the reg ahead of its clock driver). prp-v2prp-mclk_derived green; full suite 615/616, 0 regressions.

Why the LEC "passed" before

It did not — there was no LEC on the prp→prp output at all. lhd compile … --emit-dir pyrope: reported status:pass (exit 0) regardless of what the writer produced, because the writer only wrote a /* TODO */ comment and never raised a diagnostic. The safety net (above) closes that hole; the prp-p2p-* family adds the missing LEC gate.
