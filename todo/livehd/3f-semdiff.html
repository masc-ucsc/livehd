3f-semdiff alg=functional — n:m gap matching between two LGraphs, feeding the LEC skip

Status. The v1 structural engine is LANDED (pass/semdiff: semdiff::structural_match, anchored frontier propagation meet-in-the-middle, the greppable match attribute, matching_names flop/mem anchoring, lhd semdiff mirroring the lec CLI). Hier-LEC already consumes it as the "structurally identical ⇒ PROVEN, no solver" per-def skip. The canonical per-def digest (old lec-M3 remainder) moved to 2f-fcore's unified cache, which also serves 2i-import's module identity. This page is the remaining v2.

Pending — alg=functional (the LEC win)

  • v1 leaves the region between the forward and backward frontiers as match=0 on both sides (a + node vs a ripple-carry-adder cone stays unmatched). v2 gives each gap its own region ID and matches gaps n:m: per-gap boundary equivalence via a bounded miter on the existing pass/lec cvc5 path; a proven gap-pair gets one shared region ID.
  • Then lec assumes the matched regions equal and hands only the still-unmatched gaps to the solver — the miter-shrink this task exists to unlock. The kind-agnostic structural_match(a, b, opts) entry point is already the shape lec calls.
  • --set semdiff.id_granularity=region labels the n:m gap-regions (same knob as v1's pair/region choice).

Tests

  • + vs RCA flips from match=0 (v1, documented boundary) to matched under alg=functional; surrounding logic still matched by v1 rules.
  • A gap-pair whose boundary functions genuinely differ keeps match=0 (no false region merge — the miter must prove, not assume).
  • lec on a design whose only diff is inside one gap: solver work restricted to that gap (assert via the per-def progress/timing lines).

Traps (carried from v1, still true)

  • Sum mixes added and subtracted operands — commutative-normalize within a port-class only, never across.
  • Width matters (unsigned bits = magnitude+1): bits stays in the node key so differing widths don't falsely match.
  • Two libraries with same-named modules: separate Hhds_graph_library instances keep gids/attr stores independent — hold the two Graph* directly.
  • match must be registered before any load/save or deserialization asserts.
