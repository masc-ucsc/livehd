Landed 2026-07-11 — full-match state pairing (lec tier-2 producer)

The simplified/fast SynAlign scheme (papers/iccad26/SynAlign), implemented inside structural_match (state_pairing/dump_state options, Match_result.state stats): full match only — no half/partial/surrounding relaxation cascade. Per unmatched flop, SRP/ERP resolved-point signature sets over the sequential graph (anchors = def IOs + tier-1 pairs; clocks/resets/comptime excluded; data pins only), commutative set-hash inverted maps, ambiguous buckets stay unmatched, fixed point ≤64 rounds. Two deliberate deviations from the paper: tokens are (anchor, min-hop-distance) annotated — plain sets leave linear chains permanently ambiguous — and ambiguous buckets are never force-picked. The reset/init-value equality precondition folds init/size into the signature kind. Consumer and verdict discipline live in 2f-lec (uncertain pairs, drop-and-retry, bounded-proven guard).

  • Measured (XSCore, 1630 defs, 18,231 state cells/side): names on = 100% tier-1 1:1 (round-tripped names); names destroyed (100%-noise mode) = tier-2 recovers 27.6% full-match pairs, residue dominated by ambiguity (symmetric queue/regfile rows). At 10% name noise: ~55% recovery with 0 wrong pairs — the paper's 100%-precision claim reproduces. Tier-2 is near-free next to the lec solve (~1m35 either way on XSCore).
  • lhd pass semdiff grew --set pass.semdiff.state_pairing / dump_state / hier / save / name_noise / noise_seed / explain_noise / synalign_maxiter; hier pairs defs by entity (same canon as hier-lec) and aggregates State_stats. 5 unit tests in semdiff_test.cpp.
  • Iteration headroom, in order: sub-instance anchors (def identity + port binding as resolved points), a bounded-set-diff "near-match" tier (recovers the observed genuine cross-side asymmetries; lec re-verifies anyway), symmetric-bucket tie-breaks, parent-context anchors for tiny defs.
  • Scope boundary (ruling 2026-07-11): matching is one problem per def pair — an impl-side inlined/flattened module leaves its flops no-counterpart in the parent pair (the hier=1 stats show the ref/impl state asymmetry and REF-ONLY defs, so the case is diagnosed, not silent). Flattening-aware matching (virtual-flatten descent + the name-preservation contract through synthesis) lives in 3a-synth, to be tried small once synthesis runs.

Pending — alg=functional (the LEC win)

  • v1 leaves the region between the forward and backward frontiers as match=0 on both sides (a + node vs a ripple-carry-adder cone stays unmatched). v2 gives each gap its own region ID and matches gaps n:m: per-gap boundary equivalence via a bounded miter on the existing pass/lec cvc5 path; a proven gap-pair gets one shared region ID.
  • Then lec assumes the matched regions equal and hands only the still-unmatched gaps to the solver — the miter-shrink this task exists to unlock. The kind-agnostic structural_match(a, b, opts) entry point is already the shape lec calls.
  • --set semdiff.id_granularity=region labels the n:m gap-regions (same knob as v1's pair/region choice).
