2f-verify lhd formal verify — incremental assert/assume BMC engine + assume mining (on the lec unroller)

Status. The engine core is LANDED (2026-07-08, V1–V3 — see git history of this page for the landing notes and user rulings): lhd formal verify proves a design's fproperty obligations by per-obligation incremental BMC on the pass/lec engine (frontier assumes, per-assert/per-cycle verdict table, witness traces, per-obligation timeout isolation); formal name.dotted { } sidecar blocks bind through import aliases to the top or to every instance of a submodule and compile through the real Pyrope pipeline into per-cycle-bound monitors (--formal glob filters); the verdict ladder upgrades bounded-proven asserts to PROVEN-unbounded via simultaneous 1-induction (reset pinned deasserted in the step, disclosed). Knobs live under ONE namespace, formal.*, with the legacy lec.* spellings as aliases (user ruling: no per-subcommand replicas); lhd formal lec = the lec command. User docs: docs/livehd/02-usage.md (flow) and docs/pyrope/05-assert.md (block syntax). Tests: //lhd/tests:lhd_formal_verify_test.

This page is now the PENDING remainder — the mining loop that turns solver-learned facts into sidecar assumes, the cross-run reuse layers, the budget scheduler, LEC consumption of formal blocks, and the memory-template attack. Depends on / affects

  • lec — the landed Encoder + BMC unroller (pass/lec/encode.cpp, query.cpp) are the substrate; this page owns the engine restructure. The lec-roadmap M5 (assume-guarantee) and M6 (persistent cache) items are subsumed by this page's frontier-assume and verdict-cache layers.
  • 2f-formal — depends on this page (do 2f-verify first): compile-time pass.formal becomes the don't-try-hard profile of this engine (same code, restricted options), replacing its self-contained single-frame Prover for property proving; its pending items (stateful asserts, reset-exclusion hard-error escalation) re-base on the verdict ladder here.
  • 3f-temporal — temporal assertion proofs will target this engine (bounded windows map onto the unroller directly).

Scheduler — global budget, escalating rounds, diagnose-last

  • Global formal.timeout owns proving time (fixes today's hazard: per-checkSat tlimit-per composes to (N+1)×timeout unbounded). Rounds: R0 free (comptime, cache hits, sidecar invariant rechecks) → R1 small slices (~1-2s per pending obligation at the frontier; the multi-property literature's cheap-first: most properties are easy) → R2+ escalation (4s → 16s → 64s…), survivors ranked by getDifficulty mass from their own failed attempts + cone size + last-round progress; stragglers sink to the end and inherit remaining budget. Frontier assumes interleave, so later rounds run in a smaller space.
  • formal.minetimeout — an independent budget (e.g. 60 when formal.timeout=120; 0 = no mining) for the diagnosis/mining phase: getTimeoutCoreAssuming names the toxic kernel ("these 2 asserts + this store chain") and the miner targets their cones — so a timed-out run's output is the assumes that make the next run succeed.
  • Determinism. Compile profile budgets by cvc5 rlimit (deterministic, machine-independent — the landed pass.formal convention); formal verify defaults to wall-clock with formal.budget_mode=rlimit available for CI.

Mining — learned facts → Houdini → ranked → Pyrope

  • Source 1: learned-literal recurrence (cheap — the solver already computed it). Harvest getLearnedLiterals(INPUT); strip cycle prefixes; a literal recurring at many cycles ("r5_st ≤ 3 at cycles 2..9") is an invariant candidate "st ≤ 3". Keep only literals over named user signals (internal st<uid> temps can't become source code). Word-level facts map back through encoder naming to LGraph signals to Pyrope names — this is why CNF-level solvers can't do this job (see cvc5 inventory).
  • Source 2: templates targeted at the pain (v2). Where difficulty / timeout cores point, instantiate hardware-invariant templates over the registers and memories in the hard asserts' cones: range bounds (seeded from BMC model values), one-hot state encodings, mutual exclusion of valid bits, register-pair equality, count == popcount(valids), valid ⇒ field-in-range per memory entry.
  • Filter: Houdini fixpoint on the persistent solver — assume all candidates at cycle c, prove each at c+1, drop failures, iterate to fixpoint. Survivors are genuine inductive invariants (true at every cycle of any bound; re-provable next run in one query).
  • Rank before emitting — a candidate must earn its keep: it appears in the getUnsatCore of a newly-provable obligation, or asserting it drops a straggler's solve time / difficulty mass. Everything else is noise (and every carried assume also bloats the formula).
  • Emit ready-to-paste formal blocks with provenance comments; inductive survivors → the auto-managed sidecar; bounded/unranked candidates → the speculative report only.

Memory strategy — invariants first, lazy concretization later (decided)

  • Why memories explode: each cycle appends stores; read-over-write chains grow with the bound; the solver case-splits on address aliasing across the whole history.
  • Attack A (ship first): per-entry invariants via comptime-unrolled for loops in formal blocks + the template pool above. Most memory-heavy asserts need a per-entry property, not the contents.
  • Attack B (later, evidence-gated): CEGAR lazy reads — start each memory read as a free symbol (sound for Proven: strictly more behaviors); on a refutation, validate the CEX against the real store history; concretize only the conflicting reads/alias facts and re-check (with the persistent solver, refinement is just new assertions — no rebuild). Pono ceg-array / Burch–Dill lineage; the existing State_box UF collapse is the same philosophy at module granularity. Build only where timeout cores show store chains remain the kernel after Attack A. The synergy: an assumed invariant constrains the free reads directly, so the abstraction is often precise enough to never build the chain.

Cross-run reuse — three layers, cheapest-to-verify first

  • L1 — the sidecar (above): survives code edits; one induction query per invariant per run.
  • L2 — per-obligation verdict cache (subsumes lec-M6). Key = hash of the serialized SMT encoding of the obligation's cone + obligation term + solver identity (logic, bv-solver, all options, cvc5 version) + engine params (phase, reset_hold, bound). Hit ⇒ skip the solve. Rules (soundness F): hash downstream of the encoder (an encoder semantics fix auto-invalidates); cache only definitive verdicts (UNSAT; SAT together with its witness model, revalidated by evaluation on hit); never cache Unknown. Payoff: edit one module ⇒ only cone-touched obligations re-run (verification_cache.json precedent, made native + sound + fine-grained).
  • L3 — learned-literal replay (formal.verify.replay=on, opt-in). On timeout, persist getLearnedLiterals(INPUT) (+ Plugin clause stream) as SMT-LIB text keyed by frame hash. Exact frame match ⇒ entailed, re-assert directly (restart partway up the mountain). Frame changed ⇒ per-literal short-timeout checkSatAssuming(¬L) revalidation, failures dropped. Guardrails (soundness D): harvest only level-0-provenance literals (collected with no obligation pushed) over named symbols; deterministic symbol naming is a completeness requirement (a rename ⇒ cache miss, never a wrong bind).

LEC integration (lhd formal lec consuming formal blocks)

  • Proven invariants as helpers — entailed facts of the impl side; asserting them in the miter prunes solver work and can never change the verdict. Pure speedup; no ref-side name mapping needed.
  • Unproven/user assumes as environment constraints — restrict the shared inputs ⇒ conditional equivalence. Legitimate and often wanted, but the verdict must disclose it ("PROVEN under 3 assumes") so a constrained pass is never mistaken for unconditional equivalence.

Soundness rules (adversarially reviewed; the non-negotiables)

  • A. Assume-at-proven-horizon. A fact proven for cycles ≤ k may be assumed at cycles ≤ k while checking anything at bound ≤ k — but "proven" means per-cycle UNSAT verdicts (a tlimit'd sweep can be "proven at 2..5, Unknown at 6"; assuming the Unknown cycle is a real masking channel), and the assumption must be the exact term proven (X-mask and width-fit included).
  • B. Raising the bound k→k′ > k: clip assumptions to cycles ≤ k (free, always sound) or re-prove at k′ first. Assuming beyond the proven horizon can retroactively prune traces whose violation is at cycle 2 (suffix constraints remove whole traces) — strictly worse than not assuming at all. A base-SAT vacuity check does not detect partial pruning.
  • C. Circularity: same-cycle mutual assumption is unsound (two-line counterexample: simultaneous flags). Sound discipline = assume siblings only at cycles strictly earlier than the cycle being checked (McMillan's circular rule / Rushby's PVS proof; Goldberg et al.'s JA-verification). With unproven mutual assumptions the guarantee is for the conjunction/first-failure only — an individual local pass is a global proof only if the whole set passes.
  • D. Learned-literal replay: a literal is entailed by the assertion stack at the moment of learning; cvc5 does not tag scope provenance, so literals learned while an obligation was pushed may depend on it. Only frame-only provenance or per-literal revalidation. Monotonicity means timed-out-run provenance per se is fine.
  • E. k-induction strengthening: the step case starts in arbitrary unreachable states — BMC-to-k facts may never be step-frame assumptions. Sound forms only: BMC discharges base cases; helpers assumed in the step must be true invariants (sidecar) or folded into the conjunctive obligation (assume helper∧P at frames 1..k, prove helper∧P at k+1). Base depth ≥ induction depth used.
  • F. Verdict cache key must determine the exact SMT script + query + solver configuration + solver binary (this repo already lived a solver-config soundness bug — the lazy-bitblaster spurious SAT behind bv-solver=bitblast-internal, query.cpp:1098-1120 — so solver identity genuinely belongs in the key). Definitive verdicts only.

cvc5 1.3.4 API inventory (verified against the vendored headers)

  • checkSatAssuming (cvc5.h:5694) + push/pop — the engine substrate; incremental is default-on in 1.3.x.
  • getLearnedLiterals(modes::LearnedLitType) (cvc5.h:6156; gate produce-learned-literals) — unit facts entailed by the current assertions; INPUT mode = input-vocabulary, the replayable/minable class. cvc5's own deep-restart replays these internally (but deep-restart is incompatible with incremental — external replay is the workaround).
  • getDifficulty() (cvc5.h:6029; gate produce-difficulty) — per-input-assertion lemma/conflict participation count; callable right after any checkSat, including timeout. The scheduler's ranking signal.
  • getTimeoutCore() / getTimeoutCoreAssuming() (cvc5.h:6063/6101; knob timeout-core-timeout, default 10s) — a subset of assertions/assumptions that alone reproduces the timeout; the Assuming form needs no prior checkSat. The diagnosis phase's tool.
  • getUnsatCore() (gate produce-unsat-cores) — which assumed facts a proof actually used ⇒ prune useless assumes, build the empirical helper-dependency DAG, rank mined candidates.
  • Plugin (cvc5.h:3620: notifySatClause / notifyTheoryLemma / check()-injects-lemmas) — live tap to stream learned clauses out before a timeout kill, and inject on restart.
  • getInterpolant/getAbduct — present but SyGuS-enumerative underneath; not credible on BMC-sized bit-vector formulas; do not plan invariant mining on them.
  • No solver-state serialization exists (Solver non-copyable; nothing checkpoints). Cross-process = SMT-LIB text round-trip (getAssertions/Term::toString + bundled InputParser, or the vendored bin/cvc5).
  • All of the above are marked experimental — wrap in option probes + try/catch; a cvc5 upgrade may change them (another reason solver version is in the cache key).
  • Why not raw CaDiCaL: it offers clause-level export/import, sound incremental inprocessing, LRAT — but you'd own bit-blasting, arrays (memories!), and UF (boxes), and CNF clauses over integer variables cannot be lifted back to word-level assumes (the whole point of mining). cvc5 already exposes CaDiCaL as its CDCL(T) backend (--sat-solver=cadical) and term-level clause taps via Plugin. Bitwuzla (~2.9× faster raw QF_BV) ships CaDiCaL inside but has none of the introspection.

Traps

  • tlimit-per is per-checkSat — with per-obligation checks the count of checkSats grows; only the scheduler's global budget prevents (N+1)×timeout runs.
  • Deepening after assuming — on --deepen, frontier facts are already horizon-clipped by construction; sidecar facts are inductive so exempt; anything bounded from L2/L3 must be re-proven at the new bound before reuse (rule B).
  • Never cache Unknown; SAT only with its revalidatable witness; hash after the encoder; solver identity in the key (rule F).
  • Experimental cvc5 APIs — probe options at startup (getOptionNames), degrade gracefully (mining off) rather than crash.
  • Symbol-name determinism — st<uid>/s0_* naming must be encode-order-stable or L2/L3 keys churn (completeness, not soundness: a rename is a miss, never a wrong bind).
