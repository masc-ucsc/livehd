2i-import Import & library-link redesign — directory namespaces, structural-id merge, parallel build

Rebuild the Pyrope import experience around three ideas, settled in a design back-and-forth with the user. (1) A library is a directory; the pub modules inside it have flat, short names (no file.entity dotting). (2) A module's identity is its structure — the forward-canonical digest (semdiff::canonical_digest, owned by 3f-semdiff, built on pass/semdiff's forward signature), not its name or its file path. (3) Same-name modules coexist: internal names are unique file.entity (the 2026-07-04 lg-drop), so at link two structurally-identical modules dedup by digest while two different modules sharing a name simply keep their distinct qualified names; only Verilog emission resolves the name — bare entity when unique, qualified on collision (landed). The same .prp compiles unchanged in the interactive single-file flow (imports resolve to sibling source) and a hermetic parallel-bazel flow (imports resolve to precompiled artifacts, discovered by lhd scan).

This replaces the friction documented in two real Verilog→Pyrope porting logs (the DINO pipelined RISC-V CPU translations): "you must list every dependency file", permanent false LSP "unresolved import" squiggles, \file.entity module names, the filename==modname instantiation failure, and the catch-all "no hardware lowering yet" reject. The issue IDs each piece closes are tabulated under What this closes.

Mental model (one line)

A library is a directory; pub modules in it have flat short names; a module's true identity is its forward-canonical structure digest, so names are just labels — internal names are unique file.entity, same-name modules coexist, and only Verilog emission disambiguates (bare entity when unique, qualified on collision).

The user experience — interactive flow (the daily loop)

Imports are logical names, resolved relative to the importing file's own directory (not cwd, not an ancestor crawl). One file on the command line transitively pulls in its siblings.

# cpu.prp:   const alu = import("alu")          # logical name
#            const rf  = import("ip/regfile.regfile")
#            const pll = import("lg:sky130_pll") # explicit external blackbox only

lhd compile foo/cpu.prp                     # resolves alu -> foo/alu.prp,
                                            # ip/regfile -> foo/ip/regfile.prp,
                                            # transitively; no file list needed
lhd compile foo/cpu.prp --emit verilog:soc.v

# emitted module names are BARE:  module alu (...)   not  \foo.alu
  • The LSP uses the same resolver, so opening one file resolves its imports off disk — the permanent false "unresolved import" squiggles disappear.
  • lg: / ln: never appear in project source — only as the deliberate override for a fixed external netlist (vendor macro / blackbox IP) that must always bind to a precompiled artifact and must never resolve to source.

The user experience — hermetic parallel-bazel flow

The same source, no edits. lhd scan drives BUILD generation; each library compiles in parallel to a lg: artifact; a link target merges them.

# gazelle runs `lhd scan *.prp`, reads the raw imports, emits BUILD:
prp_library(name = "alu", srcs = ["alu.prp"])
prp_library(name = "cpu", srcs = ["cpu.prp"], deps = [":alu"])   # alu artifact in
                                                                 # the sandbox

# each prp_library -> `lhd compile alu.prp --emit-dir lg:alu/`   (PARALLEL)
#   the cpu artifact records its `alu` instance as a hash-ref (canonical digest),
#   not a hard name binding.

prp_link(name = "soc", deps = [":cpu", ":gpu"])
#   -> `lhd compile --in lg:cpu/ --in lg:gpu/ --top cpu --emit verilog:soc.v`
#      (no `lhd link` verb: `lhd compile` links when given lg: inputs, gcc-style)
#      load all -> bottom-up digest -> bucket -> structural_match confirm
#      -> dedup identical; distinct same-name modules coexist (unique file.entity)
#      -> emit (bare entity name when unique, qualified on collision)

Hermetic: resolution sees only the sandbox-declared dep artifacts, no filesystem crawl. Incremental: touching alu.prp changes alu's digest and relinks only its hash-affected dependents.

Design decisions (locked)

  • Namespace = directory; names flat within. A graph library (a directory / an lg: artifact) is the namespace boundary. pub comb alu emits module alu. No file.entity dotting, no backslash-escaped names in the common case.
  • Identity = forward-canonical structure digest, origin-ignored. Two modules are "the same" iff their lowered graphs share the canonical digest — regardless of file path or even module name. The digest is 3f-semdiff's canonical_digest over the pass/semdiff forward signature (deterministic, order-independent), so it is stable across independent compile processes — the property the parallel build needs.
  • The two failure modes are not symmetric, so dedup is sound, never merely heuristic. A false merge (collapsing two different modules) is silently-wrong hardware — forbidden. A false split (keeping two copies of the same module) is still-correct hardware, just a duplicated module — tolerable. So the rule: dedup only when sameness is proven (digest bucket → structural_match confirm), otherwise the modules simply coexist.
  • Distinct same-name modules COEXIST — no merge-time rename (final, post lg-drop 2026-07-04). Internal names are unique file.entity, so two distinct digests sharing a short name never clash inside the design: nothing is renamed at link and no instance wiring is re-pointed. The name is resolved only at Verilog emission — each module flattens to its bare entity name when that is unique across the design and keeps the qualified name on collision (landed: inou/cgen/inou_cgen.cpp:56-91 flat_names).
  • The digest is bottom-up (Merkle). A parent's identity must include its children's identities — else two parents with identical bodies but different child alus would false-merge. Inject each resolved child digest at its instance boundary (treat it like a leaf constant) and the parent's forward-canonical digest incorporates it for free; the existing foo__u8 / foo__u16 specialization split already fits this. Consequence: a single differing leaf re-distinguishes its whole instantiation cone — the big dedup win is shared low-level libraries (common cells, a shared regfile/adder), not near-identical top-levels.
  • Resolution is importer-directory-relative; ambiguity is an error. Interactive import("x") resolves against the importing file's own directory (and paths under it), first and only. A name reachable from two roots / two inputs that are not the same module ⇒ error listing the candidates (reuse the call_resolver ambiguity check). -I is an optional escape hatch for unusual layouts, ambiguity-checked the same way.
  • Source imports are logical; lg: is the explicit override. Bare import("alu") binds to source (interactive) or artifact (bazel) by resolution context — same .prp, both flows, zero edits. Explicit import("lg:vendor_pll") always binds the precompiled netlist (external IP / blackbox) and never resolves to source. Where each is found: a Pyrope source import is searched only in the importing file's own directory or an explicit -I root (no cwd, no ancestor crawl); an lg: import is searched at the graph-library location. There is no default lgdb/ dumped in the source tree (removed — dissolves 6b).

Merge / link algorithm (no separate verb — lhd compile, gcc-style)

There is no lhd link. lhd compile compiles, links, or both depending on its inputs — like gcc over .c / .o. Given several lg: inputs and a --top it runs the merge below; given source it compiles; given both it does each. lhd scan stays a separate, build-system-facing tool — bazel/gazelle or makefiles read its raw-import JSON to generate deps; it is not on the compile path.

  1. Load every unit graph from the inputs (artifacts in the bazel flow; the transitively parsed source closure in the interactive flow).
  2. Walk the instance DAG bottom-up; for each module compute semdiff::canonical_hash, injecting each already-resolved child's digest at its instance boundary (Merkle).
  3. Dedup key = digest. Bucket all modules by digest; within a bucket run structural_match to confirm, then collapse the bucket to one module and rewire all instances of any member to it. (Distinct names, same digest ⇒ still one module under a single canonical name — origin/name is ignored per the identity rule.)
  4. Name coexistence — no rename. Surviving (distinct-digest) modules that share a short name simply coexist under their unique file.entity internal names; nothing is renamed and no instance is re-pointed. Report every dedup.
  5. Emit. Verilog emission flattens each module to its bare entity name when that is unique across the design and keeps the qualified name on collision (landed: inou/cgen/inou_cgen.cpp:56-91 flat_names); instance references match the emitted name.

Artifact format (forward-canonical hash-ref)

Extend the per-library lg: artifact (its library.txt / manifest.json) so the link step can dedup and rebind without re-elaboration:

  • Per module: its forward-canonical digest and its port signature (names, widths, signedness, order).
  • Per Sub instance: a by-digest reference to the instantiated module (the child's digest) instead of a hard name binding — so a dedup at link is a pure relabel, and a cross-library instance binds to whichever artifact carries the matching digest.
  • A per-library compile consumes its own .prp files plus the lg: artifacts of the libraries it imports (to read dep interfaces and embed dep digests). It emits only its own modules.

What this closes (from the porting logs)

  • A7 / 6a — no on-disk import search → importer-dir-relative resolution makes lhd compile foo/bar.prp (and the LSP) resolve siblings automatically; no more "list every file".
  • 6c — \file.entity escaped names → Verilog emission flattens to bare module names by default, qualified only on a real collision (landed via flat_names).
  • 2f / A1 — cross-file instantiation "no hardware lowering yet" and the filename==modname shadow → identity is structural and library-scoped, so name==filename is irrelevant; cross-file instances are first-class link hash-refs.
  • C1 — the catch-all "no hardware lowering yet … wrap/sat pending" that bundled callee-not-found, name collision, and missing file → cross-file instantiation is a defined link step, and unresolved/ambiguous imports get their own named diagnostics.
  • The cross-project collision the user led with — two designs both defining alu → dedup at merge if structurally identical; otherwise both coexist under their unique file.entity names and Verilog emission keeps the qualified name for the colliding one — never a silent clash.

Traps

  • The digest MUST be process-independent. It cannot depend on node gids, allocation order, or any per-run state — two libraries compiled in separate lhd invocations must produce comparable digests. The pass/semdiff forward signature is already canonical (commutative-within-port, IO-by-name, bits-folded); a digest must preserve exactly that and add nothing run-dependent.
  • Never bucket-and-dedup on the digest alone. A digest collision must be confirmed by structural_match before merging, or a hash clash becomes wrong hardware. Hash narrows candidates; the traversal decides truth.
  • Two libraries with same-named modules keep independent attr stores. Same caution as 3f-semdiff: hold the Graph* per library; don't let one Hhds_graph_library alias gids across the two.
