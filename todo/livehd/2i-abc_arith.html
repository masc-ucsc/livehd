2i-abc_arith ABC arithmetic bit-blast — Sum + comparators, selectable adder architectures

The combinational 2a-abc tech-mapper bit-blasts only the trivial cells (And/Or/Xor/Not/Mux/Hotmux/Get_mask/Set_mask/Sext/const). Any non-trivial arithmetic cell hits the loud unsupported-cell diag — e.g. the simple_add.v adder:

lhd elaborate inou/yosys/tests/simple_add.v --emit-dir lg:tmp4
lhd pass color synth lg:tmp4
lhd pass abc lg:tmp4 --emit-dir lg:tmp4b
# livehd:error:pass.abc: cell 'sum' in region 'simple_add__c2' has no
#   combinational bit-blast yet (supported: and/or/xor/not/mux/hotmux/const)

This task expands the non-trivial cells into gate networks of the existing trivial primitives before ABC, and adds the first architecture-selectable expansion: the adder. A Sum can be lowered as a ripple-carry, carry-skip, or carry-lookahead adder via --set pass.abc.adder=…, with a block_size knob for the blocked architectures. The code is structured so future complex blocks (the barrel shifter is next; multipliers, prefix adders later) plug in without touching the per-cell dispatch.

Scope (decided)

  • Implement now:
    • Ntype_op::Sum — addition (pid 0 = A, n-ary additive) and subtraction (pid 1 = B, n-ary subtractive). Result = ΣAᵢ − ΣBⱼ.
    • Comparators Ntype_op::LT, Ntype_op::GT (binary, pid 0 = A / pid 1 = B) and Ntype_op::EQ (n-ary all-equal). LE/GE/NE are not cells — they are synthesized as Not of GT/LT/EQ upstream and never reach ABC, so they need no handling here.
  • Stays unsupported (unchanged): Mult, Div, Mod, SHL, SRA, Ror — keep emitting the existing unsupported-cell diag exactly as today. (Per user: "the other blocks should just fail for the moment like sum does now.")
  • Adder architectures now: rca (ripple-carry), cska (carry-skip), cla (carry-lookahead). Future (not now): Han–Carlson, Sklansky, Brent–Kung, … — the framework must accept them as drop-ins.
  • Next task (not here): the barrel shifter with the same option pattern (pass.abc.shifter=…). See the Pre TODO bullet.

Locked design decisions

  • Seed-the-AIG, let ABC optimize. The chosen adder is only the initial structure handed to the existing ABC flow (strash; &get -n; &fraig -x; &put; &get -n; &dch -f; &nf {D}; &put). ABC re-synthesizes the AIG, so functionally-equivalent adders (RCA/CSKA/CLA) may converge to similar or identical mapped netlists — that is accepted. No structure-preservation mode; the knob seeds ABC's starting point. Correctness is the bar; observable structural difference is not required.
  • Default architecture = rca (needs no block_size).
  • block_size sets the CSKA skip-block / CLA lookahead-group width; the last block may be narrower. When unset for cska/cla it is auto-derived from the adder width W: 
    block_size = (W > 16) ? W/4 : (W > 8) ? W/2 : W      // clamp to ≥ 1
    User override via --set pass.abc.block_size=N. (RCA ignores it.)
  • Comparators reuse the adder. LT/GT are a subtraction (a + ~b + 1) read off the carry-out / sign bit, so they go through the same selectable adder. EQ is pure Xor+reduce (no adder).

CLI surface

The adder options are pass.abc.adder and pass.abc.block_size, under the command-path namespace (lhd pass abc ⇒ pass.abc.*):

lhd pass abc --top add_test lg:orig --emit-dir lg:net \
    --set pass.abc.adder=cla --set pass.abc.block_size=4
lhd pass abc … --set pass.abc.adder=rca        # default; block_size ignored
lhd pass abc … --set pass.abc.adder=cska       # block_size auto = W/4|W/2|W
lhd pass abc … --set adder=cla                 # abbreviated (prefix supplied by 'pass abc')

# pass.abc.* (all abc options): library, flow, seq, delay, load, verbose, adder, block_size

The --set namespace + abbreviation machinery is a separate prerequisite, 2h-set_path: it makes every option namespace match the command path (pass.abc.*) and lets the key drop any prefix the command words to its left already supply — so --set pass.abc.adder=cla, --set abc.adder=cla (after pass), and --set adder=cla (after pass abc) are all equivalent. 2i only has to register the two new labels (adder, block_size) on the pass.abc method; the kernel parsing lands in 2h.

Code structure — the complex-cell expander seam

Keep the trivial-gate AIG constructors (abc_const_bit/abc_not/abc_bin/ abc_bit) in abc_map.cpp exactly as they are. Add a backend-agnostic bit-vector algebra + adder library that complex cells lower into. The library never includes ABC headers, so it is unit-testable against a plain software bit model.

// pass/abc/abc_arith.hpp  — header-only templates, no ABC dependency
//
// Ops is any type exposing: Bit zero(); Bit one(); Bit inv(Bit);
//                           Bit and_(Bit,Bit); Bit or_(Bit,Bit); Bit xor_(Bit,Bit);
// Bit vectors are LSB-first, equal length == the adder width W.

enum class Adder_kind { rca, cska, cla };
std::optional<Adder_kind> parse_adder_kind(std::string_view);   // diag on unknown
int  default_block_size(int width);                              // W/4|W/2|W, ≥1

template<class Bit> struct Add_result { std::vector<Bit> sum; Bit carry_out; };

template<class Bit, class Ops>
Add_result<Bit> build_add(Adder_kind, int block_size, Ops&,
                          std::span<const Bit> a, std::span<const Bit> b, Bit cin);

// shared primitives the builders + comparators use
template<class Bit, class Ops> struct FA { Bit sum, carry; };
template<class Bit, class Ops> FA<Bit,Ops> full_adder(Ops&, Bit a, Bit b, Bit cin);
template<class Bit, class Ops> std::vector<Bit> bv_invert(Ops&, std::span<const Bit>);

// comparators (reuse build_add for LT/GT)
template<class Bit, class Ops>
Bit build_lt(Adder_kind, int bs, Ops&, std::span<const Bit> a, std::span<const Bit> b, bool is_unsigned);
template<class Bit, class Ops>
Bit build_eq(Ops&, const std::vector<std::span<const Bit>>& ops_bits);   // all-equal

Adding a new architecture = new *_add helper + an Adder_kind entry + one parse_adder_kind string + one build_add switch arm. Adding a new complex cell (barrel shifter next) = a new builder in this file + a new dispatch arm in abc_map.cpp that speaks only Ops. This is the documented seam for "non-trivial gates expand to complex blocks."

Wiring in abc_map.cpp

  • Map_options (+abc_map.hpp): add Adder_kind adder = rca; and int block_size = 0; // 0 ⇒ auto. pass_abc.cpp parses pass.abc.adder (diag on unknown) + pass.abc.block_size.
  • Build one Ops adapter per region wrapping the existing closures: zero()=abc_const_bit(false), one()=abc_const_bit(true), inv=abc_not, and_/or_/xor_ = abc_bin(·,·,'&'|'|'|'^'). Bit = Abc_Obj_t*.
  • New dispatch arms before the final else (the unsupported-cell fallthrough stays for Mult/Div/Mod/shift).

Per-cell lowering (exact)

Sum

  • Gather A drivers (get_port_id()==0) and B drivers (==1) via inp_edges (n-ary each).
  • W = out_bits (the cprop/upass.bitwidth-resolved result width — already wide enough for carry growth, e.g. u32+u32→u33). Build each operand as W bits with abc_bit(drv,i), which already sign/zero-extends per the driver's own signedness.
  • acc: if A non-empty, acc = A[0]; for each further A[k], acc = build_add(kind, bs, acc, A[k], cin=zero).sum. If A empty, acc = {W × zero}.
  • For each B driver Bⱼ: acc = build_add(kind, bs, acc, bv_invert(Bⱼ), cin=one).sum (two's-complement subtract acc − Bⱼ).
  • Emit acc[0..W) as the output bits (carry beyond W dropped = truncation to the declared result width). Single A & no B ⇒ acc = A[0], no adder built.
  • bs = opts.block_size > 0 ? opts.block_size : default_block_size(W).

LT / GT

  • GT(a,b) ≡ LT(b,a) — implement by swapping the operands into build_lt.
  • build_lt(a,b): unsigned = is_unsign(a) && is_unsign(b); W = max(bits(a),bits(b)) + 1 (one guard bit ⇒ a−b can't overflow W bits). Operand vectors via abc_bit (auto-extend per own sign). (diff,cout) = build_add(kind, bs, a, bv_invert(b), cin=one) ⇒ a − b. lt = unsigned ? inv(cout) : diff[W-1] (no carry-out/borrow ⇒ a<b unsigned; guard-bit sign ⇒ a<b signed; mixed-sign falls out because the unsigned operand zero-extends into a non-negative signed value).
  • Result is 1-bit: out[0]=lt, any higher out bits = zero.

EQ (n-ary all-equal)

  • ≤1 driver ⇒ const 1. Else W = max(bits(dᵢ)) + 1 (guard bit so a signed-negative never equals an unsigned value by bit accident).
  • eq = ⋀_{j≥1} ⋀_{b<W} inv( d_j[b] ⊕ d_0[b] ) using abc_bit-extended bits. No adder.
  • out[0]=eq, higher bits zero.

Verification

  • Framework unit test (subtask 1) — exhaustive-ish functional check of each adder architecture + each comparator against a software reference, decoupled from ABC/Liberty. Catches construction bugs (carry chains, block boundaries, sign/guard-bit handling) cheaply and deterministically.
  • End-to-end LEC (subtask 3) — the only way to confirm the bit-blast is sound after ABC's lossy optimization. netlist module ≡ restruct twin via lhd check with the gensim cell models linked, for each architecture and a non-default block size. A negative control (deliberately wrong adder) must fail LEC.
