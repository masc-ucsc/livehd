1b Bundle storage split — attrs / named / unnamed; drop the __ prefix

Bundle (lnast/bundle.{hpp,cpp}) is the comptime value/tuple representation the upass runner threads through every pass. Today it stores everything — attributes, named fields, and positional fields — in a single std::map<std::string, Entry, Canonical_less> keyed by a flat dotted path (foo.bar.__bits, 0.__enumentry, name, …). Attributes are told apart only by a __ prefix on the last segment. Every operation therefore re-parses strings: prefix matching (match_first_partial), category sniffing per segment, the del_int walk, the concat group-by-segment, and the scalar→.0 promotion passes.

This task replaces the flat map with a recursive node holding three typed containers per level, and drops the __ attribute prefix entirely. The payoff: traversal and copy become structural instead of string-driven, the sorted-iteration views collapse to plain concatenations, and the data/attribute namespace overlap (a field literally named __x aliasing the attribute space) disappears.

Depends on

None — internal IR refactor. The concat / scalar-collapse / promotion semantics it preserves are LEC-pinned by the existing tuple tests, so it can land independently of 1d / 1r (both also touch upass tuple handling — sequence to avoid churn, but no hard ordering).

Target shape

Each Bundle node becomes a tree node carrying its own scalar plus three containers (sub-bundles are shared_ptr<Bundle>; bare names throughout — no __):

  • Entry trivial + immutable — the collapsed scalar value when this node is a leaf (today's "0" / scalar case).
  • std::map<std::string, Entry> attrs — attributes at this level, keyed by bare name (bits, not __bits). foo.__bits becomes named["foo"]->attrs["bits"].
  • std::map<std::string, std::shared_ptr<Bundle>> named — named children (already sorted → alphabetical iteration for free).
  • std::vector<std::shared_ptr<Bundle>> unnamed — positional children, integer index = vector slot.

Canonical order (attrs < named-alpha < unnamed-numeric) is then implicit: iterate attrs, then named, then unnamed. Canonical_less and the per-segment segment_category string sniffing go away.

The API reshape is narrowly about avoiding copies — not a redesign. The legacy dotted-key methods are fine and stay. Today the sorted-iteration views (entries()/non_attr_entries()/get_attrs()/top_levels()) materialize a fresh std::vector on every call (bundle.hpp: "Materialized on each call"). With the data already living in three ordered containers, the node can expose direct, zero-copy accessors — attrs(), named(), unnamed() returning references/spans into the containers — and hot consumers read those instead of paying a per-call allocation. Where direct access doesn't fit, the existing API is kept as-is.

Design notes

The __ alias bug this fixes. Because attrs and data share one keyspace, a data field literally named __x would land in the attribute namespace, and the detector has to special-case ___ (triple underscore) to avoid false positives (bundle.hpp:232, bundle.cpp:373,618). With separate containers the namespaces no longer overlap — a field bits and an attribute bits coexist cleanly. The Pyrope-facing doc rule already says attributes have no __ prefix (pyrope_bundle_attribute_rules); this brings storage in line with the contract.

Attribute names actually in flight (full set, so B can be exhaustive): bits, sbits, ubits, min, max, dp_assign, enumentry, pub, pub_unit. Note ubits/sbits appear only in bitwidth.cpp's suffix checks and may be stale vs. the doc rule (size = bits/max/min only) — confirm and prune while there.

Behavior that must round-trip exactly (the risky corners — all currently LEC-verified):

  • Scalar carriers with attrs. add_int copies attribute leaves under a scalar key so get_bundle(key) round-trips the parse-time __enumentry tag even though is_scalar() ignores attrs. In the new shape that is "a leaf node may still have a non-empty attrs".
  • Scalar→.0 promotion. The set() promotion pass that rewrites a scalar slot to slot.0 when a deeper insert arrives (and the symmetric concat promote_scalar_to_sub) becomes a structural "wrap the leaf's scalar into unnamed[0]".
  • concat merge rules. attrs = rhs-wins; named = per-name merge with nil-drop and scalar-pack into a growing sub-tuple; unnamed = append at the next free index. Preserve precisely — these are pinned by the ++ / bundle-literal tests.
  • Single-element collapse. (c=3) == 3 — a one-entry bundle reads as its scalar while still carrying the one field.

Blast radius (small, scoped). Attribute-API callers: inou/prp/prp2lnast.cpp (10× has_attr), upass/constprop/upass_constprop.cpp (get_attr/set_attr ×4, get_attrs ×2), upass/core/upass_core.hpp, lnast/tests/symbol_table_test.cpp. Iteration consumers: constprop + the symbol-table test only. __ hard-codes outside the bundle: bitwidth.cpp, cgen_verilog.cpp. The public API stays stable through A–C (so that diff is just B's two suffix-matchers + the test); the wider cross-file churn is deliberately concentrated in D, where constprop moves onto the zero-copy accessors to shed the per-call vector materialization.
