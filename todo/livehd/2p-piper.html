2p-piper pass.piper — move pipeline-timing resolution out of upass_tolg into an lgraph pass

Goal (unchanged): consolidation. The pipeline-timing resolution — σ interval propagation, @[N] discharge, depth narrowing — lives today as an inline class at the tolg seam. Extract it into a standalone, re-runnable lgraph pass pass.piper (no pass/piper/ exists yet), leaving tolg to lowering + attr stamping only. The original design premise is dead: there is no Ntype_op::Stage cell and none is needed — the landed mechanism carries pipeline depth as comptime pins on plain Flop cells, and the resolution/narrowing/discharge already landed inside tolg. This page re-specs the extraction against that reality.

Landed mechanism — the starting point

Pipeline depth is a cell parameter, never replicated cells: every Flop has comptime const pipe_min/pipe_max sink pins (pids 8/9, graph/cell.cpp:248-257; unset = depth 1; min==max = fixed; min<max = tool's choice — cgen realizes pipe_min stages, cgen_verilog.cpp:2167-2180). upass.timecheck gives LNAST-level best-effort early diagnostics (discharges trivially-fixed @[N] by stamping const("checked"), defers hard cases); upass.pipe appends the %pipe_* output flops for pipe lambdas. The real work is the inline Time_checker (upass_tolg.cpp:5499-6258, run at the tolg seam ~6885 for pipe/mod/comb): (1) Tarjan SCC classifies flops STATE vs feedforward stage (comb-loop + register-feedback errors); (2) two-pass forward (min,max) σ interval propagation (inputs (0,0), consts unify with anything, comb = equal-meet, stage Flop adds its depth, Sub adds its pinned attrs::time_range, state pins σ(q)=σ(din)); (3) narrows LN-inserted pipe output flops to the body deficit by rewriting their pipe_min/pipe_max const pins — (0,0) realizes as a wire, σ>min is the latency-exceeded error; (4) discharges every pending_time record (attrs.hpp:208-215) — verified records get .del(), mismatches error. The stage[N]/@[N] LEC equiv suite (prp-equiv-*, inou/prp/tests/equiv/) and most error cases (inou/prp/tests/errors/) already exist — they are the parity bed for the migration, not pending work.

Pending

  1. Extract Time_checker → pass/piper/. The blocker is that the checker consumes six in-memory maps handed over from the lowering builder (take_pending_checks / flop_depths / sub_times / plain_reg_flops / inserted_flops / wire_cuts, upass_tolg.cpp:5450-5469) — that coupling is what keeps it inside tolg. Three are already durable on the graph (pending_time attr, pipe_min/pipe_max pins, Sub attrs::time_range); the other three (plain_regs_ state-eligibility, inserted_ LN-pipe-flop marker, wire_cuts_ comb-cycle markers) need small marker attrs or structural re-derivation. Standard plumbing: pass/piper/{BUILD, pass_piper.{hpp,cpp}, piper_test.cpp}, Pass_plugin registration (template pass/sample), pass order tolg → piper → cprop. Keep the comb-top acyclicity-only phase (2c-wire) working across the seam — either piper handles comb tops too, or tolg retains a minimal comb-loop check.
  2. Real range narrowing. Today narrowing only fires on LN-inserted output flops with a fixed body σ ("ranged body sigma — declared depth stands", upass_tolg.cpp:5956), and discharge demands exact interval equality (upass_tolg.cpp:6038) even though the attr contract documents "computed ⊆ asserted" (attrs.hpp:208-212). piper should do the constraint intersection the original spec wanted — intersect ranged flop depths against @[N]/interface constraints, pick the minimum legal depth where a range stays free, error on empty intersection — and settle subset-vs-equality for discharge.
  3. 2 missing error tests (3 of the 5 originally listed exist: cycle_check_mismatch/cycle_lhs_mismatch, pipe_bad_mix/call_operand_misaligned, reg_home_mismatch). Re-specified: (a) unsatisfiable depth range — a ranged stage[A..=B] whose interval cannot meet a fixed @[N]/output cycle (the ranged-vs-fixed conflict path is untested; mod_cycle_range only covers fixed-vs-declared-range); (b) leftover-obligation guard — the residual-Stage-at-cgen guard is obsolete (ranged pipe_min<pipe_max flops are legal at cgen), but its landed analogue is real: "records still present after the checker are a compile error" (attrs.hpp:212) has no enforcement point outside the checker's own run — piper should own a leftover-pending_time check, exercised by a gtest on a hand-built graph.
  4. Decide where the global σ timing lives. Today σ is computed per-lambda at the tolg seam and thrown away; cross-module latency flows only through the pinned per-instance time_range on Sub nodes. Decide: v1 piper stays per-graph with Sub contributions (matching today), or persists σ/per-module latency (an attr or library side table) so a later hierarchical run and LSP queries can read it without re-solving.

Dropped from the old spec

  • Ntype_op::Stage cell, is_type_stage(), tolg create_stage_cell emission — superseded: depth pins on plain Flops landed instead.
  • Residual-Stage guards in cgen/abc/lec/writers — nothing undecided survives; a ranged depth is a legal parameter (cgen realizes pipe_min).
  • Stage-rewrite step + per-rewrite LEC — flop chains are realized directly by cgen from the depth parameter; equivalence is covered by the existing prp-equiv-* suite.
  • Folding upass.pipe into piper — insertion is already unified on ranged Flop pins; upass.pipe stays as the one LNAST insertion site, piper owns only the narrowing.
  • Still out of scope (as before): piper-side retiming/rebalancing and .[valid] bubble clock-gating — synth owns physical placement.
