2p-piper pass.piper — lgraph pipeline timing: verify @[N] + insert stage

Pyrope's pipeline timing has two primitives (06c-pipelining): @[N] — a pure, exact cycle-count type check (asserts a value lands at absolute cycle N counted from the enclosing mod/pipe inputs; never inserts flops; mismatch = compile error), and stage[N] — the action that inserts N pipeline stages (the RHS delivered N cycles later; mod-only). Today this logic is smeared across three places and the insertion side is wrong/incomplete, so a verifier run against the lowered graph is checking a graph that does not yet reflect real stage semantics. 2p-piper adds a single lgraph pass pass.piper that owns both the verification and the flop insertion, decided where the global timing information actually lives.

Why one pass. tolg lowers statement-by-statement and has no global timing picture, so it cannot soundly decide how many flops a stage[…] needs (especially ranged / tool-picked counts — it currently hard-errors not supported yet at upass_tolg.cpp:2539,2584). piper holds the σ (stage) solution and the @[N] constraints, so the decision belongs there. You cannot meaningfully verify until insertion is correct, hence verify + insert together.

New cell: Ntype_op::Stage

A first-class undecided pipeline stage: the value exists but its flop count is not yet chosen. Codegen is illegal while one survives — the gate that enforces "we cannot generate code until the stage is decided".

  • Shape. Stage(din, pipe_min, pipe_max[, clock_pin, reset_pin]); driver pin 0 = the staged value. pipe_min/pipe_max = allowed flop-insertion count (pipeline depth, relative, ≥0) carried as const-driven sink pins — the same sink names the Flop already uses for its pipe depth (upass_tolg.cpp:2603-2604). Named pipe_* deliberately — not min/max, which would collide with bw_min/bw_max (the integer bitwidth range, a different concept). 0 flops ⇒ resolves to a wire (bypass); k ⇒ a k-deep flop chain.
  • Width comes from din. The staged output inherits din's bits/sign ("whatever din had") — no bw_* stamping on the cell, only the pipe_* depth. Resolves the 1-bit-truncation trap that bites hand-stamped cells.
  • Clock required, reset optional. clock_pin is needed to materialize the flops; reset_pin defaults to none — a pipeline-stage flop is pure feedforward transport (reset value don't-care / retiming-free), so reset is carried only when the source semantics demand a reset value.
  • Always feedforward. State lives in reg, never in a stage — so in piper's SCC a Stage is a cut edge, never part of a state loop (a Stage with a possible 0 count only ever appears where it cannot close a comb loop: output padding feeds nothing internal).
  • Plumbing. add the enum + attr registration in graph/cell.hpp / graph/cell.cpp (registration site cell.cpp:34-35); add is_type_stage() next to is_type_flop/is_type_register in graph/node_util.hpp:113-121.

tolg — emit Stage cells (unify every insertion point)

The Stage cell subsumes all deferred flop insertion, one mechanism:

  • create_stage_flop (upass_tolg.cpp:2508) → create_stage_cell: emit a Stage with the legal pipe_min/pipe_max range instead of materializing Flops. fixed stage[N] on a value ⇒ pipe_min=pipe_max=N; on a pipe/mod call (picked latency M, callee min cmin) ⇒ the caller-side deficit pipe_min=pipe_max=M−cmin; stage[] / stage[A..=B] / bare-or-ranged pipe ⇒ the full legal range pipe_min=a, pipe_max=b (these are exactly the cases tolg errors on today).
  • pipe-output padding (today's LN-inserted ___pipe_* flops) ⇒ a Stage at the output with pipe_min=0, pipe_max=deficit (σ ≤ N, pad N−σ).
  • Keep the attr-stamping piper reads: pending_time on driver pins / GraphIO sinks (attrs.hpp:194-201), Sub-instance time_range (attrs.hpp:180-187), io_meta stages_min/stages_max (lnast.hpp:123-124).
  • The two currently in-memory bits passed to Time_checker (the plain_regs_ state-eligibility set and wire_cuts_ Verilog comb-cycle markers) become small marker attrs so they survive the pass boundary. Everything else (SCC membership, enable-driven detection, ___pipe_ name tag) is re-derivable structurally.

pass.piper — resolver + verifier

Port the algorithm from the inline Time_checker (upass_tolg.cpp:4125-4815, run() at 4221-4623) into pass/piper/piper_timecheck.{hpp,cpp}, then extend it to resolve Stage cells:

  1. Build node digraph (exclude consts / graph-input pins; honor the comb-cycle wire-cut markers), Tarjan SCC → classify each flop STATE (Verilog-origin OR enable-driven OR in a feedback SCC) vs FEEDFORWARD; a non-state flop inside a feedback SCC = "register feedback through stage registers" error; a flopless SCC = combinational loop.
  2. Forward σ interval propagation (topo order): input pins σ=0; consts unify with any cycle; comb op = equal-meet / interval intersection of operands (empty = cross-stage mix error); Stage(x, pipe_min=a, pipe_max=b) contributes σ(out) ∈ [σ(x)+a, σ(x)+b]; state flop σ(q)=σ(din) (pinned, re-fixpointed); past[n](x) σ=σ(x)+n; Sub adds its pinned time_range.
  3. Pin by constraints. Each @[N] (driver pin / mod-output sink) and the output contract intersects the interval at that point. Where a Stage's effective count is forced to a point, use it; where a range stays free, pick the minimum legal depth (fewest flops). An empty intersection is a compile error naming the offending @[N] / output and both stages.
  4. Rewrite each Stage. chosen k=0 ⇒ bypass din to the consumers, delete the cell; k≥1 ⇒ a k-deep Flop chain (clock/reset from the Stage, bits/sign from din, retime=true so downstream pass.abc/synth may move them), reconnect consumers to the last q.
  5. Discharge. Verify computed σ against every pending_time (exact == for @[N] and mod outputs; ≤ + pad for pipe outputs), .del() the attr on success; any leftover pending_time = compile error (matches the existing leftover-is-error contract, attrs.hpp:189-201, upass_tolg.cpp:4549-4623).
  6. LEC. Each resolved result is checked logically equivalent to the canonical reference (comb body + N appended output flops), the contract the docs guarantee (06c-pipelining "retiming-equivalent to the canonical spelling").

Guards — no codegen with an undecided Stage

A residual Stage cell is a hard error in cgen and every structural consumer (pass.abc, pass.lec, pass.partition, pass.semdiff, the prp/verilog writers): "undecided pipeline stage — run pass.piper first". Centralize via one helper (e.g. graph_util::assert_no_stage(g) or a check in the cgen cell dispatch).

Decisions locked (clarification 2026-06-20)

  • Extract + extend. piper is the single standalone, re-runnable lgraph timing pass; the inline tolg checker is moved into it, not duplicated.
  • Verify + insert together in v1 — because the current (insertion-less / wrong) graph cannot be soundly verified.
  • Literal placement + LEC: minimum legal depth where free, retime=true, no piper-side retiming, .[valid] deferred.
  • Stage cell unifies every insertion point; tolg emits the legal range, piper decides the count.
  • Cell shape Stage(din, pipe_min, pipe_max[, clock_pin, reset_pin]): depth named pipe_* (not min/max — avoids the bw_* bitwidth collision); bits/sign inherited from din; clock required; reset optional (default none, pure-transport flop).
  • @[N] is exact (not "at least N"): the "pad up to / at least N" behavior belongs to plain/pipe-output padding (σ ≤ N) and to stage[N], never to the @[N] check itself.

Risks / open

  • Interval σ over Stage ranges + comb meets is a difference-constraint solve (weighted union-find, per 06c-pipelining); v1 keeps it simple (propagate intervals, intersect at meets, pin at @[N], min-legal where free) — confirm this covers the real corpus before reaching for a general solver.
  • Coverage parity between upass.timecheck (LNAST best-effort) and piper is currently unverified; the migration must not silently drop a check the inline Time_checker caught.
  • Keep pipe_min/pipe_max (pipeline depth) strictly separate from bw_min/bw_max (integer bitwidth) — same shape, different meaning. The staged output's bits/sign is inherited from din, which sidesteps the 1-bit-truncation stamping trap, but the resolver's materialized flop chain must still propagate that width down the chain.
