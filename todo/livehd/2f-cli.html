2f-cli lhd tool — unified ln/lg inspector & attribute debugger

A single lhd tool <verb> command family for inspecting LNAST (ln:) and LGraph (lg:) artifacts — cat / grep / diff / tree — with first-class access to per-node/per-pin attributes (color, bits, signed, src, …). It replaces the current lhd ln.cat / lhd ln.diff (deleted) and folds the LGraph attribute dump/diff into the same surface. Verbs are polymorphic over ln:/lg:; the focus of this task is the lg side, keeping ln consistent.

Why

Attribute annotations (most pressingly color from pass.color, but equally bits/signed/delay/src) live on the LGraph and are today nearly invisible: a developer can run pass.color and get an opaque "uncolored-node" fatal from pass.partition (pass/partition/pass_partition.cpp:248-254) naming exactly one node, with no way to see which nodes lack the attribute or why. The motivating bug: the synth algorithm leaves some nodes uncolored and there is no debugging flow. This task builds the general observable so we (and a coding agent) can:

  • Debug attribute holes — e.g. "which nodes did synth leave uncolored, and where in the source?" → lhd tool grep --target node color:nil lg:cpu.
  • Diff attributes across a change — see what a flag/algorithm/code edit moved → lhd tool diff lg:before lg:after --target node --attr color.

Related: the PRPCOLOR pre-TODO (hierarchical Pyrope-like coloring) and pass/color are the primary consumers of this debugger; the 3y-agent CLI shares the kernel-command and JSONL conventions; this is the same "lhd subcommand over the kernel" pattern as 2n (lhd lsp).

Command surface (agreed)

Form: lhd tool <verb> [options] <inputs…> — inputs are kind-prefixed ln:DIR / lg:DIR; the prefix selects the representation, verbs are polymorphic where it makes sense.

Verbs

verbinputsbehavior
cat1structured dump (nesting/ports/hierarchy preserved); default attrs shown; filters optional (narrow scope, keep structure)
grep≥1search — requires ≥1 filter, may span multiple libs, flat matches prefixed lib/entity, pipe-friendly
diff2unified-diff -/+ output, -C n = classic text-line context; deterministic id-ordered listing keeps hunks stable for same-source compares
tree1instance hierarchy rooted at --top (inst : def + node counts); --hier full by default

cat/diff work on both ln: and lg: (parity with the deleted ln.cat/ln.diff). grep/tree are lg-focused for now; ln support is a later nicety.

Options (shared)

  • --target node|pin|edge|all — default all. The split follows where attributes live: node = color/srcid · pin = bits/signed/delay · edge = pin-to-pin wiring. This is the "nodes vs pins vs connectivity" axis.
  • --top <name> — pick module (defaults to the lib's top; error if ambiguous).
  • --hier [N] — bare = all instance levels, N = depth bound, omitted = flat (just --top's own nodes). tree defaults to full.
  • --attr <csv> — override the default attribute columns.
  • --hops N — focus radius around filter matches (cat/grep); the "show me around this node" knob (replaces a dedicated --focus; you select the center with a name:/id: filter).
  • --max N — row cap, default ~200, --max 0 = unlimited; truncates with a "N more… narrow with --top/filters or raise --max" footer. Primary guard against "very long very fast".
  • --diag-fmt pretty|jsonl — reuse the existing global diag format flag; pretty (greppable) is default, jsonl is the structured form.
  • --emit <file> — reuse the kernel --emit slot to write to a file (else stdout). NOTE: kernel --emit today targets IR artifacts (lg/.v/.prp) and is_kernel_label() reserves files/path/odir — the emit machinery must be extended to accept a text/jsonl tool-output target.

Default attribute columns (--attr overrides; identity kind_nid[:name] always shown)

  • node = color, src
  • pin = bits, signed
  • edge = bits

Filter grammar (shared by cat/grep/diff; space-separated terms = AND)

  • id:12 (numeric → id) · name:Mult (text → name); a bare token auto-disambiguates (numeric = id, non-numeric = name) since ids are numeric and names are not.
  • strings (name:, src:): substring by default, ~ = regex, = = exact — name:add, name:~^alu_, name:=add_3.
  • numerics (id:/color:/bits:): equality + comparisons + ranges — bits:>8, bits:<=4, bits:8..16; color:nil = unset.
  • generic <attr>:value for any roster attr — signed:…, delay:…, src:alu.prp:5 (substring on resolved path:line).
  • edges: from:X = edges driven by X, to:Y = edges into Y; either side optional, combine for a specific wire. X/Y are node/pin selectors (numeric=id, text=name).

jsonl shape (--diag-fmt jsonl)

Flat, one type-tagged record per entity; --target selects which types stream out; nil → null:

{"t":"node","mod":"alu","nid":12,"kind":"Mult","name":"prod","color":null,"src":"alu.prp:5"}
{"t":"pin","mod":"alu","nid":12,"pin":"a","bits":8,"signed":true}
{"t":"edge","mod":"alu","from":"Mult_12.y","to":"Sum_8.b","bits":16}

Canonical uses (the color-debug goal, end-to-end)

lhd tool grep --target node color:nil lg:cpu             # uncolored nodes (== the partition-fatal set)
lhd tool cat  --target node color:nil lg:cpu --top alu   # ...same, in structural context
lhd tool grep name:Mult_12 --hops 1 --target edge lg:alu # wiring around one node
lhd tool diff lg:before lg:after --target node --attr color  # what a flag/algo change moved
lhd tool tree lg:cpu --top cpu                           # instance hierarchy

Design decisions (converged — record of why)

  • Unified lhd tool verb family. One noun (tool) + bare verbs (cat/grep/diff/tree), kind-prefixed inputs. Subsumes ln.cat/ln.diff.
  • Attributes are default-on, not a separate verb. cat lg: always shows a default attribute set; --attr narrows/widens; grep filters; diff compares. "cat shows, grep filters, diff compares" — attributes are a facet of the existing verbs.
  • cat keeps structure. The graph's map/connectivity matters, not just a flat attribute table.
  • --target defaults to all. Informative by default; rely on --top/--hier/--max/filters to trim. --target picks which entity's attributes (node vs pin vs edge), which aligns with where attributes are stored.
  • grep = search, cat = dump. grep requires ≥1 filter and is the multi-lib search (output prefixed lib/entity); cat is the (optionally-filtered) single-artifact dump. Difference is intent, not output shape.
  • diff = grep's filter grammar + unified-diff output. Same filters/target/scope; -C n = classic text-line context; deterministic id-ordered listing (within a module) keeps hunks stable for same-source flag/algo compares. (Pairing is positional over that stable listing — see traps re: code edits that renumber.)
  • tree = instance hierarchy. Per-instance lines (inst : def) rooted at --top, node counts; --hier full by default. (Hardware mental model; mirrors the existing inou.graphviz.fromlg.hierarchy parent→child emitter.)
  • Filter syntax = field:value, AND-combined, substring-default strings (~ regex, = exact), numeric comparisons/ranges, from:/to: for edges, bare-token id/name auto-disambiguation.
  • Size guard = default --max cap (truncate-with-footer), not a verbosity dial — detail is already governed by --target/--attr.
  • Reuse the global --diag-fmt for pretty/jsonl (one switch for all machine output) and the kernel --emit for file output.
  • jsonl = flat, type-tagged, one record per entity (stream/jq/diff-friendly), not nested-per-node.
  • ln.cat/ln.diff are deleted, not aliased — single consistent surface.

Implementation notes / reuse hooks (from codebase survey)

CLI wiring

  • Add a read-only kernel command tool_command(opts,res) routed from run_engine_command() (lhd/lhd_kernel.cpp), modeled on the pass_command / ln_cat_command blocks. Sub-verb from opts.files[0] (cat/grep/diff/tree), inputs via gather_ir_inputs, load_lg_into_var. No Hhds_graph_library::save() — read-only.
  • Delete ln_cat_command/ln_diff_command dispatch + their command names; update callers/tests; emit a helpful "use lhd tool cat ln:…" error for the old names.
  • Flags ride the EPRP registry / --set validation (check_known_set_passes, kSetPasses) like every other command; --diag-fmt/--emit already exist.

Graph walk & attribute access

  • Walk g->forward_class() + livehd::color::is_partitionable(node) — the same walk/predicate pass.color and pass.partition use, so the dump agrees with the consumer. (Do not clone dump_graph_text's fast_class() walk — different order/scope.)
  • Identity: graph_util::debug_name(node) → Type_nid[:name]; node_name_of, type_op_of.
  • color: graph_util::node_color_of(node) / has_node_color(node) — the unified hier-or-flat reader (matches partition). Sentinel NO_COLOR=0.
  • pin attrs: bits_of(pin), is_unsign/pin_signed, delay.
  • src: copy the Cgen_verilog::note_src idiom (inou/cgen/cgen_verilog.cpp:228-238) — node.attr(hhds::attrs::srcid) → g->source_locator().resolve_span(id) → file+start_line (+col via line_offsets).
  • HHDS attribute stores are iterable per-tag (for (auto& [k,v] : g->attr_store(Tag{})), cf. lnast.cpp:82-88) but there is no tag reflection (Attr_tag_registry is private) — so the dumper uses a fixed roster of the known tags in graph/attrs.hpp (color, hier_color, bits, pin_signed, pin_delay, time_range, const_value, srcid). Adding a roster attr is a one-liner; document the roster.
  • Related existing observables to keep in mind (don't duplicate gratuitously): lhd compile --dump lg (dump_graph_text), inou.graphviz.from (already color-codes nodes; uncolored partitionable nodes render plain white — invisible), and inou.attr.save (dumps colors to JSON).

Key correctness traps

  • color:nil ≠ "synth-skipped". In color_synth.cpp the 4 skip sites only stop a node from originating a fresh id; a skipped Mult/Div/ wide-Sum that is a sink of a colored predecessor still gets colored, and conversely the skip cascades (a skipped node never propagates its id to its sinks, so those sinks may end uncolored too). So color:nil is the genuinely-uncolored set (== the partition-fatal set), not a node-class you can kind:Mult your way to. The tool should report the set honestly; it must not claim to explain why.
  • Walk forward_class(), not fast_class() — to match the consumer's node order/scope.
  • hier_color only resolves in a hier-traversal context (node.is_hier()); a flat per-def dump shows flat color. Per-instance color (compact=false) needs the pass_color hier_range walk, not a flat clone. (Note: today apply_coloring writes flat color even in compact=false, so per-instance variance is mostly moot — verify before relying on either way.)
  • srcid gaps: synthesized nodes (finalize_regs / create_stage_flop) carry SourceId_invalid → src shows unknown; combined ids resolve only their primary anchor. src=? is ambiguous (synthesized vs unstamped) — surface it, don't hide it.
  • diff pairing: the unified-diff is positional over the deterministic listing — clean for same-source flag/algo compares (nids deterministic from one compile), noisier for code edits that renumber nids. Accepted tradeoff; a name-keyed or structural mode is a later option if needed.
  • --emit overlaps the IR-emit slot — extend the kernel emit path to accept a text/jsonl tool target without colliding with lg/.v/.prp emission.
