2f-semdiff Structural diff/match between two LGraphs (a structural LEC)

pass/semdiff marks corresponding nodes/pins across two designs with a new match attribute so the diff is greppable. It is a structural LEC: same API shape as pass/lec (two designs, two tops, a C++ API the kernel calls directly), but instead of an SMT proof it establishes a structural correspondence and isolates the regions that differ. It is the MATCH step in the LEC plan (pass/lec/lec.md — "create the PARTITION and MATCH first"): a future lec can run semdiff first, assume the matched region equal, and hand only the unmatched gaps to the solver — shrinking the miter.

Depends on

None to start. Reuses primitives from pass/submatch (commutative-aware woothash structural keys) and pass/lec/encode.cpp (IO-by-name pairing, flop_state_key() name normalization, Sub-by-defname, memory-by-signature).

Concept & CLI — mirror lec

The command and API parallel lec deliberately, so the two stay interchangeable and lec can call semdiff internally.

# structural diff of two designs (two tops, lec-style), marks both lg: in place
lhd semdiff --ref lg:tmp1 --impl lg:tmp2 --top adder
lhd semdiff --ref lg:gold/ --impl lg:opt/ --ref-top rca --impl-top fast_add \
            --set semdiff.matching_names=true --set semdiff.id_granularity=pair

# then inspect via the existing tool grep (match field + -v added by this task)
lhd tool grep -v match=0 lg:tmp1        # what in ref matched
lhd tool grep    match=0 lg:tmp2        # what in impl is unique (the diff)
lhd tool grep    match=3002 lg:tmp1 lg:tmp2   # one corresponding group, both sides
  • Sides & tops: exactly like lec — --ref KIND:PATH / --impl KIND:PATH (verilog:/pyrope:/ln:/lg: or bare path), top picked per side via --ref-top/--impl-top, falling back to --top, falling back to the sole module (reuse the pick() lambda + load_side_graphs() from lec_command, lhd_kernel.cpp:2295-2318).
  • Persistence: v1 marks the two graphs in place and saves both libraries back (Hhds_graph_library::save(dir), like pass color at lhd_kernel.cpp:2514). v1 therefore requires lg: sides for the mark-and-grep workflow; a non-lg: side needs an explicit --emit-dir lg:DIR/ to be greppable. (The C++ API itself is kind-agnostic.)

C++ API — the structural-LEC entry (mirror lec::prove_equal)

namespace livehd::semdiff {

struct Semdiff_options {
  std::string alg            = "structural"; // v1; future: region | functional
  bool        matching_names = false;        // anchor internal flops/mems by hier name
  std::string id_granularity = "pair";       // pair | region
};

struct Match_result {
  uint32_t regions = 0;     // distinct match IDs assigned
  uint32_t a_matched = 0, a_unmatched = 0;
  uint32_t b_matched = 0, b_unmatched = 0;
  double   similarity = 0;  // matched / total
};

// Stamp the `match` attribute on nodes+driver-pins of BOTH graphs: a shared ID for
// corresponding nodes, 0 for nodes with no counterpart. Mirrors lec::prove_equal's
// signature so lec can pre-match before building its miter.
Match_result structural_match(hhds::Graph* a, hhds::Graph* b, const Semdiff_options& opts = {});

}  // namespace livehd::semdiff

Also register a pass.semdiff Eprp method mirroring pass.lec (graphs[0]=ref, graphs[1]=impl) so the engine is reachable as a pass for tests; the lhd semdiff command calls semdiff::structural_match directly, exactly as lec_command calls lec::prove_equal rather than routing through pass.lec.

Design decisions (locked)

  • Match model. Anchored frontier propagation, meet-in-the-middle — neither a global structural-hash bucket nor a strict lockstep.
  • v1 engine. Exact structural, node-level. A + node vs a ripple-carry-adder cone stays match=0 on both sides until a later algorithm; commutative (a+b vs b+a) and identical trees match.
  • Anchors. Primary IO paired by port name; constants by value. matching_names=true additionally anchors internal flops/mems by hierarchical name (default off — structure flows through them otherwise).
  • Direction. Forward from inputs first, then backward from outputs for still-unmatched nodes; either direction matching is enough (union). A branch halts at its divergence frontier, and stops early when it reaches an already-matched node (forward and backward met ⇒ that path is fully equivalent).
  • Gap. The region between the forward and backward frontiers is distinct: match=0 in v1; a future alg=functional gives it its own region ID and matches it n:m (the +↔RCA case).
  • ID granularity. --set semdiff.id_granularity=pair (default) = one ID per matched node-pair (symmetric siblings share one ID, the only v1 n:m); region = union-find transitively-matched nodes between the same frontiers into one ID. Same knob later labels n:m gap-regions.

The match attribute

  • Add match_t { value_type = uint32_t; storage = hhds::flat_storage; } to graph/attrs.hpp (mirror color_t); register "livehd::attrs::match" at static init in graph/cell.cpp (before any library load, so saved graphs deserialize the tag).
  • sparse layout (the default), not dense: dense reserves 0 as the "absent" sentinel, but match=0 must be a real, greppable value (the unmatched marker).
  • Stamp on the node and its driver pin(s) with the same ID, so both node-target and pin-target tool grep work. IDs shared across both graphs (one counter, written to both); assign in forward_class() topological order for run-to-run determinism.

Subtasks — pending

  • v2 — alg=functional (later, enables the LEC win). Per-gap boundary-equivalence via a bounded miter on the existing pass/lec cvc5 path, assigning each proven gap one region ID → n:m +↔RCA. Then wire lec to call semdiff::structural_match first and only solve the unmatched gaps (the problem-size reduction this task exists to unlock). The v1 engine already exposes the kind-agnostic semdiff::structural_match(a, b, opts) entry lec would call.

Tests (lhd/tests/lhd_semdiff_test.sh)

  • Identical designs → every node match≠0; grep match=0 empty both sides.
  • a+b vs b+a → fully matched (commutativity).
  • One side with an extra unconnected gate → that gate match=0, rest matched (assert counts).
  • + vs RCA → v1: both adder cones match=0, surrounding logic matched (documents the v1 boundary; flips to matched under future alg=functional).
  • matching_names: a renamed internal flop that breaks pure structure → off: gap; on: matched.

Traps

  • Sum mixes added (a) and subtracted (b) operands — commutative-normalize within a port-class only, never across.
  • Width matters (unsigned bits = magnitude+1): include bits in the node key so differing widths don't falsely match.
  • Two libraries with same-named modules: separate Hhds_graph_library instances keep gids/attr stores independent — hold the two Graph* directly.
  • match must be registered before any load/save or deserialization asserts.
