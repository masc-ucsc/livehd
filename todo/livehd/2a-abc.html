Conceptual flow

lhd elaborate prp/verilog files                       --emit-dir lg:orig
lhd pass color synth     --top xx lg:orig                                  # colors lg:orig in place
lhd pass abc             --top xx lg:orig             --emit-dir lg:xx_netlist   # partition + ABC map per color
lhd pass partition       --top xx lg:orig             --emit-dir lg:xx_restruct  # same modules, original logic
lhd pass liberty gensim  <file.lib>                   --emit-dir lg:models        # behavioral model per cell
# LEC each pair: lg:xx_netlist/<color> ≡ lg:xx_restruct/<color>  (lg:models resolves the cell Subs)

xx_netlist and xx_restruct are structurally identical (same top, same per-color module IOs); only the module bodies differ ⇒ clean one-to-one LEC pairs.

Depends on

  • 2p-partition — the region→module decomposition. pass abc reuses it: partition already extracts each color's module IOs and builds the per-color body graphs (those bodies are the ABC inputs). ABC just swaps each body's internals for the mapped netlist, keeping the IOs.
  • 2p-partition → "Hierarchical-input partition" (prerequisite — LANDED 2026-06-14). color + partition now descend a multi-module hierarchy: partition processes every def reachable from --top children-first and re-links Sub instances to the partitioned child defs (a pre-existing flop-output cgen name/width collision was fixed in the same change). LEC-verified on the hier_comb/hier_seq fixtures (synth + acyclic) via lhd/tests/lhd_partition_hier_test.sh. So 2a-abc can consume a hierarchical lg:orig. NOTE for abc: runtime wrap/sat has no tolg lowering, so standalone tolg-lowered modules (the partition outputs abc maps) must avoid narrowing assignments.

Locked design decisions

  • Output = real standard-cell mapping. Map to a user-provided Liberty (--set pass.abc.library=…; dev/test default $HAGENT_TECH_DIR/sky130_fd_sc_hd__tt_025C_1v80.lib). Each mapped gate becomes a 1-bit blackbox Sub named exactly after its Liberty cell (Mio_GateReadName). LEC functionality for those Subs comes from the companion pass liberty gensim (below), so LEC stays self-contained with no PDK-Verilog dependency.
  • Seq vs comb, selectable. --set pass.abc.seq=true|false; default false. Build false (combinational cones) first, then true (flops cross into ABC as latches). Either way flops stay native LGraph Flop — ABC only optimizes/maps combinational logic; sequential cells are never mapped to library DFFs.
  • Pass-managed I/O + templated flow. The pass builds the Abc_Ntk from each module, auto-injects read_lib <library>, runs the user's pass.abc.flow command string, then extracts the mapped netlist. flow is pure opt/map commands; {D}/{L} placeholders are substituted from pass.abc.delay/.load (yosys/OpenROAD convention). Sensible defaults per seq/comb when flow is unset.

Flop-name preservation (seq=true) — first-class

A register is one Ntype_op::Flop (Q = driver pin 0, bits=N; pins din(3)/clock_pin(2)/enable(4)/reset_pin(7)/async(0)/initial(1), graph/cell.cpp:234–255); name in hhds::attrs::name (gu::node_name_of, gu::is_type_flop, gu::bits_of). ABC latches are 1-bit, so an N-bit flop ⇒ N ABC latches, and scorr/dretime may change the flop count.

  • Going in: record each region flop's (root_name, bits, srcid, nid); name each ABC latch from the flop name per bit via Abc_ObjAssignName; keep abc_latch_name → (root_name, bit, orig_nid) (the old pass used latchname2id + an _%r regex).
  • Single-root remap (key insight): strip bit/index suffixes (_<n>, [<n>]) to roots; if a region has exactly one distinct root flop name, attribute every ABC output latch (however retiming reshaped them) to that one root and reassemble into a single Flop (bits = Σ widths) named the root.
  • Coming out: recover each output latch's root via the in-map; latches newly minted by retiming inherit the name of the nearest original flop in their fanin cone, else a deterministic <region>__r<n>. Reconnect clock_pin/reset_pin/enable from the stripped side-maps (old conn_latch pattern).

Source-map carry-through — first-class

Per-node provenance is hhds::attrs::srcid (a uint64_t SourceId); cross-graph carry is dst->source_locator().import_from(src->source_locator(), id); many-to-one is src_loc.combine({ids}) (order-significant, parents[0] primary). Partition already stamps srcid on the bodies it builds — ABC discards that when it replaces the body, so pass abc must re-stamp so the netlist points back to the original pre-ABC RTL as closely as ABC's lossy optimization allows.

  • Snapshot each original body node's srcid before conversion; name ABC PIs/POs after the original module IO + matched flops so provenance hints survive strash/dch.
  • On read-back, attribute each generated gate to the output cone it feeds: backward-DFS from each PO and stamp every gate in that cone with the PO's original output-driver srcid (via import_from); a gate feeding several POs gets combine(...). Flops carry their matched original flop's srcid directly.

Companion pass — pass liberty gensim

A separate pass/liberty with a gensim subcommand reads a Liberty file and emits one LGraph behavioral model per cell, so the netlist's blackbox cell Subs resolve for LEC.

lhd pass liberty gensim <file.lib> --emit-dir lg:models
  • CLI: sub=="liberty", positional gensim + a plain .lib file (not lg:) → labels["files"]; require --emit-dir lg:dir → labels["odir"]. Add {"liberty","pass.liberty"} to kSetPasses, //pass/liberty:pass_liberty to lhd/BUILD. pass/liberty/BUILD also depends on @abc.
  • Impl: Abc_Start() + read_lib file.lib, then walk the derived Mio library (Abc_FrameReadLibGen → Mio_LibraryForEachGate). For each combinational cell create a Graph named exactly Mio_GateReadName with input pins = Mio_PinReadName and the gate output pin; lower the function to LGraph And/Or/Not from Mio_GateReadTruth→SOP (Mio_GateReadSop) for ≤6-input cells (formula-parse fallback for wider). Names/pins match the netlist Subs exactly (same read_lib-derived library guarantees it). Sequential cells deferred — flops are native in the netlist.

Verification — LEC each pair

lgcheck (inou/yosys/lgcheck, yosys miter + sat -prove) compiles each side to standalone Verilog but does not resolve blackbox Subs from a separate library. So link lg:models into the netlist library (reuse the kernel's elaborate.merge-style library merge, lhd_kernel.cpp:~516), cgen both sides to Verilog, and lgcheck each module by name. Sequential pairs (post-dretime, differing flop counts) use lgcheck's BMC/induction path with reset sequencing. Ships as a bazel equiv test.

Design notes / risks

  • Bit-blast boundary (multi-bit module IO ↔ 1-bit gates) is the largest new chunk — the old Join/Pick per-bit handling becomes modern bit-select/concat.
  • ABC frame is global/stateful — single Abc_Start/Stop, load library once, reset the current network per module.
  • Mio name/pin fidelity — gensim models and netlist Subs must byte-match; both go through the same read_lib-derived Mio library to guarantee names + pin order.
  • Provenance is lossy — strash/dch destroy per-node mapping; output-cone attribution is the most faithful cheap option (not exact per-gate lineage).
  • Per-partition options (future). The Pre TODO floated per-region ABC options (reg foo::[abc="opt1=2;opt2=xx"]) alongside the global --set pass.abc.*. Start global; per-region flow overrides keyed off the color metadata can come later.
