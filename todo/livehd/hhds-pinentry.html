HHDS PinEntry/NodeEntry native bits + sign storage

LiveHD currently stores per-pin bits (Bits_t / 23-bit) and sign (1 bit) as HHDS attributes (livehd::attrs::bits / livehd::attrs::sign) — see the LiveHD docs. Attributes are flat_storage hashmaps; fine for correctness, but every read does a hash lookup. Native fields on PinEntry / NodeEntry would be one indexed load.

Design notes

Proposed layout (keeps both classes at 32 bytes by dropping ledge1 and widening Nid_bits from 42 to 48, and port_id:23):

  • PinEntry — `master_nid:48 + next_pin_id:48 + ledge0:48
  • port_id:23 + use_overflow:1 + sign:1 + bits:23 + sedges_/overflow_idx:64`
  • NodeEntry — `type:16 + next_pin_id:48 + ledge0:48 + use_overflow:1
  • alive:1 + pad:6 + sign:1 + bits:23 + sedges_extra:48 + sedges_:64`
