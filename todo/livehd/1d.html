1d Demand-driven emit + dead-code elimination

Demand-driven (lazy) emit + dead-code elimination for upass: replace the emit-everything-then-mark-sweep DCE with demand-first emission that materializes each pure value def at its use site and drops undemanded defs for free. The demand-first half already exists (uPass_coalescer); the pragmatic hybrid has landed, with the full sea-of-nodes rewrite deferred.

Depends on

Sequence after 2b (decided 2026-06-09). Doing D after 2b is also cheaper: the per-name Bundle in the runner-owned symbol table is the natural home for the "consumed" bit the demand-driven scheme needs, instead of another side map. Design notes

★ Design rationale + literature references live in the header comment block at the top of upass/runner/upass_runner.cpp — read that first. The demand-first scheme folds each pure value def to a const when possible, else copies the def subtree from the source tree, drops undemanded defs for free, and sinks defs near their uses. In compiler terms this is late scheduling of a sea-of-nodes / partial dead-code elimination (refs in the comment block: Click GCM, Knoop-Rüthing-Steffen PDE/LCM, SSA-DCE, SCCP, VDG).

Remaining — the sea-of-nodes ceiling. The demand-first half already existed: uPass_coalescer (active by default when the tree materializes) parks pure scalar defs, materializes-at-use via runner_emit_at_fn, DSE on overwrite, drops const-known defs. Until the sea-of-nodes IR lands, the post-walk sweep is the architecturally-necessary post-emit pass. See git 1d-D1 / 1d-hybrid A / 1d-hybrid B and the memory note task_1d_dce_hybrid_landed.
