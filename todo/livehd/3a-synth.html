Subtasks — pending

  • A — name-preservation contract through synthesis. The enabling requirement: when pass.abc (or any flatten/inline step) dissolves a module boundary, surviving flops keep their instance-prefixed hierarchical name (inst.reg → inst_reg-style, the same family canon_flop_name already normalizes: $…$ strip, SSA strip, dots→underscores, .reg_ collapse). Without this, post-synthesis LEC loses its tier-1 correspondence and everything below gets harder for no reason.
  • B — flattening-aware semdiff state matching (virtual flatten). Today's tier-2 matcher (3f-semdiff) runs one problem per def pair, so an impl-side inlined module leaves its flops UNPAIRED(no-counterpart) in the parent pair (visible as a ref/impl state-total asymmetry + REF-ONLY child defs in the hier=1 stats — diagnosed, not recovered). Fix inside the per-def model: when the two sides' call structures for a def DISAGREE, the collect_state BFS descends through the mismatched Sub via a resolver callback (child Graph* by gid, sink-port ↔ child-IO-decl mapping from get_subnode_io()): child IOs become stitch points — NOT anchors, the flattened side has no such nets — child flop keys enter the parent problem instance-prefixed (normalized per A, so tier-1 catches preserved names), and the (anchor,dist) tier-2 signatures work over the stitched graph unchanged. Matched children stay opaque (their own def pair handles them) — the same descend-vs-collapse policy as hier-lec. No hhds change needed for tier-2: the BFS is a manual edge walk, so crossing a boundary is bookkeeping ((graph, class_index) visited keys). The hhds visit_io traversal patch becomes relevant only if the v1 full structural match (forward_class-based fsig/bsig) must also cross boundaries.
  • C — acceptance on something small. A small hierarchical design synthesized+flattened through pass.abc LEC'd against its hierarchical source: flops pair by preserved hier name (tier-1); one deliberately renamed flop recovers via tier-2 full-match; the pass.semdiff hier=1 stats sweep shows no state-total asymmetry residue. Wire the pairs into lec's name_alias as tier-2 uncertain pairs under the 2f-lec discipline (drop-and-retry on REFUTE, never bounded-Proven).
