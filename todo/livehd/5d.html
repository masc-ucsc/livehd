Design notes

Test Organization

Tests are scattered across the codebase. Some subsystems place tests in dedicated tests/ directories (core, lgraph, lnast, inou/pyrope) while others have test files alongside source code (as of 2026-07-08: upass — 7 tests — pass/abc, pass/bitwidth, pass/color, pass/sample, pass/semdiff, inou/prp).

Rules to enforce:

  • All main C++ tests must live in a tests/ subdirectory under each component.

  • Contract tests are marked by filename, not by directory: any test/benchmark file whose name contains contract is immutable to coding agents (AGENTS.md; enforced by scripts/constracts/check_contract_untouched.sh). Contracts test only public/high-level APIs and overall integration; they are maintained by hand to guarantee stable interface behavior. The old contracts/-directory migration design is dropped — the open work is coverage: audit existing integration/API tests and rename/add contract-named tests where protection is warranted.

  • Treat compile-driver scripts as contracts, not ordinary tests. Move yosys_compile.sh, slang_compile.sh, and the other *_compile.sh scripts out of tests/ directories and place them under a consistent contracts/ structure.
