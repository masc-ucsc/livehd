2f-fcore Shared formal core — proof-DAG runner, portfolio, unified cache, assumes & hints

Formal has two main components — LEC (2f-lec) and Verify (2f-verify) — and they must share as much code, functionality, and options as possible. They already share the encoder/unroller (pass/lec/encode.cpp, query.cpp), ONE formal.* knob namespace (lec.* accepted as aliases), and the witness→lecfail.prp/formalfail.prp (the formal.prpfail knob)→lhd sim→single-VCD reproduction flow. This page owns the next shared layer: parallel proof orchestration, the verdict cache, generalized case-split/assume machinery, and hint plumbing. 2f-formal (the compile tier) is the third consumer, as the engine's don't-try-hard profile.

Goals (the yardstick for every item below)

  • Fast formal — lec and verify both.
  • Incremental reuse — small edits re-prove only what changed; an untouched submodule is instantaneous (--workdir keeps the cache).
  • Parallel exploration — on the order of 4–8 concurrent proof tasks, not hundreds; methods/assumes/hints race and collapse to consensus early.
  • Never a wrong outcome — PASS means definitively equivalent/proven; FAIL means definitively different/violated; anything else is timeout/inconclusive (not great, but OK).

1 — Proof-task DAG runner (taskflow + fork workers)

  • Vendor taskflow (header-only, MIT) via MODULE.bazel. It owns the dependency DAG and a worker pool (default 4–8, formal.jobs).
  • ONE global pool for every solver fork (ruling 2026-07-10): def-tasks, portfolio racers, and case-split cubes are all tasks in the same formal.jobs-slot pool — total concurrent cvc5 processes is bounded at formal.jobs, period. formal.partitions becomes "how many cubes to split into" (task count), no longer "how many extra processes".
  • Each solver task body FORKS a cvc5 worker process — cvc5 cross-instance thread-safety is unverified (query.cpp:125), and a child process keeps the free timeout/hard-kill we rely on for early cancellation. The taskflow thread just fork+waitpids, reusing the existing fork/pipe/serialize machinery. Sequential in-process fallback on fork failure (existing pattern).
  • DAG sources: the lec bottom-up def hierarchy (all ready leaves in flight; a parent unlocks when its children settle; a proven child becomes a collapse point) and verify's obligation set (independent except frontier-assume ordering and scheduler rounds).
  • Early consensus: the first definitive (Proven/Refuted) verdict for a subtask kills that subtask's still-running sibling branches (portfolio methods; a SAT cube in a case-split). Consensus is per-method-semantics: a case-split is Proven only when ALL cubes are UNSAT, Refuted on any SAT cube.

2 — Portfolio: parallel methods per subtask, first definitive wins

  • A subtask is one def-pair (lec) or one obligation (verify). The method ladder — cache hit → semdiff structural match (no solver) → inductive miter → BMC → case-split — races where racing pays; a definitive pass/fail from any method stops the rest.
  • Today this exists lec-only (run_auto_portfolio: ind+bmc fork race, combinational skip). Lift it into fcore so verify obligations get the same treatment; the trust asymmetries (e.g. ind-Refuted trusted only for combinational designs; REFUTE-under-collapse needs flat confirmation) stay with the consumer pages that own them.

3 — Unified canonical digest + verdict cache (--workdir)

Subsumes old lec-M3 (canonical-hash def-diff) and old 2f-verify V5 (per-obligation verdict cache) — one digest, one store, two key kinds (ruling 2026-07-10).

  • Deliverable A — canonical structural def digest: op + params + sorted connectivity + interface, stable across two independent builds of an unchanged def. This is also the module-identity digest 2i-import needs — one implementation, two consumers.
  • Deliverable B — verdict cache store under --workdir. Key kinds: (a) lec def-pair: digest(ref)+digest(impl) + the engine-identity salt — a formal-source content hash baked at build time (hash of pass/lec/, pass/semdiff/, the fcore lib, cvc5 version; ruling 2026-07-10: prover-code changes auto-invalidate with no human in the loop, unrelated lhd rebuilds keep the cache warm) + options, phase, bound — a hit needs no encode, so an unchanged submodule is instantaneous; (b) verify obligation: hash of the serialized SMT encoding of the obligation's cone + obligation term + solver identity + engine params — hashing downstream of the encoder means an encoder semantics fix auto-invalidates (soundness rule F, moved here from 2f-verify).
  • Rules (non-negotiable): cache only definitive verdicts; SAT only together with its witness, revalidated by evaluation on hit; never cache Unknown; solver identity genuinely belongs in the key (this repo already lived a solver-config soundness bug — the lazy-bitblaster spurious SAT, query.cpp:1098-1120).
  • Payoff: edit one module ⇒ only cone/def-touched proofs re-run (the external verification_cache.json harness precedent, made native + sound + fine-grained).
  • Third record kind — strategy hints (user note 2026-07-10): the cache also remembers which auto choices WON per def/obligation — winning engine, case-split selector, bound, solve time — keyed by canonical entity name, not digest, so the hint survives an edit that misses the verdict cache. On the next run the portfolio tries the known-good strategy first. Purely a heuristic ordering channel: hints can speed a proof but can never change a verdict, so they need none of the verdict-cache soundness rules.

4 — Case-split / assume-spill generalization

  • lec.partitions=N + lec.split=auto landed 2026-07-10 (auto-pick a control selector, fork disjoint cube workers, fold verdicts; ALU 18.5s→0.7s). Generalize to formal.partitions/formal.split in fcore so verify obligations can case-split on control selectors in their cones too — the "introduce assumes on an input and spill parallel sub-proofs" principle, applied to both components.
  • Soundness folding is per-method: all cubes UNSAT ⇒ Proven; any cube SAT ⇒ Refuted with that witness; any cube Unknown ⇒ inconclusive.
  • Gotchas already learned: input pins have empty pin_name_of — match by identity; use the forward cone, not backward BFS; static cone SIZE ≠ hardness.

5 — Hints: proven facts become reusable assumes

  • Producers: verify's mining loop (2f-verify), lec's winning case-split selectors and proven-def certificates, proven sidecar invariants.
  • Consumers: (a) the cache — automatic reuse next run; (b) ready-to-paste sidecar formal blocks with provenance comments, so the user can add them to the design Pyrope and make future runs faster on any machine (no workdir needed).
  • LEC hints specifically: persist the winning engine and split selector ("split on state") as §3 strategy hints (keyed by entity name, so they survive design edits); assert proven impl-side invariants in the miter (entailed facts — pure speedup, can never change the verdict).

6 — Scheduler: global budget, escalating rounds, diagnose-last (moved from 2f-verify V6)

  • Global formal.timeout owns proving time (fixes the (N+1)×tlimit-per hazard). Ruling 2026-07-10: formal.timeout is the TOTAL wall-clock budget for the whole command — a deliberate semantics change from today's per-proof-call meaning; the scheduler derives per-task slices and stragglers inherit what's left (one knob, default 120). Rounds: R0 free (comptime, cache hits, sidecar rechecks) → R1 small slices (~1-2s per pending obligation — most properties are easy) → R2+ escalation (4s → 16s → 64s…), survivors ranked by getDifficulty mass + cone size + last-round progress; stragglers sink and inherit remaining budget. Rounds map onto staged taskflow subflows.
  • formal.minetimeout — an independent budget (0 = off) for the diagnosis/mining phase: getTimeoutCoreAssuming names the toxic kernel so a timed-out run's output is the assumes/hints that make the next run succeed.
  • Determinism: the compile profile budgets by cvc5 rlimit (machine-independent); formal verify defaults to wall-clock with formal.budget_mode=rlimit for CI.

7 — Assume forms: the contract (ruling 2026-07-10)

Three forms; grammar lands in prpparse, semantics enforced here. Both _nocheck forms get a diag warning at every encounter (with the standard file:line:cmd: origin prefix).

formformal (verify/lec)synthesis (compile tier)diag
assumeover internal signals: a proof obligation; once proven, used as a constraint per the soundness discipline (2f-verify rules A–E). Over primary inputs only: an environment constraint by nature (inputs are free — there is nothing to prove it against); honored directly, and every verdict discloses it ("under N input assume(s)" — conditional equivalence/proof)used only when proven (input-only assumes: never — synthesis can't restrict its environment)—
assume_nocheck_formalfree constraint, no proof; every verdict discloses "under N unchecked assume(s)"ignored warning per encounter
assume_nocheck_synthignored entirely used unchecked (user's explicit risk)warning per encounter
  • Composition note: a LEC across a synth boundary that used assume_nocheck_synth will (correctly) refute in the excluded input space — the netlists really differ there. A user who wants the conditional LEC states the condition explicitly in a formal block — as a plain assume (checked where checkable; an input-support assume is an environment constraint by nature) or, deliberately, as assume_nocheck_formal (never checked, louder disclosure). The two are not interchangeable; the point is that formal never silently imports synth pragmas — the user restates the condition consciously.
  • assume_nocheck_synth facts never become proof hypotheses anywhere; their only consumer is the compile tier (2f-formal's EXDC item).

  • Traps carried from 2d-cex: never add probe outputs (perturbs the SAT instance into UNKNOWN); localization-only (pushing state-cut DISTINCT terms into bad can flip verdicts — flag-gate); BMC-only (the inductive CEX may be unreachable, so it carries no trace); sign-magnitude widths — compare at declared width or phantom diffs appear on the spare sign bit. (Memory-cut readback was dropped by ruling 2026-07-10 — it rides the lazy-solver getValue hazard; revisit only if it hurts in practice.)
