lec Relational equivalence engine — roadmap (pass/lec)

How and why pass/lec works today is the reference doc pass/lec/lec.md (cvc5-direct: combinational miter, inductive flop-cut + BMC-from-reset sequential, memory arrays, combinational Sub inline flattening — all landed). This page tracks the remaining work. The bottom-up hierarchical decomposition is now landed (see pass/lec/lec.md + the lec_*_test suite): stop solving the whole design as one flat miter and instead race the two engines as a portfolio (lec.engine=auto), prove each module def leaves-first, and turn every proven def into a black-box collapse point for its parents (lec.hierarchical=true). This pulls the expensive deep-state modules (e.g. the register file) and large combinational cones out of every parent proof, and — just as importantly — it fixes the correspondence-incompleteness that makes a flat whole-design proof give up (per-module the flop/IO names match exactly ⇒ complete correspondence ⇒ a real Proven; the parent then only needs instance-boundary correspondence).

Correspondence — no hard semdiff dependency

The bottom-up flow relies on name-based correspondence, which the encoder already has: I/O ports matched by name case-insensitively (Io_name_map), flops matched by canonical hierarchical name (canon_flop_name strips Yosys decoration + ___ssa_ suffixes), plus explicit --match ref=impl overrides. Whenever the two designs share the same call structure (e.g. dino's surviving registers + 14 StageReg_* instances carry identical, 1:1 names), this pairs everything with no semdiff needed.

2f-semdiff is therefore an optional accelerator, not a prerequisite: its structural MATCH only earns its keep by letting lec skip the solver on structurally-identical subtrees (the M3 def-diff reduction below). The conservative v1 is "every module in the hierarchy gets its own LEC"; semdiff is the later optimization that drops the ones it can prove identical for free.

Subtasks — later (further speedups, layered on the bottom-up flow)

  • M3 (remainder) — canonical-hash def-diff reduction. (v1 — lec.semdiff=structural per-module pass/semdiff::structural_match skip with children-proven guard — is landed; see pass/lec/lec.md.) Layer on a canonical per-def hash (op + params + sorted connectivity + interface), stable across two independent builds of an unchanged def, so structurally-identical defs are dropped with no solver call at all — a 1000-def design with a 1-def edit checks ≈1 def. This is a further speedup on top of the bottom-up flow, not a prerequisite: when the two call structures match, name-based correspondence already pairs everything, so the driver runs without semdiff; semdiff only earns its keep by letting you skip LEC on matched subtrees. Builds on 2f-semdiff (per-node MATCH) + a def-level partition of matched vs changed regions. Test: the existing hier_* fixtures with a single-def edit; assert only that def was solved.
  • M6 — persistent certificate cache. Subsumed by 2f-verify's per-obligation cone-hash verdict cache — no longer independent pending work here.
  • bitwuzla backend (opt-in). lec.solver=bitwuzla is declared but not built. Vendoring is awkward (Meson, auto-downloads CaDiCaL/SymFPU → breaks the hermetic sandbox); revisit only if cvc5's BV/array performance proves insufficient.
  • Provenance-based matching (optional hint source). Every optimization that preserves derived_from / transformation id gives the correspondence pass strong hints, so cvc5 stays the last resort, not the first. Wire once provenance attributes exist.
