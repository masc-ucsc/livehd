lec Relational equivalence engine — roadmap (pass/lec)

How and why pass/lec works today is the reference doc pass/lec/lec.md (cvc5-direct: combinational miter, inductive flop-cut + BMC-from-reset sequential, memory arrays, combinational Sub inline flattening — all landed). This page tracks the remaining work. The bottom-up hierarchical decomposition is now landed (see the section below): stop solving the whole design as one flat miter and instead race the two engines as a portfolio (lec.engine=auto), prove each module def leaves-first, and turn every proven def into a black-box collapse point for its parents (lec.hierarchical=true). This pulls the expensive deep-state modules (e.g. the register file) and large combinational cones out of every parent proof, and — just as importantly — it fixes the correspondence-incompleteness that makes a flat whole-design proof give up (per-module the flop/IO names match exactly ⇒ complete correspondence ⇒ a real Proven; the parent then only needs instance-boundary correspondence).

Correspondence — no hard semdiff dependency

The bottom-up flow relies on name-based correspondence, which the encoder already has: I/O ports matched by name case-insensitively (Io_name_map), flops matched by canonical hierarchical name (canon_flop_name strips Yosys decoration + ___ssa_ suffixes), plus explicit --match ref=impl overrides. Whenever the two designs share the same call structure (e.g. dino's surviving registers + 14 StageReg_* instances carry identical, 1:1 names), this pairs everything with no semdiff needed.

2f-semdiff is therefore an optional accelerator, not a prerequisite: its structural MATCH only earns its keep by letting lec skip the solver on structurally-identical subtrees (the M3 def-diff reduction below). The conservative v1 is "every module in the hierarchy gets its own LEC"; semdiff is the later optimization that drops the ones it can prove identical for free.

Subtasks — bottom-up hierarchical decomposition (LANDED)

These supersede the old single "M5 (remaining)" bullet, and are now all landed (see pass/lec/lec.md + the lec_*_test suite). They compose: every per-module proof and the collapsed parent proof each run under the auto portfolio, the driver wires them bottom-up, and each block emits a machine-parseable progress line the moment it resolves. The remaining items are the later speedups in the next section (M3 def-diff skip, M6 certificate cache, bitwuzla, provenance hints).

  • ✓ LANDED (new info Severity + builder, lec-block-* diags; core:diag_test, lec_progress_test). agentic progress reporting (per-block info diag). A full bottom-up run is long, so emit a machine-parseable progress line every time a block (leaf or parent) resolves to pass / fail / inconclusive — an agent driving the flow stream-parses it instead of waiting for the end. Reuse the existing diag pretty/jsonl rendering (--diag-fmt jsonl, LIVEHD_DIAG) at a new info severity that — unlike error/warning — does not touch has_errors() or the exit code (today the enum is only error/warning/note, and note is a secondary-location attachment only, core/diag.hpp:27; this adds a standalone info + an info() builder). Each record carries the block name, the verdict, and which engine reached the inference fastest: under auto, the engine that returned the trustworthy verdict first (inductive-Proven or BMC-Refuted) plus its elapsed ms; for inconclusive there is no winner, so list the attempted engines with their times/bounds. The Diagnostic struct is fixed today (no arbitrary key/value, core/diag.hpp:88-104), so this needs ~2–3 new optional fields (verdict, engine, duration_ms — or a small generic attrs map to also carry bound / witness path / node count) serialized in to_jsonl(). pass/lec already emits pass.lec diags (e.g. not-equivalent via err().fatal()), so the emit site and code/category conventions exist; add stable codes like lec-block-proven / lec-block-refuted / lec-block-inconclusive (category: a new lec/progress or reuse internal).
  • ✓ LANDED (forked ind‖bmc workers, verdict-trust asymmetry; lec_auto_test). auto — parallel portfolio engine (lec.engine=auto). Race the two real engines on the same --ref/--impl pair and take the first trustworthy verdict. (Today only bmc and the single-step inductive miter exist; ind/ic3/auto silently fall through to the inductive miter — query.cpp:122,303.) The pairing is complementary: the inductive miter is good at proving equivalence (deep state, e.g. the register file), BMC-from-reset is good at finding shallow real bugs. The core correctness content is the verdict-trust asymmetry, which the current exit-code mapping does not encode:
    • inductive Proven (UNSAT + complete correspondence) ⇒ trust as PASS, kill BMC;
    • BMC Refuted (CEX within bound, from reset) ⇒ trust as FAIL + witness, kill inductive;
    • inductive Refuted ⇒ not a real bug (single-step assumes arbitrary equal state ⇒ may be an unreachable step-case CEX; the code can't tell base- from step-case, query.cpp:1414-1440) — hint only;
    • BMC bounded-Proven (no CEX ≤ N, default 6) ⇒ not a full PASS;
    • inductive non-Proven and BMC non-Refuted ⇒ inconclusive.
    Mechanics: cvc5 cross-instance thread-safety is explicitly unverified (query.cpp:120-121), so race as separate processes (fork two lhd lec workers, first conclusive wins, kill the other) — free timeout/kill, no threading landmine. Generalizes the existing sequential phase=full two-query orchestration.
  • ✓ LANDED (--collapse / lec.collapse; hhds Hier_opaque_scope for same-library leaves; lec_collapse_test). proven-module black-box collapse (the --collapse knob). The encoder already collapses opaque Sub instances to a sound black-box — shared output symbols keyed by defname#occurrence with inputs as compare points (encode.cpp:704-765, set_shared_bbox) — but only when a Sub is unresolved/stateful/not in --lib. Add a knob (--collapse <def> / a proven-set the driver passes in) that forces that same path on for a def the driver has already proven equivalent, even when it could be flattened. For a combinational leaf this is directly sound: outputs become UF(inputs), identical on both sides, and the full free-input isolation proof discharges "equal inputs ⇒ equal outputs". Effect: the parent stops re-solving the leaf's internals.
  • ✓ LANDED (State_box UF(inputs,state)/next; shared state cut threaded across BMC; lec_seqbox_test). sequential black-box (stateful collapse points). Today's black-box models outputs as a pure function of inputs (no state) — sound only for combinational leaves. To collapse a stateful leaf (the register file, the StageReg_* set — exactly the deep-state modules that are slow and BMC-hostile), extend the box to be state-aware: outputs = UF(inputs, state), next_state = UF2(inputs, state), with the leaf's state cut corresponding on both sides and matched reset/init. Then equal inputs + equal corresponding state ⇒ equal outputs + equal next-state by congruence. Sound in the inductive (assume-equal-state) frame directly; under BMC-from-reset it additionally needs the matched-reset init so the box doesn't lose the leaf's reset behavior. This is what lets the register file be pulled out of the top proof — the single biggest win for the dino case.
  • ✓ LANDED (def DAG topo-order, leaves-first under auto, collapse proven children, CEGAR-flatten the rest; lec_hier_test. v1 sequential; parallel-leaves is a later speedup). bottom-up hierarchical driver (lec.hierarchical=true). Build the module-def dependency DAG, topo-order it, and LEC defs leaves-first, each under auto, in parallel. Record the proven set; for each parent, force-black-box its proven child instances (via the collapse knob) and LEC the parent. A child that is not provable in isolation with free inputs (context-dependent equivalence — only equal under the inputs its parent drives) is not collapsed: un-black-box it and flatten it into the parent, proving it in context (this is the M5 "CEGAR / un-black-box" fallback, now in v1). Correspondence is name-based (above) — no semdiff required when call structures match. Worked example (dino, 2 levels): round 1 proves DualIssueRegisterFile + the StageReg_* set (and the combinational defs) in parallel; round 2 proves PipelinedDualIssueCPU with the 15 stateful instances collapsed and the inlined combinational logic left flat-but-cheap. Note the asymmetry: dino_p1 inlined the 12 combinational modules, so only the stateful instances exist as collapse boundaries at the top — fine, the inlined combinational cones stay flattened.

Subtasks — later (further speedups, layered on the bottom-up flow)

  • ✓ LANDED (v1) (lec.semdiff=none|structural; the hierarchical driver runs pass/semdiff::structural_match per module first and drops a fully-matched def whose children are all proven, with no solver call; children-proven guard keeps it sound; lec_semdiff_test). The canonical-hash form below is a future refinement. M3 — structural def-diff reduction (semdiff-based skip). Once the bottom-up driver works, layer on a canonical per-def hash (op + params + sorted connectivity + interface), stable across two independent builds of an unchanged def, so structurally-identical defs are dropped with no solver call at all — a 1000-def design with a 1-def edit checks ≈1 def. This is a further speedup on top of the bottom-up flow, not a prerequisite: when the two call structures match, name-based correspondence already pairs everything, so the driver runs without semdiff; semdiff only earns its keep by letting you skip LEC on matched subtrees. Builds on 2f-semdiff (per-node MATCH) + a def-level partition of matched vs changed regions. Test: the existing hier_* fixtures with a single-def edit; assert only that def was solved.
  • M6 — persistent certificate cache. Key = canonical hash + interface hash + assumptions; value = a certificate. "Prove once, reuse forever" across runs and passes; future optimization passes consult the cache before calling cvc5. Deferred until the bottom-up flow + M3 prove out.
  • bitwuzla backend (opt-in). lec.solver=bitwuzla is declared but not built. Vendoring is awkward (Meson, auto-downloads CaDiCaL/SymFPU → breaks the hermetic sandbox); revisit only if cvc5's BV/array performance proves insufficient.
  • Provenance-based matching (optional hint source). Every optimization that preserves derived_from / transformation id gives the correspondence pass strong hints, so cvc5 stays the last resort, not the first. Wire once provenance attributes exist.

Subtasks — landed (see pass/lec/lec.md)

  • M1 — combinational equivalence. Encoder (LGraph → cvc5 BV terms) + miter + single checkSat; cross-checked vs lgcheck.
  • M2 — sequential via register correspondence. Single-step inductive flop-cut miter (the default engine when lec.engine ≠ bmc) matching flops by preserved name, plus the bmc unroll-from-reset engine (default bound 6) with reset-phase separation (free_toreset / just_reset / after_reset [default] / full) and reset auto-detection. (Note: the ind/ic3 options are not yet real k-induction/PDR — they fall through to the single-step miter.)
  • M4 — memory as SMT arrays. Corresponding memories collapse to a shared array symbol (signature + occurrence key); select/store next-state.
  • M5 (partial) — combinational Sub inline flattening via lhd lec --lib lg:DIR (e.g. ABC standard-cell netlists resolving to pass.liberty gensim models); opaque Sub → sound blackbox.
