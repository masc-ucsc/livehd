lec Relational equivalence engine — roadmap (pass/lec)

How and why pass/lec works today is the reference doc pass/lec/lec.md (cvc5-direct: combinational miter, inductive flop-cut + BMC-from-reset sequential, memory arrays, combinational Sub inline flattening — all landed). This page tracks the remaining work: the "fast incremental" reduction and the sequential/cross-run scaling that turn it from a whole-design checker into a change-scoped one.

Depends on

2f-semdiff — the structural MATCH pre-pass. The LEC reduction plan is "create the PARTITION and MATCH first": semdiff marks corresponding structure across --ref/--impl, so lec can assume the matched region equal and hand only the unmatched gaps to cvc5.

Subtasks — pending

  • M3 — structural def-diff reduction (the "fast incremental" headline). Canonical per-def hash (op + params + sorted connectivity + interface), stable across two independent builds of an unchanged def. Compare the --ref and --impl libraries def-by-def; identical defs are dropped with no solver call; only changed defs (and parents whose child interface changed) become queries — a 1000-def design with a 1-def edit checks ≈1 def. Builds directly on 2f-semdiff (per-node MATCH) plus a def-level PARTITION of matched vs changed regions. Test: the existing hier_* fixtures with a single-def edit; assert only that def was solved.
  • M5 (remaining) — sequential/black-box assume-guarantee + CEGAR. Combinational Sub inline flattening already landed (see lec.md). The remaining piece: prove a changed leaf def equivalent in isolation with free inputs (conservative ⇒ always sound; a reachable-input-only proof is unsound for congruence), then black-box it in the parent (inputs become compare points, outputs free source points on both sides) and conclude the parent equivalent without re-proof. Auto-checked soundness side-conditions (identical interface + timing, no new comb path across the boundary, discharged environment assumptions, full I/O equivalence) else flatten the region. On a spurious CEX from an isolated/abstracted proof, expand the cone / un-black-box / re-add the cut-point and retry (CEGAR).
  • M6 — persistent certificate cache. Key = canonical hash + interface hash + assumptions; value = a certificate. "Prove once, reuse forever" across runs and passes; future optimization passes consult the cache before calling cvc5. Deferred until M3/M5 prove out.
  • bitwuzla backend (opt-in). lec.solver=bitwuzla is declared but not built. Vendoring is awkward (Meson, auto-downloads CaDiCaL/SymFPU → breaks the hermetic sandbox); revisit only if cvc5's BV/array performance proves insufficient on M3/M5 proofs.
  • Provenance-based matching (optional hint source). Every optimization that preserves derived_from / transformation id gives the MATCH pass strong correspondence hints, so cvc5 stays the last resort, not the first. Wire once provenance attributes exist.

Subtasks — landed (see pass/lec/lec.md)

  • M1 — combinational equivalence. Encoder (LGraph → cvc5 BV terms) + miter + single checkSat; cross-checked vs lgcheck.
  • M2 — sequential via register correspondence. Inductive flop-cut miter (default ind) matching flops by preserved name, plus the bmc unroll-from-reset engine with reset-phase separation (free/reset/run) and reset auto-detection.
  • M4 — memory as SMT arrays. Corresponding memories collapse to a shared array symbol (signature + occurrence key); select/store next-state.
  • M5 (partial) — combinational Sub inline flattening via lhd lec --lib lg:DIR (e.g. ABC standard-cell netlists resolving to pass.liberty gensim models); opaque Sub → sound blackbox.
