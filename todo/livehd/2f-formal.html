2f-formal Compile-time formal tier — pending items (re-based on 2f-verify)

The original 2f-formal engine landed 2026-06-20: the pass/formal Prover (demand-driven cone encoder, deterministic cone-scaled rlimit), the always-on pass.formal pipeline pass, the Hotmux one-hot hard check, assert/assert_always/ assume end-to-end (fproperty materialization, proven ⇒ cgen elision, else translate_off runtime check + deferral warning), and the pass.abc proven-assume flags. Reference: pass/formal/prove.{hpp,cpp}, pass_formal.cpp, and the memory of the landed design decisions in git history of this page.

What this page is now: only the pending remainder — and its shape changed, because 2f-verify supersedes the engine underneath it.

Depends on

2f-verify — do 2f-verify first. Its V7 milestone re-bases pass.formal onto the shared incremental engine as the don't-try-hard profile: same code path as lhd formal verify, restricted options, small deterministic rlimit budget, timeout is not an error (just "not proven, keep the runtime check"), proven results feed synthesis as today. The items below are specified against that engine, not the current self-contained single-frame Prover — in particular the planned encoder de-dup (Prover's private cone encoder vs pass/lec's) is subsumed by the V7 rebase rather than done as a separate refactor.

Pending

  • Stateful asserts via the verdict ladder (old M6). Today a stateful refutation is kept as a runtime check because the single-frame witness may be unreachable. On the 2f-verify engine this becomes: 1-induction proven ⇒ elide; BMC-from-reset refuted ⇒ a reachable counterexample even in the compile profile's small budget ⇒ eligible for hard error (policy below); otherwise keep + defer. Tests: an invariant over registered logic proven & removed; a stateful assert whose only violation is unreachable stays (no false error); a genuinely reachable stateful violation within the compile budget errors with the witness cycle.
  • Reset-exclusion for user asserts (Reset_policy::ExcludeConeActive) — prerequisite for hard-erroring them. A normal assert may legitimately fail while reset is asserted, so today only the Hotmux structural check hard-errors; refuted user asserts merely defer. On the 2f-verify engine the after_reset phase machinery (reset detection + driven prologue, query.cpp:1201-1291) provides this for free in BMC mode; the single-frame induction rung still needs the cone-reset guard (collect each cone Flop's reset_pin/negreset, constrain inactive). assert_always keeps Reset_policy::Include. Once in place, enable the landed asymmetric policy's error row for user asserts: hard-error only on a definitive refutation that is reset-clean and (for the induction rung) stateless or reachability-confirmed.
  • ABC EXDC don't-care network. The flags (pass.abc.use_proven_assume, default true; use_all_assume, default false) and per-region constraint gathering landed (abc_map.cpp:196-227); building the EXDC network that actually hands the constraints to ABC's mfs/fraig is the remaining step. With 2f-verify, the constraint pool grows: sidecar invariants proven unbounded are don't-care sources too.

Policy carried forward (unchanged)

  • Asymmetric soundness: hard-error only on a definitive refutation that cannot be an artifact (stateless cone, or reachable-by-BMC witness, after assumes and the reset guard). Budget / Unknown / possibly-unreachable never error — a budget mistune can never turn a good build into a false error.
  • Only proven assumes are hypotheses (no circular self-proof); the consistency check runs before any removal; contradictory assume set ⇒ warning + removals suppressed.
  • Deferred-to-runtime ⇒ silenceable per-category warning (formal-assert-deferred, formal-onehot-deferred, …).

Future / deferred (other pages)

  • pass/memory port-sharing & banking — its own page; calls the Prover primitives (equal/bits_equal/ is_constant) with its own conservative fallback.
  • Runtime fallback execution — running runtime_check-marked asserts in sim/slop (4b).
  • Proven-certificate cache — subsumed by 2f-verify's per-obligation cone-hash verdict cache (its L2 layer); compile-profile runs get cache hits from prior lhd formal verify runs for free.
