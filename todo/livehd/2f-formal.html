Everything below stays 2f: the 2026-07-10 ruling kept stateful asserts, reset-exclusion, and ABC EXDC at 2f (its fourth item, memory templates, lives on 2f-verify); the engine rebase moved here from old 2f-verify V7.

Pending

  • Rebase pass.formal onto the shared engine as the don't-try-hard profile (was 2f-verify's V7, formal half): same code path as lhd formal verify, restricted options, small deterministic rlimit budget (machine-independent), timeout is not an error (just "not proven, keep the runtime check"), proven results feed synthesis as today. This replaces the self-contained single-frame Prover for property proving and is the step the two items below assume.
  • Stateful asserts via the verdict ladder (old M6). Today a stateful refutation is kept as a runtime check because the single-frame witness may be unreachable. On the shared engine: 1-induction proven ⇒ elide; BMC-from-reset refuted ⇒ a reachable counterexample even in the compile profile's small budget ⇒ eligible for hard error; otherwise keep + defer. Tests: an invariant over registered logic proven & removed; a stateful assert whose only violation is unreachable stays (no false error); a genuinely reachable stateful violation within the compile budget errors with the witness cycle.
  • Reset-exclusion for user asserts (Reset_policy::ExcludeConeActive) — quality. The hard-error gate is live with no reset guard (2026-07-08 ruling), so an assert whose only counterexample lies in the reset prologue produces a FALSE refutation ⇒ a spurious hard error. Reuse (do not duplicate) the query.cpp reset detection — the rebase gives the BMC rung the full after_reset machinery for free; only the single-frame induction rung needs the cone-reset guard (collect each cone Flop's reset_pin/negreset, constrain inactive). assert_always keeps Reset_policy::Include.
  • ABC EXDC don't-care network. Hand the constraints already gathered per region (abc_map.cpp:196-227) to ABC's mfs/fraig as an EXDC network. Constraint sources now: (a) proven asserts/assumes, (b) sidecar invariants proven unbounded by the verify engine, (c) assume_nocheck_synth facts — the compile tier is the only consumer of that form (fcore contract; per-encounter diag warning; the user explicitly owns the risk).

Policy carried forward (unchanged)

  • Asymmetric soundness: hard-error only on a definitive refutation that cannot be an artifact (stateless cone, or reachable-by-BMC witness, after assumes). Budget / Unknown / possibly-unreachable never error — a budget mistune can never turn a good build into a false error.
  • Only proven assumes are hypotheses (no circular self-proof); the consistency check runs before any removal; contradictory assume set ⇒ warning + removals suppressed. assume_nocheck_synth facts are never proof hypotheses — they are synthesis-only don't-cares by explicit user fiat; assume_nocheck_formal is ignored by this tier entirely.
