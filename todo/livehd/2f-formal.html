What this page is now: only the pending remainder — and its shape changed, because 2f-verify supersedes the engine underneath it.

Depends on

2f-verify — do 2f-verify first. Its V7 milestone re-bases pass.formal onto the shared incremental engine as the don't-try-hard profile: same code path as lhd formal verify, restricted options, small deterministic rlimit budget, timeout is not an error (just "not proven, keep the runtime check"), proven results feed synthesis as today. The items below are specified against that engine, not the current self-contained single-frame Prover.

Pending

  • Stateful asserts via the verdict ladder (old M6). Today a stateful refutation is kept as a runtime check because the single-frame witness may be unreachable. On the 2f-verify engine this becomes: 1-induction proven ⇒ elide; BMC-from-reset refuted ⇒ a reachable counterexample even in the compile profile's small budget ⇒ eligible for hard error (policy below); otherwise keep + defer. Tests: an invariant over registered logic proven & removed; a stateful assert whose only violation is unreachable stays (no false error); a genuinely reachable stateful violation within the compile budget errors with the witness cycle.
  • Reset-exclusion for user asserts (Reset_policy::ExcludeConeActive) — quality improvement, no longer a prerequisite. The gate is already live: a definitive root-module refutation hard-errors with no reset guard (pass_formal.cpp report_refuted, landed 2026-07-08 by explicit ruling), and the reset-aware escape hatch is moving the property into a formal block, where the 2f-verify BMC engine already carries the full after_reset machinery (structural reset_pin/negreset detection + name-polarity inference + driven reset-hold prologue, query.cpp:1197-1291 and :2871-2951). The compile-tier single-frame Prover has none of it, so an assert whose only counterexample lies in the reset prologue produces a FALSE refutation ⇒ a spurious hard error. Adding reset-exclusion there cuts those — fewer spurious errors, and fewer properties pushed into formal blocks just to get reset awareness. Implementation: reuse (do not duplicate) the query.cpp reset detection, which the V7 re-base makes natural — on the shared engine the BMC rung gets it for free; only the single-frame induction rung needs the cone-reset guard (collect each cone Flop's reset_pin/negreset, constrain inactive). assert_always keeps Reset_policy::Include.
  • ABC EXDC don't-care network. Building the EXDC network that actually hands the constraints (already gathered per region in abc_map.cpp:196-227) to ABC's mfs/fraig is the remaining step. With 2f-verify, the constraint pool grows: sidecar invariants proven unbounded are don't-care sources too.

Policy carried forward (unchanged)

  • Asymmetric soundness: hard-error only on a definitive refutation that cannot be an artifact (stateless cone, or reachable-by-BMC witness, after assumes; per the 2026-07-08 ruling the compile-tier gate runs with no reset guard — reset-exclusion above is the quality follow-up, and the reset-aware path is a formal block on the 2f-verify engine). Budget / Unknown / possibly-unreachable never error — a budget mistune can never turn a good build into a false error.
  • Only proven assumes are hypotheses (no circular self-proof); the consistency check runs before any removal; contradictory assume set ⇒ warning + removals suppressed.
