Scope ruling (user, 2026-07-20): SIMPLE LATCHES, NO TIME BORROWING

Time borrowing is a backend / physical-design technique. LiveHD's scope ends at the ABC call for synthesis, and ABC's output netlist has no clock borrowing either. This ruling collapses the expensive half of the problem. Under it a latch is exactly a flop-with-enable that commits at the CLOSING edge of its transparent window:

  • transparent-HIGH latch (transparent while gate == 1) → commits at the FALL of its gate
  • transparent-LOW latch (transparent while gate == 0) → commits at the RISE of its gate

That is Conformal's set flatten model -latch_fold / the IBM LEC normalization (US7624363). It is an ABSTRACTION WITH A PRECONDITION, not a semantics — and the scope ruling above is that precondition, so it must be CHECKED, never assumed (M3). What the ruling buys: no transparency mux in the sim scheduler, no settle/fixpoint loop, no MDL phase abstraction, no counterexample lifting — i.e. nothing research-grade on the critical path.

The abstraction also changes OBSERVABILITY, and that must be stated, not hidden: while a window is OPEN, a real latch reads through (Q follows D combinationally) but the commit-class model shows the previous commit. The two agree exactly at closed-window, end-of-cycle observations. Three consequences run through everything below: (1) the latch encoding in LEC is sound only as a both-sides normalization (like the ICG fold — never encode one side with transparency and the other without); (2) sim asserts and differential oracles may observe a latch Q only under the observation-visibility rule; (3) the shared observation point for sim / BMC / Icarus is end-of-cycle, all edges settled — pinned executable by tests/fixme/flop_verify_posneg.prp (under the alternative "between rise and fall" convention that fixture's correct answer would be indistinguishable from today's clock-blind encoding).

Master/slave pairs still work without modeling transparency. The two latches sit on OPPOSITE phases, so they are never simultaneously transparent; the master commits at one edge and the slave at the other, which reconstructs one-cycle flop behaviour out of commit classes alone. Tests: latch_master_slave (equiv, live) and latch_sim_master_slave (sim latency, fixme → M5).

The abstraction is for SIM and LEC ONLY. Verilog emission must still emit a real always_latch — the netlist handed to ABC has to contain a real latch. Same for pass/abc: a Latch is a region BOUNDARY (like a Flop) — but note that TODAY it is not (abc_map.cpp misses it, see M2). Terminology trap: an AIGER/ABC "latch" is an edge-triggered unit-delay register on an implicit global clock, NOT a level-sensitive latch — ABC's BLIF reader silently discards the .latch control tokens, so a real latch reaching ABC is a silent mismodel, not an error. Everything on this page means level-sensitive.

Clock blindness — a live soundness bug, independent of latches

pass/lec/encode.cpp never reads clock_pin or posclk — one grep hit, :679, and it is the sink classifier DISCARDING them ("config / clock: abstracted out of the relational encoding"); the Flop next-state build (:1969-2100) resolves only din/enable/reset/negreset/initial. Proven empirically: today's LEC falsely PROVES (a) a gated-clock flop equivalent to an ungated one, (b) a negedge flop equivalent to a posedge one, (c) a two-clock-domain pipeline equivalent to a single-clock one — in 17 ms each. BMC shares the same encoder (query.cpp:5419), so lhd formal verify inherits the blindness. The same blindness exists in sim (see State of play).

And there is NO in-repo formal oracle for flop edges: the lgyosys/lgcheck path is exactly as edge- and gate-blind for flops (reproduced: PROVEN on negedge-vs-posedge and gated-vs-ungated with the edges textually different in the compared Verilog). Its discriminating power is latch ENABLE POLARITY only. Consequence for the cross-model gating rule: for latch-polarity work gate against lgyosys; for anything involving flop clocks or edges the independent oracle must be event-driven simulation (iverilog) until M4 exists — and an equiv pair of mixed-edge designs "proving" via lgcheck is vacuous with respect to the edges.

This is why latches and multiple clocks belong in ONE roadmap: an ICG is latch + gated clock, so latch support that ignores the gate ships the most dangerous bug in the area. Expect M4 to CHANGE existing verdicts — correctly. Re-baselining minion is part of the milestone.

Rules

Cross-model gating is mandatory. "PROVEN before AND after the transformation" cannot see a transformation that is WRONG but applied identically to both sides — two identically-wrong machines prove equal, so a symmetric gate is vacuous by construction. Every latch/clock transformation must be gated against an INDEPENDENT checker — lgyosys for latch polarity, iverilog for anything clock/edge-shaped (see above), never our own encoder on both sides. This one mechanism catches mux placement, posclk double-negation, a flavour-blind ICG fold, and an unstable phase gate.

Verdict discipline (inherited from 2f-lec): PASS ⇒ definitively equivalent; FAIL ⇒ definitively different; anything else is inconclusive. An abstraction may only cause extra work, never a wrong verdict. Trackers that demand a refutation must grep a verbatim REFUTED — an encoder that merely degrades a false-PROVEN to UNKNOWN has not fixed the bug (lec_clock_blindness_test.sh enforces this since 2026-07-20).

Observation-visibility rule (used by every sim/BMC fixture; enforced mechanically in M5): an observation of a latch Q — a settled-region assert, a differential-oracle sample, or a sequential element in Q's fanin cone at its firing boundary — is valid iff (a) the latch was OPAQUE in the tick immediately preceding the observation, (b) at least one closing edge has completed since power-on (before the first commit, real hardware reads X while the model reads initial — latches have no reset until M7, so this window always exists), and (c) the data was STABLE across the closing edge being observed (a D change racing the gate close is model-dependent). Under (a)-(c) the real-transparency semantics (Icarus), the commit-at-closing-edge model, and a flop-with-enable encoding all agree, so fixtures pin the RULING, not an implementation choice. Divergence is not confined to transparent ticks — it PERSISTS through any state element that sampled a latch while open (taint), which is what clause (a)'s "in the observer's cone" phrasing covers.
