Scope ruling (user, 2026-07-20): SIMPLE LATCHES, NO TIME BORROWING

Time borrowing is a backend / physical-design technique. LiveHD's scope ends at the ABC call for synthesis, and ABC's output netlist has no clock borrowing either. This ruling collapses the expensive half of the problem. Under it a latch is exactly a flop-with-enable that commits at the CLOSING edge of its transparent window:

  • transparent-HIGH latch (transparent while gate == 1) → commits at the FALL of its gate
  • transparent-LOW latch (transparent while gate == 0) → commits at the RISE of its gate

That is Conformal's set flatten model -latch_fold / the IBM LEC normalization (US7624363). It is an ABSTRACTION WITH A PRECONDITION, not a semantics — and the scope ruling above is that precondition, so it must be CHECKED, never assumed (M3). What the ruling buys: no transparency mux in the sim scheduler, no settle/fixpoint loop, no MDL phase abstraction, no counterexample lifting — i.e. nothing research-grade on the critical path.

Master/slave pairs still work without modeling transparency. The two latches sit on OPPOSITE phases, so they are never simultaneously transparent; the master commits at one edge and the slave at the other, which reconstructs one-cycle flop behaviour out of commit classes alone. Test: latch_master_slave.

The abstraction is for SIM and LEC ONLY. Verilog emission must still emit a real always_latch — the netlist handed to ABC has to contain a real latch. Same for pass/abc: a Latch is a region BOUNDARY (like a Flop), never handed to ABC as a register. Terminology trap: an AIGER/ABC "latch" is an edge-triggered unit-delay register on an implicit global clock, NOT a level-sensitive latch — ABC's BLIF reader silently discards the .latch control tokens, so a real latch reaching ABC is a silent mismodel, not an error. Everything on this page means level-sensitive.

  • LEC via lgyosys — --set formal.solver=lgyosys proves latch-only AND mixed designs discriminatingly today (proves equivalent pairs, refutes enable-polarity flips). Broken:

  • Simulation emits NOTHING for a Latch. is_type_flop excludes Latch (graph/node_util.hpp:123-126) so it never joins the flops vector, and is_type_register includes it so the comb walk skips it (cgen_sim.cpp:2128) — its q is never bound and every latch design dies with a misleading combinational-loop error naming an internal node.
  • Gates are VACUOUS. An encoder refusal returns status:pass / exit 0 unless formal.strict=true, so every downstream milestone gate would be meaningless until M0.
  • The Latch cell is a 3-pin skeleton — din(3), enable(4), posclk(6) only (graph/cell.cpp:261-268). No clock_pin, no reset_pin/negreset/async, no initial. It cannot record which clock it belongs to (blocks the multi-clock half) nor express async set/reset.
  • yosys importer — $dlatchsr silently matches the $dlatch branch (7-char strncmp, lgyosys_tolg.cpp:2159) and DROPS its SET/CLR; $adlatch aborts.
  • The three "fixme latch" tests were stale-labelled: the Pyrope one passes with the latch=true spelling that postdates it; the two yosys ones fail for a non-LRM Verilog reason unrelated to latches.

Useful accident: tolg already bakes din = cond ? d : q and enable = cond, so the IR shape composes with the chosen encoding for free — substituting a hold state for that q gives ITE(en, ITE(en,d,H), H) = ITE(en,d,H). No tolg rewrite needed.

Clock blindness — a live soundness bug, independent of latches

pass/lec/encode.cpp never reads clock_pin or posclk. Proven empirically during recon: today's LEC falsely PROVES (a) a gated-clock flop equivalent to an ungated one, (b) a negedge flop equivalent to a posedge one, and (c) a two-clock-domain pipeline equivalent to a single-clock one. The same blindness exists in sim: a gated clock is computed as dead code and the flop advances every cycle anyway, and negedge flops simulate identically to posedge (posclk only selects a VCD slot, cgen_sim.cpp:1245/1513).

This is why latches and multiple clocks belong in ONE roadmap: an ICG is latch + gated clock, so latch support that ignores the gate ships the most dangerous bug in the area. Expect M4 to CHANGE existing verdicts — correctly. Re-baselining minion is part of the milestone.

Rules

Cross-model gating is mandatory. "PROVEN before AND after the transformation" cannot see a transformation that is WRONG but applied identically to both sides — two identically-wrong machines prove equal, so a symmetric gate is vacuous by construction. Every latch/clock transformation must be gated against an INDEPENDENT checker (lgyosys / iverilog), never our own encoder on both sides. This one mechanism catches mux placement, posclk double-negation, a flavour-blind ICG fold, and an unstable phase gate.

Verdict discipline (inherited from 2f-lec): PASS ⇒ definitively equivalent; FAIL ⇒ definitively different; anything else is inconclusive. An abstraction may only cause extra work, never a wrong verdict.

  • M3 — shared commit-class classifier + THE CONTRACT CHECK [deps M2]. ONE analysis, three consumers (sim, LEC, timing). For every state element compute its commit class (clock net, edge): posedge flop → (clk, RISE); negedge flop → (clk, FALL); transparent-HIGH latch → (gate, FALL); transparent-LOW latch → (gate, RISE).
    The contract check enforces the scope ruling and fails closed with a directed diagnostic naming the offending path:
    1. no combinational path between two latches that are SIMULTANEOUSLY transparent (same gate net, same polarity);
    2. no combinational path from a latch's Q back to its own D (self-timed / time-borrowing);
    3. a latch's gate must be a recognizable clock-phase expression.
    A violation is a named error — never a silent flop-with-enable approximation.
    Gate: master/slave accepted and classified onto opposite edges; a synthetic time-borrowing design REJECTED with the directed diagnostic.
  • M4 — LEC: clock awareness + latch encoding [deps M3, parallel with M5]. First close the clock-blindness holes (a standalone soundness fix): make the flop encoder read clock_pin/posclk and fold the commit class into the canon_flop_name keying. Then encode a latch as its commit-class flop-with-enable. Apply the ICG fold as a BOTH-SIDES normalization (abstract the ICG into a clock-enable on the driven flops — CIRCT's arc.state has a native enable operand for exactly this), never as a one-sided abstraction. Nobody proves ICGs in industry practice; they normalize them away.
    Gate: the three false-PROVEN reproducers (gated vs ungated, negedge vs posedge, 2-clock vs 1-clock) now REFUTE; latch fixtures PROVEN against lgyosys as the independent oracle; minion full-design LEC re-baselined. Promotes latch_icg and (with M5) latch_mixed_flop.
  • M5 — simulation: the three-region tick [deps M3, parallel with M4]. A tick is THREE REGIONS, not a sequence of per-edge calls: (1) evaluate every firing edge's next-state from PRE-TICK state, (2) commit all firing state, (3) evaluate the newly-opened region. Per-edge sequential calls are a silent off-by-one whenever two clocks toggle in the same tick — with coincident edges IEEE 1800 says both NBAs sample pre-edge values, so sequencing collapses a 2-cycle pipeline to 1, and no call order is correct in general (reverse it and the mirror design breaks). This is the act/nba region split Verilator implements. Latches join a unified state vector with their commit class. Honor clock_pin — a gated clock is silently ignored today, and since the ICG is the canonical latch application, shipping without this is a trap. Per-(clock, edge) schedules with per-region liveness, so a design pays only for the edges it actually uses.
    OPEN, settle BEFORE the cgen_sim restructure: hierarchical per-edge advance needs a mapping from the actual clock net at each instantiation site to the child's FORMAL clock port; today submodules advance atomically once, inline (cgen_sim.cpp:1964). None of the drafted plans budgeted this.
    The simeq harness is NOT a neutral oracle here: README_simeq.md documents that its golden Icarus testbench samples after BOTH clock edges specifically so negedge flops do not mismatch, justified by asserting posclk is a scheduler no-op — exactly the property this milestone removes. It will go red from a 100%-green baseline (_SIMEQ_FIXME = []). Revise the sampling convention, re-bake every golden signature via gen_equiv_tb.py --setup, and land the convention change and the negedge fix in the SAME commit. Use inou/cgen/tests/sim/diff_test.py (per-bit X-masked vs Icarus) as the latch oracle — a never-enabled latch is legitimately X, which a 1000-cycle signature hash cannot express.
    Gate: promotes latch_master_slave (one-cycle latency, not two) and latch_mixed_flop.
  • M6 — multiple clocks as the generalization of phases [deps M5]. Per-(clock, edge) schedules generalize to N clocks. Extend the tick grammar — prp_sim.cpp:1441-1462 hard-rejects >1 clock today, and lhd_sim_clock_reset_test.sh asserts that rejection AS CORRECT, so it needs rewriting here. Design sketch already exists in todo_multiclock.md (split per clock EDGE; a clock used pos+neg yields two subgraphs; a new cell for cross-domain communication).
  • M7 — async/SR latches + efficiency [deps M4, M5]. yosys importer: fix the $dlatchsr 7-char strncmp SET/CLR drop and add $adlatch (which aborts today). Efficiency, ranked by (payoff, cost): (a) kill the step()+peek() double cycle() call — >1.5×, available TODAY, entirely latch-independent; (b) raise the host compile above -O1 (lhd_kernel_sim.cpp:351); (c) per-region liveness (M5 gives this); (d) activity gating (ESSENT-style, 1.5–11.5× over Verilator on the measured ~4.6% activity factors) — greenfield, no framework exists to plug into.
