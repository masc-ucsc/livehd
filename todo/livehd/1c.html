1c Function coverage for uncalled LiveHD C++ code

Use coverage reports to find LiveHD-owned C++ functions that are never called, then add focused tests that exercise real flows before deciding whether any function should be deprecated. This task is about function coverage, not line coverage: a target is only addressed when a previously uncalled function becomes called by a self-checking test.

Scope

  • In scope: LiveHD C++ implementation and header files under this repository, including readers, passes, LGraph/LNAST/upass code, and lhd command plumbing.
  • Out of scope for fixing: generated code, external dependencies, third-party samples, and non-C++ files. If coverage tooling exposes unused LiveHD Python, Bash, Pyrope, or Verilog files, list them at the end of the task notes as potentially problematic for a human decision.
  • Immutable: contract tests and benchmarks remain read-only. If a contract fails, fix implementation or add a separate focused test.

Coverage workflow

  1. Generate or refresh coverage with @scripts/getcov.sh. This is slow enough that it should establish the initial function inventory and final confirmation, not be the inner loop for every candidate.
  2. Build an inventory of uncalled functions from LiveHD C++ files only. Group by subsystem and by likely test entry point: inou/prp/tests, inou/slang/tests, inou/yosys/tests, or lhd/tests.
  3. Before targeting one uncovered method, add a temporary logging/print probe to that function, then create or adjust a test. Iterate with focused bazel test targets or bazel test //...; this is much faster than rerunning coverage after every attempt.
  4. Remove the temporary probe before landing the test. Do not keep a test that does not trigger the targeted new function coverage.
  5. After a batch of tests lands locally, rerun coverage with @scripts/getcov.sh and update the inventory: covered, still uncovered, or candidate for deprecation.

Test strategy

  • Pyrope flow tests. Prefer small inou/prp/tests cases when the uncalled function belongs to parsing, LNAST construction, upass, compile-time evaluation, LGraph lowering, or generated-Verilog behavior. Each test must self-check correctness, following the shape of existing comptime and equiv tests rather than merely existing for coverage.
  • Verilog flow tests. Use inou/slang/tests for direct --reader slang behavior and inou/yosys/tests for --reader yosys-verilog / Yosys-driven behavior. The source should be narrow enough to explain which importer, pass, or codegen path it exercises, and the test must check an observable result.
  • lhd command tests. Some functions, such as graphviz or option-specific pass entry points, should not be triggered by ordinary Pyrope/Verilog regression tests. For these, add focused wrappers under lhd/tests, such as graphviz_check1.sh or color_synth1.py. The wrapper should call a concrete command like lhd elaborate some_file --set some.flag=value --emit-dir xxx, then check the pass-specific output in xxx.
  • Check only the intended result. A wrapper does not need full semantic proof of every generated file. It must check the specific observable that proves the option or pass path worked: expected file presence, expected graphviz marker, expected JSON/diagnostic entry, expected generated module, or expected pass-specific transform.
  • No package dependency creep. Python wrappers in lhd/tests should use only the standard library unless a local test helper already provides the needed behavior.

Subtasks - pending

  • A - function-coverage inventory. Run @scripts/getcov.sh, extract the never-called function list, filter to LiveHD C++ files, and classify by subsystem, likely entry point, and risk.
  • B - flow-test candidates. For uncovered frontend/compiler functions, create focused Pyrope or Verilog tests that drive the normal compile/elaborate/check pipeline and self-check correctness.
  • C - CLI/pass-option candidates. For uncovered functions that only make sense behind an lhd command option, add lhd/tests shell or Python wrappers that invoke the command and validate the pass-specific output.
  • D - coverage-preserving cleanup. Remove every temporary logging/print probe, discard tests that do not add function coverage, and rerun the relevant bazel test targets before final coverage confirmation.
  • E - deprecation queue. For functions that remain uncalled after reasonable test attempts, document why no normal flow should reach them and whether they should be deleted, hidden behind a feature flag, or kept as an intentional public/API hook.
  • F - non-C++ appendix. If the coverage audit exposes unused LiveHD-owned Python, Bash, Pyrope, or Verilog files, list them separately as potentially problematic. Do not make those files the main work of this task.

Acceptance criteria

  • Each landed test is self-checking and has a clear reason tied to one or more formerly uncalled LiveHD C++ functions.
  • No temporary logging/print probes remain in source code or tests.
  • bazel test //... passes after the added tests.
  • A final coverage run with @scripts/getcov.sh shows the expected function-coverage movement, and the remaining uncovered function list is triaged instead of silently ignored.
