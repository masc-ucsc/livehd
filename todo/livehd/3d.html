3d inou.cgen.sim: LGraph → executable slop

A new LGraph code generator inou.cgen.sim — a second emit target on the inou.cgen plugin, beside inou.cgen.verilog and sharing its graph-walking machinery — that lowers each compiled hhds::Graph to C++ which links the ../hlop Slop<N> value type, driven by a new --emit-dir sim:DIR/. The emitted directory is a self-contained Bazel module: cd DIR && bazel build //:sim compiles the design into a simulation cc_library.

Design pivot — read the LGraph, not the LNAST

This task was originally filed as a upass lnast_to_slop running parallel to lnast_to_lgraph (upass/tolg), walking the post-upass LNAST. It is re-scoped to a graph pass over the lowered LGraph because the LGraph is a strict subset of the LNAST and has already solved the three hard problems a slop generator would otherwise re-derive:

  • bits / sign are resolved per pin. Every driver pin carries livehd::attrs::bits (bits_of(pin)) and pin_signed (is_unsign(pin)), so a value lowers directly to Slop<N> — no bitwidth inference in the generator.
  • pipe / comb are gone. The Pyrope lambda kinds (comb/pipe/mod) are an LNAST concept; by the LGraph stage there are only mod (modules built of nodes + flops + Sub instances). A comb was inlined; a pipe output is a concrete Flop whose depth rides the pipe_min sink pin (a shift-register depth, default 1 — the same thing cgen_verilog reads). The generator keeps whatever the flops say; it does not interpret a latency range.
  • the op set maps 1:1 onto Slop. The Ntype_op dispatch is the same one cgen_verilog walks, and almost every op has a named Slop method (table below).

Latency selection is a separate, future pass (see subtask F): an LGraph→LGraph pass that picks a concrete depth within a declared [pipe_min, pipe_max] range (seed-driven) and rewrites the flops before inou.cgen.sim runs. Until it exists, inou.cgen.sim stays latency-agnostic.

Code sharing with inou.cgen.verilog

The two generators overlap on structure, not on emission. Share the first, keep the second per-target:

  • Already shared (free). The semantic helpers live in graph/node_util.hpp (livehd::graph_util::*): bits_of, is_unsign, is_type_flop/is_type_register/is_type_sub, type_op_of, hydrate_const, const_value_of, wire_name, pin_name_of, is_graph_input_pin/output_pin. Cgen_sim just using-imports them, exactly as cgen_verilog.cpp does — bits/sign/type-classification cost nothing to reuse.
  • Promote a few cgen-local statics into node_util (where their siblings already live) or a small inou/cgen/cgen_common.hpp: pin_wire_name, get_scaped_name/get_unique_decl_name, find_sink_pin, get_driver, sra_operand_signed, and the create_module_io / create_subs / create_registers / create_memories / create_combinational traversal skeleton.
  • Keep per-target (do NOT force-share). The expression model diverges: cgen_verilog inlines parenthesized expressions (Expr/pin2expr, needs_parenthesis) and emits always blocks + SV reserved_keyword escaping; cgen_sim emits flat SSA bindings (auto v = a.add_op(b);) + a functional cycle(). A backend-parameterized do_from_graph would couple two unlike emitters for no gain — each owns its process_* / build_* methods.

Per-module C++ shape (functional cycle(in)→out)

One struct per module graph. Registers (Flop/ Latch) and submodule (Sub) instances are members; inputs are cycle() arguments, outputs a returned struct. The body computes each flop's next value into a local, drives the outputs from the current state, then commits the flops on the way out (Slop is a value type, so staging in locals is trivial):

// graph "Counter": input en:u1, output count_o:u8, one u8 flop
struct Counter {
  Slop<8> count_q;                          // Flop state (member)

  struct In  { Slop<1> en; };
  struct Out { Slop<8> count_o; };

  void reset_cycle() { count_q = Slop<8>::create_integer(0); }   // reset value from the Flop's reset_pin

  Out cycle(In in) {
    auto count_d = in.en.is_known_true()
                 ? count_q.add_op(Slop<8>::create_integer(1))
                 : count_q;
    Out o{ .count_o = count_q };           // comb output from current Q
    count_q = count_d;                     // commit flop at the edge
    return o;
  }
};

Each Slop<N> width N comes from bits_of(pin); signedness from is_unsign(pin). (Slop is always signed and is a superset of unsigned, so unsigned pins lower as the same type — masking only where a width boundary needs it, via get_mask_op.)

Subtasks — pending

  • A — inou.cgen.sim skeleton + emit wiring. The sim emit-kind, the emits_need_graphs gate, the emit_sim_outputs/sim_into hook, the pass registration, and the combinational lowering (the op table above: walk forward_class(), name each driver pin, emit one auto binding per node). Plus the standalone-module writer (MODULE.bazel + BUILD + per-module .hpp + manifest). Milestone 1 success = cd DIR && bazel build //:sim compiles for the combinational + single-flop designs. No driver, no LEC yet.
  • B — sequential elements. Flop/Latch as members: read din (find_sink_pin(node,"din")), the q driver (get_driver_pin(0)), the reset value/polarity (reset_pin + negreset/async consts) into reset_cycle(), and honor the pipe_min depth as a shift-register of that many stages. Slop has no unknowns, so a flop with no reset randomizes (Slop::unknown(N)/rand_op) at reset_cycle() instead of carrying X. Memory → std::array<Slop<bits>, size> with the read/write ports modeled by mem_type (0 sync / 1 async / 2 array) inside cycle().
  • C — hierarchy (Sub instances). A Sub node becomes a nested struct member typed by its callee module name (node.get_subnode_io()->get_name()); the parent's cycle() calls child.cycle({…}), wiring the callee's input port decls from the Sub's sink pins and reading its outputs back from the returned struct (the get_input_pin_decls()/ get_output_pin_decls() ordering cgen already uses). Header #includes follow the instance graph; the --top graph is the root struct.
  • D — name↔member checkpoint table. Emit, per module, a {canonical-name → member} map and each register's declared width (fit_to), so the 2d-cex_debug cex.json replay can load_state/dump_state/set_inputs by name across layout/version. This generator is the natural source — no C++ reflection on the structs (see hlop/todo_checkpoint.md items 7–8). Reuses Slop::from_pyrope/to_pyrope for the values.
  • E — runnable driver + benchmark. A main.cpp + sim_main cc_binary in the emitted module: a reset/cycle loop with a register-state hash mode, the benchref/sample/slop design from hlop/todo.md item 3 as the first vehicle. Then LEC/cross-check against verilator / icarus / cpp_native (state-hash equality), and a perf number (cycles/sec).
  • F — latency selection (separate future pass, NOT in inou.cgen.sim). An LGraph→LGraph pass that, for a flop carrying a declared [pipe_min, pipe_max] range, seed-picks one concrete depth (--set …seed=N; default deterministic = min) and rewrites the flop so re-emitting explores legal timings to stress latency-insensitive / retiming logic. inou.cgen.sim consumes whatever depth the flops carry, so this pass composes in front of it with no changes to the generator. Relates to the combined LG pass1 stage checker (it narrows the range by sigma today).
