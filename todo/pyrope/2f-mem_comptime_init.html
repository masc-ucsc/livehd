2f-mem_comptime_init Comptime-computed / inferred-type memory init

Residual of 2f-mem_multidim (multi-dim memories + literal/scalar init contents landed 2026-06-12). A reg-array initializer must today be a tuple literal or a scalar broadcast: upass.tolg resolves the declare's init ref through tuple_recs_ (literal tuple_adds only), so a comptime-computed bundle is invisible to it, and a reg declare without an array type never reaches the memory path at all. The fix is upstream of tolg: the runner/coalescer must materialize a fully-comptime bundle initializer into a literal on the emitted declare (and, for the inferred form, synthesize the comp_type_array from the initializer's declared element type — the element envelope already rides the bundle's __elem_max/__elem_min attrs).

Failing examples (08-memories.md "Pyrope array syntax")

comptime mut reset_value:[3][8]u5 = nil
for i in 0..<3 {
  for j in 0..<8 {
    reset_value[i][j] = j
  }
}
reg mem2:[3][8]u5 = reset_value  // error: initializer must be a comptime constant or tuple literal
reg mem3 = reset_value           // error: reg initializer is not a compile-time constant (scalar-reg path)
