2f-enum_hier hierarchical enum: child entries must keep the parent's bits

For a hierarchical enum, a child entry's encoding must be parent_bits | new_child_bit — "the lower hierarchy level bits are kept". Parent-level encoding is already correct, but children do not OR in the parent bit: they get a standalone code. That single defect breaks the child int values, parent-equals-child subtree matching, and in / bitwise-union over hierarchical leaves. (The existing passing enum_hier.prp uses only flat enums, so nesting was never actually exercised.)

Encoding (doc-verbatim, 03-bundle.md:604-657)

enum Animal = (bird = enum(eagle, parrot), mammal = enum(rat, human))

int(Animal.bird)         == 0ub000001   // parent — OK today
int(Animal.bird.eagle)   == 0ub000011   // parent bit (1) | new bit (2)  — WRONG today
int(Animal.bird.parrot)  == 0ub000101
int(Animal.mammal)       == 0ub001000   // parent — OK today
int(Animal.mammal.rat)   == 0ub011000
int(Animal.mammal.human) == 0ub101000

Animal.bird == Animal.bird.parrot        // parent ⊂ child (shared parent bit) — WRONG today
const human_rat = Animal.mammal.rat | Animal.mammal.human
Animal.mammal     in human_rat           // bit-containment membership — WRONG today
not (Animal.bird  in human_rat)

Subtasks — landed

  • A — child entries inherit the parent bit.
    • Prp2lnast::lower_enum_def now threads a shared one-hot bit counter (DFS pre-order) plus a parent_bits mask through the hierarchy: a nested enum(...) entry claims a fresh bit first, then its children OR it in (parent_bits | (1 << bit)). The parent carrier IS the children bundle and records its own bare bit as a __enumval attr. Flat enums are unchanged (parent_bits == 0, one bit per entry = the old 1 << i).
  • B — parent-equals-child subtree match.
    • process_eq_ne_impl gained a bit-aware fast path: when BOTH operands are enum values, a hierarchical parent equals any descendant in its subtree (parent.bits & child.bits == parent.bits). Leaves, siblings, and ordinal/payload enums (never parents) keep EXACT equality, so flat-enum equality is not widened.
  • C — in / bitwise union over hierarchical leaves.
    • Enum membership now distinguishes the RHS: an enum-TYPE bundle (carries entry tags) keeps identity membership; a tagless scalar union (rat | human) uses bit-containment (x.bits & union == x.bits). int() on a parent reads its enumval attr (enum_scalar_of).
