2f-splice drop ++; make ... the only concatenation

The ++ / ++= operators are removed from the language. All concatenation, extension, and append now go through the ... splice operator, which inserts a tuple's fields into the surrounding tuple literal (or argument list). The docs in ../docs have been rewritten for this (uncommitted); the upstream tree-sitter-pyrope grammar already dropped ++, but the local ../tree-sitter-pyrope has not been synced yet — doing so before this task lands would make every ++ site a parse error.

This task supersedes the old TBD pointer 2f-inline_concat in 15-tbd.md (rename that reference to 2f-splice when the docs are committed). It also absorbs the adversarial findings that were really about concatenation: the tuple_append family and the array_nil_shape array-builder family.

The new model (from ../docs/docs/pyrope/03-bundle.md)

  • Merge: const c = (...a, ...b) — splice both into a new literal (field-name merge rules unchanged: duplicate field with differing values = error).
  • Extend: const y2 = (...y, const qq=2) — add fields to a copy.
  • Mid-literal insert: (1, ...mid, 6) (positional already works today).
  • Append idiom (replaces ++=): mut acc:[] = nil; for i in 0..<5 { acc = (...acc, i) }. A mut var that is already a tuple may self-splice (arity grows, base type unchanged).
  • Call-argument spread: const rest = (const b=2, mut c=3); foo(a=1, ...rest).
  • Self-splicing a non-tuple scalar stays a compile error ("a variable's type cannot change"). Already correct today — keep it.

Field kinds: a tuple-literal that is a declaration (the examples above, e.g. const rest = (const b=2, mut c=3)) needs a mut/const on each named field — that enforcement is 2f-mutconst (landed). Splice literals here must follow it; bare fields are still fine in call-argument spreads / comparisons / typed-var assignments.

Equiv test spec (subtask D)

// inou/prp/tests/equiv/splice_merge.prp
/*
:name: splice_merge
:type: equiv
:verilog_top: splice_merge.spl
:pyrope_top: splice_merge.spl
*/
comb spl(in1:u8, in2:u8) -> (out:u16) {
  const a = (mut p = in1, mut q = in2)
  const m = (...a, const r = 7)        // splice-merge with runtime fields
  out = (m.p + m.q + m.r) << 1          // out = (in1 + in2 + 7) * 2
}
// inou/prp/tests/equiv/splice_merge.v  (golden; verify with lgcheck)
module \splice_merge.spl (
  input  [7:0]  in1,
  input  [7:0]  in2,
  output [15:0] out
);
  assign out = ({8'd0, in1} + {8'd0, in2} + 16'd7) << 1;
endmodule
