2f-ufcs_ref_self result-bound ref self UFCS copy-return + const-tuple extension method

A result-bound -> (self) UFCS call (mut a3 = a1.set_x(x=20)) wrongly mutates the original receiver, and attaching an extension method to a const tuple (t1.double = fn) is rejected as a const rebind.

Scope (applies to every 2f-* adversarial-finding task)

  • Fix only the tests listed below. Do not fix unrelated failing adv_* tests.
  • The fix must not increase the total number of failing tests (run the full inou/prp suite).
  • Comptime / error tests only.

Failing tests

  • inou/prp/tests/comptime/adv_ufcs_05.prp — mut a3 = a1.set_x(x=20) mutates a1 (must be copy-by-value return)
  • inou/prp/tests/comptime/adv_ufcs_14.prp — t1.double = t1_do_double extension-method attach aborts as const t1 rebind

Root cause (verified)

ufcs_05: the comb inliner records a write-back for EVERY ref param including ref self and emits caller_var = <tag>self in the epilogue (upass_runner.cpp ~2196/2310). For a callee returning -> (self) whose result is bound to ANOTHER var, that self write-back is a copy-return and must be suppressed (06-functions.md:716-727). ufcs_14: t1.double = fn lowers to a 3-child field store; uPass_attributes::process_tuple_set calls record_assign(target="t1"), counting the field write as a second binding of the const root and aborting.

Proposed fix

(1) In try_inline_func_call, compute callee_returns_self and result_is_bound (reuse the dst-consumed scan at ~1677); skip the self param write-back when both hold (keeps in-place a1.set_x() and non-self ->(result) methods writing back). (2) In uPass_attributes::process_tuple_set (upass_attributes_tuple.cpp:183), charge the binding to the FIELD PATH (t1.double), not the const root — a new field has null type_info so record_assign returns early (attach allowed), a genuine const field still errors.

Difficulty / risk

Medium (conf 0.78). Fix 2 unblocks the abort but the casserts also depend on the double method field actually reaching t1's bundle through constprop's ST — verify post-fix. Keep tup_method.prp/setter_multi_field.prp/tuple_decorator_complex.prp green.

Done when

Both tests pass; the precedent UFCS/decorator tests stay green.
