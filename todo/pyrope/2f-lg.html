2f-lg lg attribute (explicit lgraph/module name)

pub comb foo::[lg="name"] pins the generated lgraph (and hence the netlist/Verilog module name), replacing the compiler-mangled file.entity name. Rules: pub-only, comptime string, lambda kinds only, non-sticky, renames the artifact but never the import key. No code handles the attribute today. Landing this also simplifies 2f-lgimport testing: its Pyrope-produced units can be imported via a pinned lg: name instead of the mangled one.

Failing example (check the emitted module name)

// chip.prp
pub comb my_top::[lg="chip_top"](a:u8, b:u8) -> (r:u9) { r = a + b }
lhd compile chip.prp --top my_top --emit verilog:chip.v --workdir tmp
grep -q "module chip_top" chip.v        # fails today (mangled name emitted)
# other files still: const f = import("chip.my_top")  — key unchanged
