2f-codeblock_expr code-block-as-expression does not yield its last expression

A { ... } code block used as an expression should evaluate to the value of its last expression. Instead the value is dropped: the binding folds to 0/nil. It breaks even the trivial single-expression block {7}, so this is the whole block-as-value lowering (the inner-scope last expression is not propagated out), not a scope/decl subtlety.

None.

Equiv test spec (subtask B)

The block depends on runtime inputs in1/in2 (its value cannot be constant-folded), and the block result feeds a shift so the output is a wide unambiguous non-negative integer (d = 300 + in1 keeps a = 300 + in1 - in2 ≥ 45 so no underflow / signed output).

// inou/prp/tests/equiv/codeblock_expr.prp
/*
:name: codeblock_expr
:type: equiv
:verilog_top: codeblock_expr.blk
:pyrope_top: codeblock_expr.blk
*/
comb blk(in1:u8, in2:u8) -> (out:u16) {
  const a = {mut d = 300 + in1; d - in2}   // block-as-expression: a = 300 + in1 - in2
  out = a << 2                              // out = (300 + in1 - in2) * 4
}
// inou/prp/tests/equiv/codeblock_expr.v  (golden; verify with lgcheck)
module \codeblock_expr.blk (
  input  [7:0]  in1,
  input  [7:0]  in2,
  output [15:0] out
);
  wire [15:0] a = 16'd300 + in1 - in2;   // always >= 45, no underflow
  assign out = a << 2;
endmodule
