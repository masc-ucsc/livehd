2f-comptime_param comptime [...] parameter slot in a lambda declaration

The documented comptime [...] parameter slot between a lambda name and its arg list (comb g[n:int=1](x) -> (r), 04-variables.md:48 / 06-functions.md:213-227) has no grammar production, so the declaration fails to parse ([...] is mis-parsed as a stray tuple_sq).

Scope (applies to every 2f-* adversarial-finding task)

  • Fix only the tests listed below. Do not fix unrelated failing adv_* tests.
  • The fix must not increase the total number of failing tests (run the full inou/prp suite).
  • Comptime / error tests only.

Failing tests

  • inou/prp/tests/comptime/adv_comptime_param_01.prp — comb g[n:int=1](x)->(r) declaration parse

Root cause (verified)

function_definition_decl (../tree-sitter-pyrope/grammar.js:504) declares only generic (<...>), pipe_config, input ((...)), and output — there is NO bare [...] comptime slot. The parser consumes [n:int=1] as a tuple_sq after the name, collapsing the statement into an ERROR ("syntax error: expected 'op_mul'"). The whole family is unimplemented, not regressed.

Proposed fix

Add a comptime [...] param slot to function_definition_decl between generic and input (matching [GENERIC] [COMPTIME] [INPUT]), reusing arg_list's item shape in square brackets; add the needed GLR conflict entries (comptime_param_list vs tuple_sq/array_length). This requires a grammar regen + MODULE.bazel pin bump. Then harvest the new comptime field in prp2lnast (mirror the generic-name + comptime mode-string machinery) and reinterpret the call-site override g[3](x=2) when the callee is a known comptime-parameterized lambda.

Difficulty / risk

Hard. Grammar change to a vendored/generated, pinned parser (regen + pin bump) plus multi-stage lowering. The grammar production alone clears the parse error; full comptime binding is a follow-on. Edit ../tree-sitter-pyrope with a local override (do not commit).

Done when

adv_comptime_param_01 parses and (with binding) folds; no regression in bracket-indexing / array-type / pipe[N] parses.
