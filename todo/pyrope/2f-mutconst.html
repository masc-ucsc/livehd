2f-mutconst a mut tuple literal's named field must declare mut/const

A named field inside a mut tuple literal must carry a kind keyword (mut or const). A bare ff = 1 in a mut tuple should be a compile error, but the front-end currently accepts it silently. (A const outer tuple may use bare named fields — they inherit const — so the rule is specific to mut literals.)

Correct vs incorrect

mut y = (ff=1, zz=3)              // ERROR: named fields in a mut tuple need a kind keyword
mut y = (mut ff=1, const zz=3)    // OK
const m = (a=1, b=0)              // OK: const outer — bare fields inherit const
