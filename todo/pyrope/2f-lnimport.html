2f-lnimport Test LNAST-level import (pyrope and slang units)

Plain import resolves at the LNAST level: the frontend translates it (process_import_statement), lhd scan lists imports, and the upass resolver consumes them — but no test exercises any of it (zero import tests in inou/prp/tests/). The imported unit's LNAST can come from a Pyrope file or from --reader slang Verilog. First of the import chain: 2f-lnimport → 2f-lgimport → 2f-defer.

Failing-coverage example (comptime, two pyrope files)

// lib_add.prp
pub comb my_add(a, b) -> (r) { r = a + b }
comb private_one(a) -> (r) { r = a }      // no pub: must not be importable

// top.prp
const lib = import("lib_add")             // all pub entries
cassert(lib.my_add(1, 2) == 3)

const f = import("lib_add.my_add")        // a single pub entry (dot selector)
cassert(f(a=3, b=4) == 7)
// import("lib_add.private_one")          // error: not pub

Failing-coverage example (slang unit, verilog/pyrope mix)

// add_sub.v — ingested with --reader slang (becomes an LNAST unit)
module add_sub(input [7:0] a, input [7:0] b,
               output [8:0] add, output [8:0] sub);
  assign add = a + b;
  assign sub = a - b;
endmodule
// top.prp — imports the slang-read unit at the ln level
const add_sub = import("add_sub")

mod top(a:u8, b:u8) -> (o:u9@[0]) {
  o = add_sub(a=a, b=b).add
}
// golden: module top(a, b, o); assign o = a + b; — LEC
