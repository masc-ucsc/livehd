2p Stable / deterministic SSA & tmp ids

Replace the opaque global-counter ___N SSA/tmp names with ids that are deterministic across recompilations and stable under small source edits. Derive each id from local, stable properties — the target variable's own name plus a counter scoped to that variable/block (e.g. foo_0, foo_1) — not from a single global monotonic counter.

Goal: adding or changing code ahead of some logic must not renumber the ids of the unrelated code that precedes it. A one-line change should produce a minimal diff in dumps/diagnostics, so golden files and incremental recompiles stay stable instead of churning end-to-end.

Non-goal — not a source-location map

This is deliberately not about encoding file:line:col into the id (the old foo_l42_a idea). Line/col-based names are the opposite of stable: inserting a line ahead shifts every downstream line number and rewrites every later id. Source positions stay available for diagnostics via the HHDS Source_locator / 1f source-map ID, carried separately from the SSA id rather than baked into it.
