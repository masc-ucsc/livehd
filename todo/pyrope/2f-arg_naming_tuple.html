2f-arg_naming_tuple compact tuple argument bound to a tuple-typed param (flatten/unwrap)

io_meta flattens a tuple-typed param/output into dotted-leaf entries (p:(first,second) → p.first,p.second), but the inliner epilogue, named-arg expander, and positional binder then treat those dotted leaves as caller-visible field keys — losing the single-tuple grouping and auto-unwrap.

Scope (applies to every 2f-* adversarial-finding task)

  • Fix only the tests listed below. Do not fix unrelated failing adv_* tests.
  • The fix must not increase the total number of failing tests (run the full inou/prp suite).
  • Comptime / error tests only.

Failing tests

  • inou/prp/tests/comptime/adv_mix_generic_tuple_03.prp — compact p=(first=,second=) mis-bound; output not auto-unwrapped (fcall-unknown-arg)
  • inou/prp/tests/comptime/adv_mix_generic_tuple_04.prp — positional tuple-type param xs:(int,int,int) lowers to an EMPTY tuple_add (zero io inputs)
  • inou/prp/tests/comptime/adv_overload_18.prp — single positional tuple literal (x=3,y=4) bound to a tuple param leaves p.y nil

Root cause (verified)

One defect, three surfaces. The multi-output inline epilogue (upass/runner/upass_runner.cpp:2298-2309) builds dst = (o.name=val, ...) with o.name = the flattened dotted leaf p.first, so constprop nests it as inner.p.first instead of auto-unwrapping the single output to inner.first; the named-arg expander (~1828) then builds "p"+"."+"p.first" and fails. Test 04: emit_arg_type (prp2lnast.cpp:5128-5161) only handles typed_identifier/typed_field/assignment items and SKIPS bare positional type elements (int), so a positional tuple-type param gets zero io leaves. Test 18: the positional binder (~1861-1969) does only scalar/kind disambiguation, binding the whole tuple to the first leaf.

Proposed fix

(A, primary — fixes 03) Output epilogue: group io.outputs by first dotted segment; if exactly one root, bind dst to that group stripping the root prefix (single tuple output → unwrapped (first=,second=)). (D — fixes 04) emit_arg_type: add an else-branch for a bare positional type element that synthesizes a positional-keyed arg-shape store (xs.0,xs.1,...). (C — fixes 18) Positional binder: when a single positional actual is a bundle and the remaining unset params share one dotted root, expand the bundle fields into those leaf params (mirror the named expansion).

Difficulty / risk

Medium-high. The output epilogue is on every callee return; must keep multi-scalar ->(b,c), single-scalar auto-unwrap, destructure, and the 1q/1r pipe/mod output-flop paths green. The positional-tuple classification must agree between the overload probe and the real bind. (Spec: 06-functions.md:350-363, 595/618.)

Done when

All three tests pass; fcall*/overload*/generic* stay green.
