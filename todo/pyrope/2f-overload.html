2f-overload Overload-gathering call dispatch

Pyrope overloads by gathering lambdas into a tuple and dispatching the call to the unique matching signature: const add = [add1, add2]; add(x). The init constructor overload sets already resolve (upass_runner), but general call dispatch over a gathered set does not.

Failing example (comptime)

comb add_int(a:int, b:int) -> (r:int) { r = a + b }
comb add3(a:int, b:int, c:int) -> (r:int) { r = a + b + c }

const add = [add_int, add3]

cassert(add(1, 2) == 3)        // dispatches to add_int (arity)
cassert(add(1, 2, 3) == 6)     // dispatches to add3
// add(1) — error: no matching overload
// ambiguous match — error: must be unique
