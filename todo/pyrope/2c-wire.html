2c-wire wire single-driver nets (replaces .[defer])

A new declaration kind alongside const/mut/reg. A wire is a single-driver combinational net: like mut it builds its value from if/match, but unlike mut (whose value is the last write in program order, and where read-before-write is an error) a wire may be read before its driver appears textually — every read sees the one resolved driver, position-independent (the Verilog continuous-assign / net model). wire x = nil forward-declares an undriven net; wire x = e declares and drives it.

This lets a designer close a module-interconnect ring without ordering gymnastics, and route a computed reset/flush into a register reset_pin — the two cases that today force inlining (no clean way to express the hazard/stage-register ring across module boundaries). It replaces .[defer], which is removed: name the value as a wire and feed both the register and the consumer.

Rules to enforce. Exactly one driver (a mux from if/match or mutually-exclusive conditional writes counts as one); a second unconditional assignment, an undriven wire (still nil), or an incompletely-driven wire are compile errors. wire removes only textual ordering, not cyclic dataflow: a wire combinationally driven by a function of itself is still a real combinational loop and must be rejected (SCC over combinational edges); rings are legal only when a register breaks them. NOTE: this is independent of the feedforward-stage-vs-state register classification (see the cycle-0-state default item) — wire fixes interconnect ordering, not latency.

Failing examples (pyrope/verilog pairs, sketch)

// ring.prp — a use precedes its driver; a reg breaks the cycle
mod ring(a:u8) -> (out:u8@[0]) {
  wire fb = nil
  reg  acc:u8 = 0
  const sum = a + fb     // reads fb BEFORE it is driven (legal: wire)
  out = acc
  acc = sum              // the reg breaks the loop
  fb  = acc              // fb's single driver, textually last
}
// ring.v — expected mapping (fb is just a net == acc; no flop for fb)
module ring(input clock, input reset, input [7:0] a, output [7:0] out);
  reg [7:0] acc;
  assign out = acc;
  always @(posedge clock) acc <= reset ? 8'h0 : (a + acc);
endmodule
// flush.prp — computed clear wired into reset_pin via a forward wire
mod flush(in:u8, do_flush:bool) -> (out:u8@[0]) {
  wire clr = nil
  reg  r:u8:[reset_pin = clr] = 0
  out = r
  r   = in                  // pure feedforward data path, no flush mux
  clr = reset or do_flush   // driver appears after the reset_pin use
}

Lowering work (upass / tolg / cgen)

  • Resolve a wire as a single-driver net with position-independent reads — decouple the lnast textual/SSA order from the dataflow edge so a read before the driver binds to the driver, not to nil.
  • Driver checks: error on a 2nd unconditional write, on an undriven (nil) wire, and on an incomplete conditional driver (a path that leaves it undriven).
  • Keep the combinational-loop SCC check active through wire edges (a comb self-loop is still an error; only a register may break the ring).
  • reset_pin = <wire>: connect a computed reset/flush net to a register's (synchronous) reset; works whether the wire is driven before or after the reg declaration.
  • Remove the .[defer] lowering path, its tests, and its docs; redirect to wire.
