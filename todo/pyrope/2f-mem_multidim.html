2f-mem_multidim Multi-dimensional memories & memory init contents

Two related gaps in chapter 08: multi-dimensional arrays/memories ([4][8]u8, row-major) and per-entry memory initialization / reset contents. cgen already carries mem-wrapper-missing and mem-multiclock-init unsupported diagnostics; no test covers either.

Failing example (comptime part)

mut b:[4][8]u8 = 13
cassert(b[2][7] == 13)
b[1][3] = 7
cassert(b[1][3] == 7 and b[2][7] == 13)

Failing example (pyrope/verilog pair — init contents)

// mem_init.prp
mod mem_init(raddr:u2) -> (q:u8@[0]) {
  reg mem:[4]u8 = (1, 2, 3, 4)   // per-entry reset contents
  q = mem[raddr]
}
// mem_init.v — should LEC-match (reset restores 1,2,3,4)
module mem_init(input clk, input reset, input [1:0] raddr, output [7:0] q);
  reg [7:0] mem [0:3];
  integer i;
  always @(posedge clk)
    if (reset) begin
      mem[0] <= 8'd1; mem[1] <= 8'd2; mem[2] <= 8'd3; mem[3] <= 8'd4;
    end
  assign q = mem[raddr];
endmodule
