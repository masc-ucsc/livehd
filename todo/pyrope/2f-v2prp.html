Current state ‚Äî the writer dies at the module boundary

upass/prp_writer was written to round-trip Pyrope-origin LNAST (driven by pass.prp_writer + the lhd_prp_writer_test / prp_writer_roundtrip_test golden round-trip). On slang-origin LNAST it fails on the very first node:

$ lhd compile --reader slang ./inou/prp/tests/equiv/trivial_if.v --emit-dir pyrope:tmp1/
$ cat tmp1/trivial_if.prp
/* TODO: unhandled node type 18 */

Node type 18 = io (lnast/lnast_nodes.def) ‚Äî the module-port declaration node. The Pyrope front-end never emits io, so write_node()'s dispatch switch has no case for it and funnels straight to the default /* TODO: unhandled node type ‚Ä¶ */ comment. Every Verilog input ‚Äî comb (trivial_if) and sequential (defer_reg) alike ‚Äî dies here, before any logic is reached. Today: 0 / 61 .v files round-trip.

The reference path is fragile on this corpus too: --reader yosys-slang ./inou/prp/tests/equiv/trivial_if.v fails ‚Äî yosys read_slang rejects the dotted escaped module name \trivial_if.fun3 ("Found control character or space ‚Ä¶ in RTLIL identifier"). The equiv modules are named <file>.<top>. So the comparison plumbing itself (subtask F) is part of the work.

Target output shape

The emitted Pyrope must look like the hand-written equiv goldens ‚Äî a named comb/mod lambda with typed IO:

// comb (combinational)
comb fun3(a:u3, b:u7, cond:boolean) -> (res:u4) {
  if cond { res = a + 1 } else { res = b#[0..<4] }
}

// mod (sequential): registers + per-output latency @[n]
mod dreg(a:u8) -> (o:u9@[0]) {
  reg r:u9 = 0
  r = a + 1
  o = r.[defer]
}

// memory: reg array
mod memdut(a:u8, wsel:u2, rsel:u2, we:bool) -> (z:u8@[0]) {
  reg t:[4]u8 = nil
  if we { t[wsel] = a }
  z = t[rsel]
}

Module name is <file>.<top> (e.g. trivial_if.fun3). Body refs are bare names that match the signature (modern Pyrope has no $/% IO sigils), so the writer's existing ref/const printing mostly works once the signature is emitted.

  • A,B ‚Äî module structure + lambda body. DONE. write_top() detects the slang-origin io child and calls the new write_module(), which emits pub comb|mod NAME::[lg="<file>.<top>"](in:T, ‚Ä¶) -> (out:T@[], ‚Ä¶) { ‚Ä¶ } and walks the top ‚Üí stmts body directly (declares hoisted above the attr writes). comb vs mod is chosen by body_has_state() (any reg/latch declare); port types come from render_type_at() (incl. comp_type_array ‚Üí [N]T).
  • C ‚Äî sequential / registers. DONE. Sync-reset regs keep their explicit body reset mux (reg r:T, no init ‚Äî the nil sentinel is suppressed, giving the flop no reset pin); clock/reset stay explicit io ports and tolg reuses them. Async regs fold the slang attrs onto the declaration (reg r:T:[init=N, reset_pin=rst, async=‚Ä¶, negreset=‚Ä¶]) with the importer‚Üísource attr remap (initial‚Üíinit, sync‚Üíasync inverted). tolg's lower_attr_set now honors the canonical source spellings async/init.
  • D ‚Äî memories. DONE. Reg arrays emit reg t:[N]T:[fwd=‚Ä¶]; the slang reader now harvests initial begin mem[k]=v end power-on contents (scalar broadcast or tuple literal) and folds them into the declare initializer. tolg re-drives the memory fwd mask in finalize_mems so an attr folded onto the declaration (emitted AFTER it by prp2lnast) still takes effect.

  • G ‚Äî generated naming. DONE. The writer pins the module name with ::[lg="<file>.<top>"] so the re-compiled module is byte-identical to the golden's (a non-dotted chip_top no longer doubles to chip_top.chip_top), letting a single --top drive both LEC sides.

Design notes

  • Two writers, only one in scope. pass/prp_writer/pass_prp_writer.cpp is the pass wrapper; upass/prp_writer/lnast_prp_writer.cpp (Lnast_prp_writer) is the actual walker ‚Äî that is the file to fix. Driven by lhd compile --emit-dir pyrope: ‚Üí emit_pyrope_outputs() in lhd/lhd_kernel.cpp.
  • The dispatch. write_node()'s switch is the entry point; it has no Lnast_ntype_io case (node 18) and no real func_def body ‚Äî both go to the default TODO comment. That is the whole reason slang output is empty.
  • Reuse what's there. render_type() already reconstructs uN/sN/bool/int from a prim_type_int range; pending_decl_ tracks storage-class keywords; write_declare / write_store / write_delay_assign / the infix/prefix helpers already emit most leaf statements.
  • Don't regress Pyrope-origin round-trip. lhd_prp_writer_test and pass/prp_writer/tests/prp_writer_roundtrip_test must stay green ‚Äî the writer still has to round-trip Pyrope-origin LNAST.
