2f-bw_maxmin .[bw_max]/.[bw_min] debug attribute reads

The bitwidth pass already tracks the actual value range per expression (bw_max/bw_min in upass_runner), but the attribute-read spelling is not wired. Declared-range reads (.[max]/.[min]/.[bits]/.[signed]) work. bw_* reads are debug-only: legal inside cassert/assert, a compile error anywhere non-debug (each elaboration may compute a different legal range — branching on it would not converge).

Failing example (comptime)

mut x:u8 = 30
cassert(x.[max] == 255 and x.[min] == 0 and x.[bits] == 8) // works today
cassert(x.[bw_max] == 30 and x.[bw_min] == 30)             // fails: read not wired

// must be a compile error (non-debug use):
if x.[bw_max] != 30 { x = 30 } else { x = 40 }
