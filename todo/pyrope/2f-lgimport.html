2f-lgimport Test lg: import (compiled lgraph units)

✅ LANDED 2026-06-14. The two-step lg: flow now works and is a checked-in test (inou/prp/tests/imports/lg_import_test.sh → bazel prp-import-lg): compile a unit into an lgdb, then import("lg:name") from a Pyrope source and LEC the result. Fixes: (1) compile_command ignored lg: inputs on the pyrope path — it now load_merges each lg: input library into the working library before tolg (the same linker mechanism the ln:+lg: elaborate path uses), so import("lg:name") resolves the pre-compiled graph by name and the call lowers to a Sub instance. (2) A wrong port name at the call site asserted inside resolve_sink_port; tolg now pre-checks gio->has_input and emits a clean port-mismatch diagnostic. Covered: slang-producer two-step LEC (o == a+b), not-found diagnostic, port-mismatch diagnostic, and an lg: unit from a Pyrope producer (::[lg="name"] for a stable name vs the mangled file.entity). Full prp suite 478/478, no regressions. Unblocks 2f-defer's mix test.

import("lg:name") imports an already-compiled lgraph — e.g. a Verilog module compiled earlier, or a previously built Pyrope unit — instead of an LNAST unit. Some lg: linking machinery exists (see the 3y-agent residuals), but there is no basic import-from-lg test. Middle of the import chain: 2f-lnimport → 2f-lgimport → 2f-defer (whose mix test needs exactly this).

Nice-to-have dep

  • 2f-lg (landed) — the ::[lg="name"] attribute pins the generated lgraph name, so the Pyrope-produced half of the test can import("lg:lib_add") against a stable, predictable name instead of the compiler-mangled file.entity form. Not blocking (the mangled name works), but it simplifies the test setup and keeps the goldens readable.

Failing-coverage example (two-step, verilog/pyrope mix)

# step 1: compile the Verilog module into the lgdb
lhd compile add_sub.v --reader slang --top add_sub --workdir tmp
// add_sub.v
module add_sub(input [7:0] a, input [7:0] b,
               output [8:0] add, output [8:0] sub);
  assign add = a + b;
  assign sub = a - b;
endmodule
// top.prp — step 2: import the compiled lgraph, not the source
const add_sub = import("lg:add_sub")

mod top(a:u8, b:u8) -> (o:u9@[0]) {
  o = add_sub(a=a, b=b).add
}
// golden: assign o = a + b; — LEC
// import("lg:no_such_lg") — error: clear "lgraph not found" diagnostic
