1f-is_typename remove the is/!is nominal operator and the .[typename] attribute

Pyrope drops nominal type identity entirely. Structural checks (does, equals, case) become the only type-comparison operators. The is/!is operator — which lowered to a typename-attribute equality (a.[typename] == b.[typename]) — and the .[typename] attribute it depended on are both removed.

This is a pure removal/simplification: it deletes a compare-tier operator, an LNAST node type, and a tracked attribute, shrinking the front-end and several tests. It is filed in Group 1 so the comparison-operator tier and the grammar are already trimmed before the Group 2 grammar rework (2f-splice).

None. Do before 2f-splice (the other grammar-touching task) so the binary_compare_op tier is settled first.
