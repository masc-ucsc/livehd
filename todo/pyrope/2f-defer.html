2f-defer .[defer] end-of-cycle reads

✅ LANDED 2026-06-14 (lgraph lowering + cross-language feedback). Per the user's model, .[defer] is a deferred lgraph edge: don't wire it at the read site; remember (consumer ← base) and connect the base's LAST dpin after every statement is lowered. tolg (lower_attr_get → single-driver Or passthrough buffer + resolve_defers after finalize_regs): a plain var resolves to its final SSA driver (logical_last_, SSA versions collapsed), a reg to its accumulated din (next-state), never q. The runner keeps the defer base un-folded (emit_op_with_fold); the frontend exempts it from the undefined-read check (it may reference a later/same-cycle write) and rejects the LHS write form (defer-lhs-write). Cross-language feedback: the verilog/pyrope mix (c = tmp.add.[defer]) wires + LECs as (a+b)-3. A defer edge may legally close a same-cycle cycle (Verilog permits comb loops; a later lgraph pass detects/handles real ones), so the Time_checker cuts each defer buffer's in-edge (defer_cut_nids_) rather than flagging the loop — no per-output analysis needed. Tests: equiv defer_comb, defer_reg, and the shell test defer_mix_test.sh (bazel prp-defer-mix). Bonus tolg fixes uncovered: a multi-output lg: call bound to a named var resolves its fields (aliased sub_results_) and a forward field read is deferred to end-of-pass (pending_tgets_). The comptime casserts + slop/codegen for .[defer] are a separate future item (per the user).

x.[defer] is an RHS-only read of the value a variable/register holds at the end of the current cycle, after all in-cycle writes have accumulated. It is same-cycle wiring, not a temporal construct — no flop. There is no x.[defer] = … write form. Today the frontend recognizes defer only as an attribute name; nothing resolves the read.

Depends on: 2f-lnimport → 2f-lgimport → this task (the mix test below imports the Verilog add_sub as an lgraph).

Failing example (comptime)

mut x = 100
mut a = 1
x = a.[defer]   // reads a's end-of-cycle (final) value
a = 200
cassert(x == 200)

reg r:u8 = 1
if r == 1 {
  r = 50
  cassert(r.[defer] == 50)  // bare `r` still reads q (1)
}

Failing example (verilog/pyrope mix — connect a call's output to its own input)

.[defer] lets a single call wire one of its outputs back into one of its inputs. This is legal whenever the callee has no comb path from that input to that output — here add depends only on (a,b) and sub only on (c,d), so feeding add → c is acyclic at the netlist level even though the Pyrope statement is textually self-referential. The callee is plain Verilog (slang reader), so the per-output dependency analysis must cross the language boundary.

// add_sub.v — common Verilog module, two independent functions
module add_sub(input  [7:0] a, input [7:0] b,
               input  [8:0] c, input [7:0] d,
               output [8:0] add, output signed [9:0] sub);
  assign add = a + b;   // depends on (a,b) only
  assign sub = c - d;   // depends on (c,d) only
endmodule
// test_defer.prp
const add_sub = import("lg:add_sub")   // the compiled Verilog module (see 2f-lgimport)

mod test_defer(a:u8, b:u8) -> (out:i10@[0]) {
  mut tmp = add_sub(a=a, b=b, c=tmp.add.[defer], d=3)
  // tmp.add = a + b
  // tmp.sub = tmp.add - 3
  out = tmp.sub        // out = a + b - 3
}

// negative case — MUST be a compile error (true comb cycle: sub depends on c):
// mut bad = add_sub(a=a, b=b, c=bad.sub.[defer], d=3)
// test_defer_gold.v — should LEC-match
module test_defer(input [7:0] a, input [7:0] b, output signed [9:0] out);
  assign out = {1'b0, a} + {1'b0, b} - 10'sd3;
endmodule
