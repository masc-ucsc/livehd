2f-ref_wrap_sat wrap/sat lost on a ref (inline+alias) write; ref on mod/pipe must error

ref in a comb means the comb is inlined and the ref parameter is replaced by the caller's variable itself (an alias, not a copy), so writes to it mutate the caller's value. That inline-alias drops the wrap/sat qualifier: a plain write through a ref propagates, but a wrap/sat-qualified one does not (the same qualifier folds fine at top level). Separately, ref is comb-only: on a mod/pipe boundary it must be a compile error, but is silently accepted today.

None.

A — wrap/sat must survive the ref inline-alias

When the comb inliner replaces a ref param with the caller's variable, the wrap/sat qualifier on the write must ride along (it becomes a wrap/sat write to the caller's variable — which already folds correctly at top level). Plain ref writes already work; only the qualified ones are lost.

comb inc(ref a:u8) -> () { wrap a += 1 }
mut v:u8 = 5
inc(ref v)
cassert(v == 6)        // today: cassert false (wrap write not applied to v)
// control (works today): the same comb with plain `a += 1` propagates.

  • inou/prp/tests/comptime/adv_ufcs_21.prp — wrap/sat self.v += x in a ref self method (250+10 → wrap 4 / sat 255).
  • inou/prp/tests/comptime/adv_init_ctor_18.prp — wrap/sat self.field = v in an init body (v=17 → wrap 1 / sat 15; stored verbatim today).

B — ordinary ref on a mod/pipe parameter is a compile error

Ordinary non-self ref requires inlining + aliasing, which only a comb does. A mod/pipe is a real boundary, so a non-self ref parameter there must be rejected. Today it is silently accepted. ref self methods are the exception: a UFCS call such as foo.f1(...) expands locally against the receiver and does not expose the receiver as a Verilog module port.

Tests to fix (already failing — "no error emitted"):

  • inou/prp/tests/errors/adv_ref_mod.prp — mod m(ref a:u8).
  • inou/prp/tests/errors/adv_ref_pipe.prp — pipe[1] p(ref a:u8).

The :error: regexes are placeholders (ref) — align to the real diagnostic message.
